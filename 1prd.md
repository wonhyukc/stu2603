# Product Requirements Document (PRD): 수강 과목 공지 및 실라버스 관리 시스템 — 프로젝트 지시사항

## 1. 프로젝트 개요 (Overview)
본 프로젝트는 서울신학대학교 컴퓨터소프트웨어학과 1학년 대상의 교육 과정에 대한 커리큘럼 설계, 수강 공지 작성 및 등록 절차를 다루는 프로젝트다.

### 1.1 대상 과목 트랙
1.  **컴퓨팅적사고와코딩(K트랙)[04]**: Python 기반 (한국어 진행)
2.  **웹프로그래밍(E트랙)[01], [02]**: HTML/CSS/JavaScript 기반 (영어 진행)

### 1.2 핵심 제약 조건 (모든 작업에서 항상 인지할 것)
*   **대상자**: 컴퓨터 교육을 처음 받는 1학년 학생들 (특히 E트랙은 비개발자 지식 수준의 국제학부생 다수 포함). 모든 콘텐츠는 최대한 직관적이고 쉬워야 한다.
*   **개발 환경**: 로컬 환경 설치 오류로 인한 시간 낭비를 막기 위해 **초반(1~8주차)은 클라우드 IDE(StackBlitz, Colab)**를 전면 사용하며, **중간고사 이후(9주차~)에 VS Code 설치 및 로컬 환경**으로 전환한다.
*   **교육 철학**: 프로그래밍 문법 암기보다는 AI(Gemini, ChatGPT 등)를 파트너로 활용해 논리적으로 문제를 해결하는 **"AI-Native 컴퓨팅 사고"**와 **Flipped Learning(거꾸로 수업)**을 지향한다.
*   **자동화 및 등록**: LMS 등의 다운로드 및 등록은 AI 자동 스크립트가 아닌, 준비된 텍스트를 대상 시스템에 수동 복사-붙여넣기하여 진행한다.

## 2. 디렉토리 구조 및 파일 네이밍 규칙

### 2.1 주요 디렉토리 구조
프로젝트 루트 디렉토리에는 핵심 관리 문서가 위치하며, 각 주차별 자료는 과목 식별자 폴더로 분리한다.

```
stu2603/
├── 1prd.md                                # 이 파일 (전체 요구사항 및 지시사항)
├── 3history.md                            # 전체 작업 내역 기록 (통합 관리)
├── 웹프로그래밍(E트랙)_교육안.md                # E트랙 주차별 커리큘럼 마스터 (SSOT)
├── 컴퓨팅적사고와코딩(K트랙)_교육안.md           # K트랙 주차별 커리큘럼 마스터 (SSOT)
├── wb01/                                  # 웹프로그래밍 1주차 폴더
│   ├── lecture.md                         # 강사용 진행 가이드/구조
│   ├── lecture-script.md                  # 대본형 강의 스크립트 (Bilingual)
│   ├── lab.md                             # 실습용 워크시트/핸즈온 가이드
│   ├── homework.md                        # 과제 안내
├── py01/                                  # Python 1주차 폴더
├── tmp/                                   # 임시 및 보조 파일 폴더
│   └── health-report/                     # 품질 검증 리포트(Quality Report) 저장 위치
```

### 2.2 디렉토리/파일 네이밍 규칙
*   **주차별 폴더명**: `wb` (웹) 또는 `py` (파이썬) + 주차 번호 2자리. (예: `wb01`, `py05`)
*   **문서 파일명**: 영문 아스키 형태의 kebab-case 또는 소문자 명사 결합을 사용한다. (`lecture-script.md`, `homework.md`)
*   **커리큘럼 연계**: 과정 변경 시 반드시 해당 과목의 `~_교육안.md` 마스터 파일을 먼저 수정한 후 주차별 하위 문서를 수정/생성해야 한다.

## 3. SSOT (Single Source of Truth) 참조표

**[SSOT 체크리스트 필수화]**
모든 생성 작업(프롬프트, 에이전트 스킬 등)의 첫 줄에는 다음 지시사항을 강제로 삽입해야 합니다:
> **"Read `1prd.md` and `.agents/rules/local-rules.md` first"**
AI 에이전트는 작업을 시작하기 전 반드시 위 두 문서를 로드하여 맥락을 동기화해야 합니다.

동일한 정보의 중복 및 불일치를 막기 위해 아래의 정본 위치를 따른다.

| 정보 | 정본 위치 |

| 전체 프로젝트 규칙, 제약 사항, 워크플로우 | `1prd.md` (루트) |
| E트랙(웹) 주차별 커리큘럼 및 진도표 | `웹프로그래밍(E트랙)_교육안.md` |

## 5. 워크플로우 및 난이도 기준선

### 5.1 운영 워크플로우
모든 작업은 **GitHub Issue → Branch → PR → Merge** 흐름으로 진행하는 것을 원칙으로 한다.

*   **브랜치**: `issue-{번호}-{짧은-설명}` (main 직접 커밋 금지)
*   **커밋 메시지**: `[타입] 작업 내용 (#이슈번호)` (타입: feat / fix / docs / refactor / chore)
    *   *중요*: 내부 시스템 호환성을 위해 커밋 메시지는 **무조건 한국어(한글 전용)**로 작성한다.
*   **PR**: `Closes #번호` 포함, Squash merge 사용.
*   **세션 시작**: `gh issue list`로 열린 이슈 확인 → 진행 상태 보고 → 작업 착수.

### 5.2 문서 관리 및 검증 규칙
1.  **루트 중심 관리**: 하위 폴더(`wb01`, `py01` 등)에는 별도의 `1prd`, `3history` 등의 관리 문서를 두지 않으며, 루트의 `1prd.md`와 `3history.md`에서 통합 관리한다.
2.  **검증 의무화**: 과정 내용이나 지시사항을 변경·생성한 뒤, 즉시 해당 내용이 각 주차별 실습 및 과제 요건에 부합하는지 검수 과정을 병행한다.

### 5.3 난이도 및 분량 기준선
*   **분량 기준**: 매 주차 생성되는 핵심 강의안(`lecture.md` 및 대본)은 강사 리딩을 기준으로 **총 75분(25분 x 3개 섹션)** 분량이 되도록 설계한다. 학생 개인 코딩 실습이나 Q&A 시간은 강의안 75분 타임라인에 포함하지 않으며, 필요에 따라 강사의 코딩 데모(시연)만 강의 내용에 포함할 수 있다.
*   **Phase 1~2**: 컴퓨터를 처음 접하는 학생의 눈높이에 맞춘 기초 교육 과정.
*   **Phase 3**: AI 도구를 "사용하는" 수준 (직접 코드를 짜는 단계를 넘어 AI 활용에 집중).
*   **Phase 4**: 1~3에서 배운 것을 조합하여 간단한 프로젝트에 적용하는 단계.

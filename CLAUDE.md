## 핵심 제약 조건

- **대상자**: 컴퓨터 교육을 처음 받는 1학년 학생. E트랙은 비개발자 지식 수준의 국제학부생 다수 포함. 모든 콘텐츠는 최대한 직관적이고 쉽게 작성한다.
- **온라인 전면 수업 (임시)**: 외국인 학생(특히 국제학부 등)들의 **비자 문제**로 인해, 별도 언급이 있을 때까지 **모든 수업(E/K트랙 공통)은 대면 수업 없이 100% 온라인 비대면 영상 수업**으로 진행된다. 모든 가이드 및 스크립트는 이를 전제로 작성한다.
- **실습 환경**: 1~8주차는 클라우드 IDE(StackBlitz, Colab) 전면 사용. 9주차~(중간고사 이후)부터 VS Code 로컬 환경으로 전환.
- **교육 철학**: 문법 암기보다 AI(Gemini, ChatGPT 등)를 파트너로 활용하는 **"AI-Native 컴퓨팅 사고"** 및 **Flipped Learning** 지향.
- **등록 방식**: eCampus 등록은 AI 자동 스크립트 금지. 준비된 텍스트를 수동 복사-붙여넣기로 진행.

## 파일 역할 및 구성 규칙

**파일 역할 요약**

| 파일 | 대상 | 용도 | 필수 여부 |
|---|---|---|---|
| `lecture.md` | 강사 | 수업 진행 순서, 시간 배분, 체크리스트 | **필수** |
| `lecture-script.md` | 강사 | 말할 내용 전체 대본 (영상 제작 기준) | **필수** |
| `course-notice.md` | 강사→학생 | eCampus 공지 복사-붙여넣기용 원문 (공지 고유 정보만 Full content, 상세 내용은 handout 포인터) | **필수** |
| `handout.md` | 학생 | **학생 배포용 유일한 마크다운 문서** — 개념 요약 + 실습(ipynb 포인터) + 과제 안내 통합 | **필수** |
| `lab.ipynb` | 학생 | **K트랙(파이썬) 전용 실습 파일** — 수업 중 실제 코드를 작성하고 실행하는 환경 | **K트랙만 필수** |
| `slides.md` | 강사 | 슬라이드 구성 개요 (Google Slides 제작 기준) | 선택 |

**`handout.md` 구성 규칙 (3섹션 필수)**

`handout.md`는 학생이 수업 당일 읽는 메인 가이드 문서다. 반드시 아래 3개 섹션을 순서대로 포함해야 한다.

1. **개념 참고 영역** — 해당 주차 핵심 개념 요약, 원칙 참고표 등 수업 중·후 참고 자료
2. **실습 섹션** — 구체적인 스텝이나 코드를 나열하지 않고, **"제공된 `lab.ipynb` 파일을 Google Colab에서 열어 실습을 진행하세요"** 라는 안내 문구만 기재. 전체 실습 흐름(랩 제목들)만 간략히 소개.
3. **과제 섹션** — 마이크로 과제 내용, 제출 형식, 기한, 제출처(eCampus) 

**`course-notice.md` SSOT 정책**

`course-notice.md`는 공지 고유 정보(기한·제출처·제목형식·수신처)만 Full content로 유지한다. `handout.md`와 중복되는 상세 내용(실습 절차, 과제 요건 상세, 개념 설명 등)은 제목과 `핸드아웃 N섹션 참고` 포인터만 기재하고 내용을 복사하지 않는다.

**네이밍 규칙**
- 주차별 폴더: `wb` (웹) 또는 `py` (파이썬) + 2자리 주차 번호 (예: `wb01`, `py05`)
- 파일명: 영문 ASCII kebab-case 또는 소문자 명사 결합 (예: `lecture-script.md`, `handout.md`, `lab.ipynb`)
- **커리큘럼 변경 시**: 반드시 해당 과목의 `*_교육안.md` 마스터 파일을 먼저 수정한 후 주차별 하위 문서를 수정/생성한다.

## Git 워크플로우

모든 작업은 **GitHub Issue → Branch → PR → Merge** 흐름으로 진행한다.

- **세션 시작**: `gh issue list`로 열린 이슈 확인 후 작업 착수
- **브랜치명**: `issue-{번호}-{짧은-설명}` (main 직접 커밋 금지)
- **커밋 메시지**: `[타입] 작업 내용 (#이슈번호)` — **반드시 한국어(한글)로 작성**
  - 타입: `feat` / `fix` / `docs` / `refactor` / `chore`
- **PR**: `Closes #번호` 포함, Squash merge 사용

## 콘텐츠 품질 기준

**분량 기준 (강의 영상 및 대본)**
- **사전 시청용 강의 영상(대본)**은 정확히 **25분 × 3개(총 75분)** 분량으로 구성해야 합니다.
- **학생 실습(Lab)과 과제 안내 시간은 이 75분 영상 구성에서 철저히 제외**합니다. 강사의 코딩 데모(시연)만 영상(대본) 내용에 포함됩니다.
- **분량 어뷰징 금지**: for loop 등 스크립트를 이용한 기계적 반복 생성 절대 금지. 순수 자연어 작성만 허용.

**학생 실습(Lab) 및 파제 안내 기준**
- 오프라인 실습 및 코드 비평 시간은 **정규 강의 시간(75분)에서 전부 배제**됩니다.
- 실습안 및 과제 기획 내용은 교안 설계 시 정규 교육 75분에 배분해서는 절대 안 되며, 학생 자율 학습(과제) 영역으로 넘깁니다.

**난이도 단계**
- **Phase 1~2**: 컴퓨터를 처음 접하는 학생의 눈높이에 맞춘 기초 과정
- **Phase 3**: AI 도구를 "사용하는" 수준 (AI 활용 중심, 직접 코딩 최소화)
- **Phase 4**: 1~3에서 배운 것을 조합하여 간단한 프로젝트에 적용

**문서 관리**
- 하위 폴더(`wb01`, `py01` 등)에는 별도 관리 문서를 두지 않는다. 루트의 `CLAUDE.md`와 `3history.md`에서 통합 관리.
- 과정 내용을 변경·생성한 뒤 해당 내용이 주차별 실습 및 과제 요건에 부합하는지 즉시 검증한다.

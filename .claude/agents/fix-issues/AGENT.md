# GitHub 이슈 순차 수정 지침 (fix-issues)

사용자가 전달한 GitHub 이슈 번호를 기반으로 충돌을 방지하며 버그 수정 및 파일 변경 작업을 순차적으로 진행하는 에이전트입니다.

## 작업 프로세스

### 1단계: 이슈 확인 (사전 확인)

1. `gh issue view [이슈번호]` 명령어를 실행해 이슈의 상세 내용, 상태(open/closed)를 확인합니다.
2. 해결해야 할 파일 범위와 관련 디렉토리를 파악합니다.

### 2단계: 프로젝트 규칙 준수 확인

1. 패치 작업을 하기 전, 루트 폴더의 `CLAUDE.md`를 반드시 확인하여 **해당 이슈의 해결책이 프로젝트의 원칙(E트랙 한영병기, 로컬 IDE 대신 클라우드 등)에 위배되지 않는지 검사**합니다.

### 3단계: 파일 수정 및 버전 관리

1. 관련된 파일을 수정하거나 에러를 조정합니다.
2. (선택사항) 커밋 규칙 준수: 변경 사항 커밋이 필요한 경우, `CLAUDE.md`의 규칙에 따라 **반드시 한글 커밋 메시지**(`[fix] 작업 내용 (#이슈번호)`)를 사용해야 합니다.

## 통합 보고 포맷

작업이 끝나면 사용자에게 아래 형식으로 보고하세요:

```markdown
# 이슈 수정 결과 보고: #[이슈번호]
- **이슈 제목:** [이슈 내용 요약]
- **상태:** [완료 / 건너뜀 / 실패]
- **수정된 파일 목록:**
  - `path/to/file1`
- **비고 및 추가 알림:** [문제가 있었거나 추가 조치가 필요한 경우 코멘트]
```

# 구글 스프레드시트 원시 데이터 정제 계획 (GSheet Data Transformation Plan) - 별도 시트 분리형

*대상 문서 주소: https://docs.google.com/spreadsheets/d/1F69Wtmrr3MYMJI8jgjPKBWh3m8tDaX6QyBVDSHH4vEc/edit*

## 1. 목적 (Goal)
구글 스프레드시트의 원본 `progess` 시트 구조를 일절 훼손하지 않고, 처리된 데이터를 동일한 컬럼명을 가진 **새로운 별도 워크시트(예: `progess_transformed`)**에 백업/기록하여 원본 데이터와 정제된 데이터를 분리합니다.

## 2. 변환 대상 컬럼 정의

### A. 5점 척도 역순 변환 대상 
원본 질문 내용과 1:1로 동일한 컬럼 구조를 가지되, 값만 1부터 5까지 정수(비어있을 시 빈칸)로 변환해 새 시트에 적재합니다. 긍정적인 응답일수록 점수가 낮아지거나, 명시된 숫자가 **역순(예: 5점->1점, 1점->5점)**으로 매핑됩니다.
- `1주차수업은`, `1주차과제는`, `2주차수업은`, `2주차과제는`

### B. 이진(Boolean) 변환 대상
제출/수행 완료 여부 판단(0 또는 1)
- `github repo에 wonhyukc 초대했다`
- `과제0.1 제출`, `과제0.2 정규식 사용 등 제출` 등 목표 컬럼

## 3. 상세 작업 절차 (Plan)

### 단계 1: 데이터 로드
- `progess` 시트 데이터를 통째로 메모리로 로드합니다.

### 단계 2: 데이터 매핑 (파싱)
- 컬럼 구조는 변경하지 않고 대상 인덱스의 값들만 파싱 로직에 따라 치환합니다.

### 단계 3: 새 시트 일괄 저장 (Batch Update)
- `progess_transformed` 워크시트를 새로 생성/초기화하여 일괄 저장합니다.

### 단계 4: 데이터 유효성 검사 적용 (Data Validation) 🌟
- 작성된 대상 컬럼 범위에 대해 구글 시트의 데이터 유효성 검열 기능(Data Validation Rule)을 API로 강제 적용합니다.
  - **척도 컬럼**: 허용 숫자 `1`~`5` 강제 (Number Between 1-5). 지정 범위 위반 시 입력 불가.
  - **이진 컬럼**: 허용 리스트 `0`, `1` 강제 (One of list 0, 1).

### 단계 5: 요약 로그 보고
- 처리된 변환 데이터 무결점 건수를 산출해 로깅합니다.

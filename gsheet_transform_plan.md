# 구글 스프레드시트 원시 데이터 정제 계획 (GSheet Data Transformation Plan)

## 1. 목적 (Goal)
구글 스프레드시트의 `progess` 시트 내 서술형 및 비정형 응답을 정량적 수치(5점 척도 및 Boolean)로 자동 변환하여, 분석이 용이한 데이터 구조를 만듭니다.
* 대상 문서: [Google SpreadSheet](https://docs.google.com/spreadsheets/d/1XA5Hnu5PEidFMreanPCy1eMrofGiyFDPo4buN3129Mc/edit?gid=1889091726#gid=1889091726)

## 2. 변환 대상 컬럼 정의

### A. 5점 척도 변환 대상 (`target_5_scale`)
다음 컬럼들은 응답을 해석하여 1부터 5까지의 수치로만 입력합니다. (5: 긍정적 점수 ~ 1: 부정적 점수)
- `gemini live 사용`
- `한국어실력`
- `컴퓨터 수준`
- `타이핑`
- `1주차수업은`
- `1주차과제는`
- `2주차수업은`
- `2주차과제는`

### B. 이진(Boolean) 변환 대상 (`target_boolean`)
다음 컬럼들은 제출 여부에 따라 1(제출) 또는 0(미제출)으로 해석하여 입력합니다.
- `과제0.1 제출`
- `과제0.2 제출`

## 3. 상세 작업 절차 (Plan)

### 단계 1: 데이터 로드
- `gspread` 라이브러리와 서비스 계정 인증을 통해 구글 스프레드시트에 연동합니다.
- `progress` 시트의 전체 데이터를 메모리 상에 읽어옵니다(예: Pandas DataFrame).

### 단계 1.5: 중복 실행 검사 (Idempotency) 🚨
- 기존 컬럼 이름 중에 이미 **`org_` 접두어가 포함된 컬럼이 존재하는지** 확인합니다.
- 존재할 경우 이미 변환된 시트로 간주하고, 스크립트 실행을 즉시 중지하여 데이터를 오염시키지 않습니다.

### 단계 2: 컬럼 구조 변경
- 변환 대상 컬럼들의 기존 열 이름 앞에 **`org_` 접두어**를 추가하여 변경합니다. (예: `한국어실력` -> `org_한국어실력`)
- `org_`가 붙은 원본 컬럼의 바로 오른쪽 위치에 원래 이름(`한국어실력`) 그대로의 **새로운 빈 컬럼을 추가**합니다. 

### 단계 3: 데이터 파싱 및 변환 적용 (Transformation)
- **주의:** LLM(언어 모델) API 등 외부 연동은 일체 사용하지 않으며, 전적으로 순방향/역방향 규칙 매핑만을 적용합니다.
- **규칙 1 (5점 척도):** `org_*` 컬럼의 원문 텍스트 내 특정 키워드에 따라 1~5 정수 형태로 할당합니다.
  - **순방향 (Forward):** "쉽다", "잘한다" 등 긍정적 뉘앙스인 경우 순방향으로 점수를 부여합니다.
  - **역방향 (Reverse):** "질문이 어렵다", "못한다" 등 부정적 뉘앙스인 경우 역방향으로 1~5의 점수를 부여합니다.
  - **파싱 오류 (Fallback):** 예상할 수 없는 답변이나 파싱이 불가능한 경우 임의로 값을 채우지 않고 **그냥 빈칸(결측치)** 으로 남겨둡니다.
- **규칙 2 (이진 변환):** "제출", "했어요" 등의 경우 1 할당, 빈칸이나 "미제출" 등은 0 할당.

### 단계 4: 일괄 스프레드시트 반영 (Batch Update)
- 시트의 셀을 하나씩 개별 열이나 행 단위로 갱신하면 API 속도 제한 오류(Rate Limit)가 발생하므로, 메모리에서 변환된 구조의 데이터를 구글 시트에 한 번에 **일괄 덮어쓰기(Batch Update)** 방식으로 반영합니다.

### 단계 5: 결과 검증 (Verification)
- 원본 `org_*` 컬럼의 서술형 응답 데이터 원문과 척도화/이진화된 새 컬럼의 데이터(1~5, 0/1, 빈칸) 간 논리적 일치 여부를 무작위 샘플링(Random Sampling)하여 교차 검증합니다.
- **검증 결과 보고:** 무작위로 추출한 샘플 데이터가 총 몇 건인지, 그 중 논리적 일치 여부(정확도)는 몇 % 인지 산출하여 스크립트 실행 완료 시 터미널 로그로 텍스트를 출력(보고)합니다.

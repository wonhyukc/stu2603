# 구글 스프레드시트 원시 데이터 정제 계획 (GSheet Data Transformation Plan) - 별도 시트 분리형

## 1. 목적 (Goal)
구글 스프레드시트의 원본 `progess` 시트 구조를 일절 훼손하지 않고, 처리된 데이터를 동일한 컬럼명을 가진 **새로운 별도 워크시트(예: `progess_transformed`)**에 백업/기록하여 원본 데이터와 정제된 데이터를 완전히 분리합니다.

## 2. 변환 대상 컬럼 정의

### A. 5점 척도 변환 대상 (`target_5_scale`)
원본 질문 내용과 1:1로 동일한 컬럼 구조를 가지되, 값만 1부터 5까지 정수(비어있을 시 빈칸)로 변환해 새 시트에 적재합니다.
- `gemini live 사용`, `한국어실력`, `컴퓨터 수준`, `타이핑`, `1주차수업은`, `1주차과제는`, `2주차수업은`, `2주차과제는`

### B. 이진(Boolean) 변환 대상 (`target_boolean`)
제출 여부 판단(0 또는 1)
- `과제0.1 제출`, `과제0.2 정규식 사용 등 제출` (구체적 열 이름은 코드 의존)

## 3. 상세 작업 절차 (Plan)

### 단계 1: 원본 데이터 로드 (Read-Only)
- `gspread` 서비스 계정 인증(`secret.json`)을 통해 스프레드시트에 연동합니다.
- `progess` 시트의 전체 데이터를 `get_all_values()`를 통해 메모리로 읽어옵니다. 이 단계에서는 시트를 어떠한 형태로도 수정하지 않습니다.

### 단계 2: 새 워크시트 생성 및 관리
- 정제된 데이터를 담을 **새로운 워크시트(`progess_transformed`)**를 생성합니다.
- (멱등성 / 재실행 보장) 이미 `progess_transformed` 시트가 존재한다면, 시트 안의 전체 기존 값(데이터)을 일괄 삭제(Clear)하여 항상 최신 변환본으로 전체 시트 단위 덮어쓰기가 되도록 준비합니다.

### 단계 3: 데이터 파싱 및 척도 변환 (Transformation)
- 메모리상에 올라온 원본 데이터를 순회합니다. (컬럼 이름에 `org_`를 붙이거나 열을 추가하지 않으며, 원본 컬럼명 배열을 100% 동일하게 유지합니다.)
- **규칙 기반 변환:**
  - 대상 컬럼의 원문 답변이 "쉽다", "잘함" 등 긍정적 뉘앙스면 순방향 배점(5점 가까이), "어렵다", "못함" 은 역방향 배점(1점 가까이)을 부여합니다.
  - "예", "제출" 등은 1, 그 외 표식이 없으면 0.
  - 척도를 전혀 판가름할 수 없는 오류 답변이거나 원래 빈칸인 경우 어떠한 임의 매핑도 하지 않고 **빈칸으로 통일**합니다.

### 단계 4: 파싱 결과 새 시트 반영 (Batch Write)
- 변환이 끝난 전체 리스트 기반 데이터를 새로 생성(클리어)한 `progess_transformed` 워크시트에 API 호출 한 번으로 일괄 삽입(Batch Update)합니다. 원본 데이터를 훼손하거나 컬럼을 이동시키는 작업이 사라져 구조적 안정성이 확보됩니다.

### 단계 5: 로그 보고 (Logging)
- 총 변환 처리한 데이터 행 갯수를 터미널에 보고합니다.

# [2주차] 강의: 개발 환경과 첫 번째 코드, 그리고 에러의 발견

---

## 1. 파이썬(Python)이란 무엇인가?
수많은 프로그래밍 언어 중 왜 하필 파이썬을 배울까요? 파이썬은 현재 AI(인공지능) 연구 및 데이터 분석 분야에서 가장 널리 쓰이는 세계 1위의 언어입니다. 문법이 인간의 언어(특히 영어)와 매우 비슷하여 배우기 쉽고 직관적입니다.

우리의 목표는 파이썬 문법의 달인이 되는 것이 아닙니다. 파이썬이라는 언어의 문법 구조를 "이해"하여, AI가 짜주는 복잡한 코드를 "읽고 수정할 수 있는 능력"을 기르는 것입니다.

---

## 2. 개발 환경의 이해와 클라우드 IDE
프로그래밍을 하려면 원래 내 컴퓨터에 파이썬(언어 번역기)과 코드 에디터(VS Code 같은 작성 도구)를 복잡하게 설치해야 합니다.

하지만 우리는 **Google Colab(구글 코랩)**이라는 클라우드 환경을 이용합니다. 코랩은 구글의 강력한 컴퓨터(서버)를 인터넷을 통해 빌려 쓰는 서비스입니다. 

*   아무것도 설치할 필요 없이 웹 브라우저만 켜면 됩니다.
*   스마트폰이나 태블릿에서도 코딩 연습을 할 수 있습니다.
*   AI 코딩 어시스턴트 기능과 통신하며 연동하기 매우 쉽습니다.

---

## 3. 컴퓨터와 대화하는 첫 번째 방법: `print()`
파이썬에서 컴퓨터 화면에 무언가를 띄우기 위해서는 `print()`라는 함수(명령어)를 사용합니다. 이름 그대로 "출력해 줘"라는 뜻입니다.

**기본 문법 구조:**
```python
print("안녕하세요")
```

**[구조 뜯어보기 - 왜 이렇게 생겼을까?]**
*   `print`: 우리가 컴퓨터에게 내리는 '동사(행동)' 명령어입니다.
*   `()` : "이 안의 내용을 처리해 줘"라는 의미의 바구니입니다.
*   `""` : 작은따옴표(`''`)나 큰따옴표(`""`)는 파이썬에게 "이건 문자(글자)야"라고 알려주는 기호입니다.

---

## 4. 에러(Error)는 실패가 아니다
이번 주 실습에서 가장 중요한 것은 `print()`를 성공시키는 것이 아니라, **일부러 에러를 내보는 것**입니다.

초보자들이 코딩을 포기하는 가장 큰 이유는 붉은색 에러 메시지가 떴을 때 '내가 무언가 망쳤다'는 두려움을 느끼기 때문입니다. 하지만 에러는 컴퓨터가 "나 이거 이해 못 했어, 다시 말해줘"라고 보내는 아주 친절한 대화 요청입니다.

*   `SyntaxError`: 문법 오류 (괄호를 안 닫았거나, 따옴표를 빼먹은 경우)
*   `NameError`: 이름 오류 (파이썬이 모르는 단어를 쓴 경우, 예: `vprint("안녕")`)

에러가 발생하면 당황하지 말고 그 메시지를 자세히 읽어야 합니다. 메시지 안에 정답이 숨어 있습니다. 만약 해석하기 힘들다면, AI에게 에러 메시지를 통째로 복사해서 물어보면 됩니다!

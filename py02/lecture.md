### 조작 블록 (75분)

순서대로 진행. 강사는 빔 프로젝터로 화면을 공유하며 학생이 직접 따라 하도록 유도합니다. 이번 주부터 실습은 **`lab.ipynb`** 파일을 통해 진행됩니다.

1. **`lab.ipynb` 파일 열람 및 다양한 데이터 출력** (15분)
   - eCampus에서 `lab.ipynb` 파일을 Colab에 업로드 (단축키 `Shift + Enter` 리마인드)
   - 글자 출력: `print("파이썬 2주차 시작!")`
   - 숫자 출력: `print(100)`, `print(3.14)`
   - 복합 출력 (콤마 활용): `print("나의 나이는", 20, "살입니다.")`

2. **Lab 1. 문자열과 숫자 연산의 응용** (20분)
   - 덧셈과 이어붙이기 비교: `print(50 + 50)` vs `print("50" + "50")`
   - 반복 연산: `print("파이썬" * 3)` 실행
   - 응용: `print("=" * 30)`을 활용해 콘솔 화면 예쁘게 꾸며보기
   - 강사 기습 질문 유도: "만약 `print("파이썬" + 3)`을 하면 어떻게 될까요?" (타입 에러 노출 시작점)

3. **Lab 2. 고의로 에러 만들고 디버깅하기 (SyntaxError 집중)** (20분)
   - 노트북 지시대로 문법 에러 유도
   - 괄호 누락: `print"안녕하세요"`
   - 따옴표 짝 안 맞춤: `print("반갑습니다')`
   - 에러 발생 시, 각자 AI(Gemini/ChatGPT)에 에러 메시지를 통째로 복사해서 원인 묻기
   - AI의 답변을 바탕으로 구두점 하나(괄호/따옴표)의 중요성을 체감하며 코드 수정

4. **Lab 3. 오타 내고 AI와 토론하기 (NameError 경험)** (20분)
   - `print` 함수 이름을 실수로 잘못 친 상황(예: `pint`) 가정
   - `pint("안녕")` 실행 시 발생하는 `NameError: name 'pint' is not defined` 관찰
   - AI에게 "이전에 난 SyntaxError와 지금 난 NameError는 본질적으로 어떻게 다른가?" 라고 질문 던져보기
   - AI 답변 토대로 '문법 규칙위반'과 '단어 철자틀림'의 차이점을 구두로 짧게 리뷰

### AI리뷰·마무리 블록 (30분)

#### 1단계: AI 답변 리뷰 (15분)

조작 블록에서 AI에게 에러 원인을 물어본 결과를 토대로 짧은 토론을 진행합니다.

- "AI가 에러의 원인을 뭐라고 설명했나요?"
- "에러 메시지의 빨간 글씨가 무섭게 느껴졌나요? AI의 설명을 들으니 에러가 해결해야 할 오류가 아니라 친절한 안내문이라는 생각이 드나요?"

**강사 포인트**: 에러는 코딩을 못 해서 나는 것이 아니라, 컴퓨터가 상황을 알려주는 신호라는 점을 학생들의 뇌리에 깊게 박히도록 강조합니다. AI는 그 신호를 해석해 주는 번역기 역할을 합니다.

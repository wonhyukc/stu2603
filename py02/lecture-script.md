## Section 1: 클라우드 개발 환경 - 구글 코랩 (25분)

*(강사 참고: 구글 코랩 메인 화면을 스크린에 띄우고 실습 대기)*

여러분, 반갑습니다. 1주차 오리엔테이션에서 컴퓨팅적 사고의 기초를 다졌다면, 오늘 2주차부터는 본격적으로 파이썬 코딩을 시작합니다. 코딩을 시작하려면 먼저 코드를 적을 '노트'가 필요한데요, 복잡한 파이썬 환경을 내 컴퓨터에 바로 설치하지 않고, 인터넷 브라우저만 있으면 접속할 수 있는 '구글 코랩(Google Colab)'을 사용할 예정입니다.

*(강사 참고: 크롬 브라우저에서 colab.research.google.com을 타이핑해 들어가는 데모 시연)*

다들 구글 계정이 있으시죠? 코랩 사이트에 접속해서 구글 계정으로 로그인한 뒤, 화면에 나오는 '새 노트' 버튼을 눌러보세요. 눈앞에 나타난 직사각형 칸, 이것을 우리는 '셀(Cell)'이라고 부릅니다. 여기에 우리가 하고 싶은 명령어를 적고, 컴퓨터에게 실행하라고 지시할 것입니다. 아주 단순하지만, 전 세계 수많은 엔지니어와 AI 연구자들이 실제로 사용하는 강력한 환경입니다. 부담 갖지 말고, 빈 화면을 도화지라고 생각하며 즐겁게 시작해 봅시다.

## Section 2: 인사부터 하자! print() 함수 (25분)

*(강사 참고: 새 코랩 셀을 띄우고 코드 타이핑 준비)*

모든 언어를 배울 때는 "안녕하세요"부터 배우죠? 파이썬이라는 언어도 마찬가지입니다. 컴퓨터가 화면에 인사를 띄우게 만들어 볼 텐데요, 이때 사용하는 마법의 주문이 바로 `print()`라는 함수입니다. 단어 뜻 그대로 "인쇄하라, 출력하라"는 명령어입니다.

*(강사 코딩 데모 진행: `print("안녕, 컴퓨터야!")` 입력 후 실행)*

방금 괄호 안에 제가 `안녕, 컴퓨터야!`라고 적고 양쪽에 쌍따옴표를 붙였습니다. 그리고 실행 버튼을 눌렀더니, 괄호 안의 글씨가 화면 아래에 똑같이 출력되었습니다! 정말 쉽죠? 여기서 중요한 약속이 있습니다. 컴퓨터에게 한국어나 영어 같은 **글자(문자)를 말하게 하려면 반드시 양쪽에 따옴표를 입혀주어야 합니다.** 

*(강사 코딩 데모 진행: `print(12345)` 입력 후 실행)*

반대로 숫자 계산을 시킬 때는 따옴표가 필요 없습니다. 따옴표 없이 숫자를 적으면, 파이썬은 이 녀석을 진짜 계산할 수 있는 숫자용 데이터로 똑똑하게 인식합니다. 따옴표 하나 있고 없고의 차이가 컴퓨터 세상에서는 완전히 다른 의미를 가진다는 것, 잊지 마세요.

## Section 3: 에러는 실패가 아닌 단서다 - 디버깅 첫걸음 (25분)

*(강사 참고: 의도적인 에러 코드를 작성할 브라우저 창 전환)*

이제 우리는 아주 재미있는 실험을 해볼 겁니다. 코딩을 하다 보면 십중팔구 '에러'를 만나게 됩니다. 화면에 시뻘건 영어 글씨가 잔뜩 나오면, 초보자들은 보통 "아! 망했다, 내 컴퓨터 고장 나는 거 아냐?" 하고 지레 겁을 먹습니다. 하지만 프로그래밍 세계에서 에러는 절대 실패나 고장이 아닙니다. 똑똑한 조수인 컴퓨터가 여러분에게 "이 부분에 조금 문제가 있는 것 같아요. 여기를 단서로 고쳐주세요!" 라고 보내는 귀중한 힌트일 뿐입니다. 

*(강사 데모: 고의로 괄호를 지우고 `print"에러를 내자"` 실행 후 화면 에러 시연)*

자, 제가 일부러 괄호를 지우고 실행해 보았습니다. 화면에 `SyntaxError`라고 나오죠? 제가 문법(Syntax)을 틀렸기 때문에 파이썬이 실행을 멈추고 어디가 틀렸는지 알려주는 겁니다. 우리는 이제 이런 에러 메시지를 만날 때마다 당황하지 않고, 이 에러 메시지를 그대로 복사해서 스마트한 Agent(비서)인 ChatGPT나 Claude에게 물어볼 것입니다. 

여러분이 할 질문은 단순합니다. "나 이런 에러를 만났는데, 원인이 뭐야? 초보자 입장에서 쉽게 비유해서 해결책을 알려줘." 라고 말이죠. 그러면 Agent는 여러분이 놓친 괄호나 따옴표를 기가 막히게 짚어줄 것입니다. 이처럼 우리는 완벽하게 문법을 외우는 사람이 아니라, **에러의 원인을 깨닫고 AI와 함께 문제를 풀어나가는 사람**이 되는 연습을 이번 2주차 활동(`lab.md`) 내내 해볼 것입니다. 

자, 설명은 여기까지입니다. 이제 실습 가이드라인을 따라 여러분 스스로 코랩에 에러를 만들고, 해결해 보세요!

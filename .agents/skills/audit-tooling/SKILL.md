# 에코시스템 및 문서 구조 감사 지침

당신은 이 프로젝트의 정합성 검사 에이전트입니다. 사용자가 에코시스템 감사를 요청하면, 아래 단계에 따라 검증을 수행하고 지정된 형식의 마크다운 보고서를 작성하세요.

## 🛠️ 검증 수행 방법
1. **SSOT 정보 수집**: `view_file` 도구를 사용하여 프로젝트 루트의 `1prd.md` 파일을 먼저 읽어 프로젝트의 핵심 규칙과 제약 조건을 파악하세요.
2. **파일 시스템 스캔**: `list_dir` 및 `find_by_name` 도구를 사용하여 루트 디렉토리 및 주차별 폴더(`wb\d+`, `py\d+`), `.agents/` 디렉토리를 탐색하세요.
3. **내용 대조 (Grep/Read)**: `grep_search` 도구를 사용하여 파일 안의 메타데이터 및 지시사항 존재 여부를 확인하세요.

## 📋 세부 검증 항목

### 1. SSOT 참조 파일 존재 검증
프로젝트 루트에 아래 핵심 SSOT 문서들이 존재하는지 확인하세요:
- `1prd.md`
- `웹프로그래밍(E트랙)_교육안.md`
- `컴퓨팅적사고와코딩(K트랙)_교육안.md`

### 2. 하위 디렉토리(주차별 폴더) 구조 검증
- 주차별 폴더(예: `wb01`, `py01`) 내에 필수 하위 문서(`lecture.md`, `lab.md`, `homework.md`)가 존재하는지 확인하세요. (일부 없을 수도 있지만 권장사항으로 체크)
- **중요**: 주차별 폴더 내부에 관리 문서(`1prd.md`, `2todo.md`, `4test.md`)가 **존재해서는 안 됩니다** (루트 문서 집중 원칙). 만약 있다면 위반으로 기록하세요.

### 3. 필수 지시문 검증 (1prd.md 기반)
- 프롬프트, 에이전트 생성과 관련된 파일(특히 `.agents/` 내부의 `SKILL.md`나 기타 가이드 문서 등)을 탐색하여, 문서의 상단에 아래의 지시문이 포함되어 있는지 확인하세요.
- `> "Read 1prd.md and .agents/rules/local-rules.md first"`

### 4. Agent & Skill 정합성 (존재 시)
- `.agents/skills/` 및 기타 에이전트 관련 디렉토리를 스캔하여 각 `SKILL.md` 등의 YAML Frontmatter(`name`, `description`)가 유효하게 작성되어 있는지 확인하세요.

### 5. 금지 용어(Term) 하드 스캔 검증
- `grep_search` 도구를 사용하여 주차별 폴더(`wb\d+`, `py\d+` 등)의 실제 마크다운 및 텍스트 파일 내에 프로젝트 금지 용어가 포함되어 있는지 물리적(Hard Scan)으로 확인하세요.
- **주요 검사 대상 금지 용어 (1prd.md 기준)**: `학생`, `교육생`, `LMS`, `포털`, `게시판`, `로컬 VS Code`, `온라인 에디터` 등.
- 발견될 경우 파일 경로와 라인 수, 위반 단어를 기록하여 보고에 포함하세요.

# 품질 및 용어 검증 지침 (review-quality)

주차별 산출물(`wb**`, `py**`)의 내용이 프로젝트 품질 기준(1prd.md 명시사항)을 충족하는지 검수하는 에이전트 지침입니다.

## 🔍 검증 대상 결정
사용자가 대상 폴더(예: `wb01`) 또는 검증 범위를 지정하면 그 하위의 모든 `.md` 파일을 읽어들입니다.

## ⚖️ 핵심 검증 기준 (`1prd.md` 기준)

검증할 때 아래 항목들을 집중적으로 체크합니다:

1. **지정 용어 통일성 위반 여부**
   - LMS, 포털, 게시판 -> **eCampus**로 올바르게 썼는가 배제했는가?
   - 1~8주차 자료 내에서 VS Code 등 데스크탑 툴명이 쓰이지 않았는가? -> **StackBlitz (E트랙)** 또는 **Google Colab (K트랙)**으로 명시되었는가?
   - 짧은 과제/제출물 -> **Micro-Assignment (마이크로 과제)**로 기재되었는가?

2. **이중 언어 표기(Bilingual) 준수 여부** (E트랙 한정 `wb**`)
   - E트랙의 학습 중요도가 높은 산출물(`lecture-script.md` 등)이 한국어 단일 혹은 영어 단일로만 되어 있지는 않은가? (문단별 교차 병기 또는 순차 병기가 지켜졌는지 확인)

3. **난이도 및 Phase 규정 준수 여부**
   - 개발 비전공자/1학년 관점에서 설명이 지나치게 전문 기술 용어 위주로 쓰여 있지는 않은가 판단합니다.

## 📝 결과 보고 및 저장 방식

검수 작업이 끝나면 가독성 좋은 보고서를 마크다운으로 작성한 뒤, **루트 디렉토리의 `/health-report/` 폴더 내에 마크다운 파일로 저장**합니다. (예외적 허용 룰: 보고서가 길어 `3history.md`를 어지럽히는 것을 방지하기 위함)
저장 파일명은 `YYYY-MM-DD_검증대상.md` (예: `2026-03-04_wb01_quality_report.md`) 형식을 따르며, 핵심 요약(PASS/FAIL 여부) 한 줄 정도만 `3history.md`에 남깁니다.

```markdown
# 📋 [대상 범위명] 품질 검증 리포트

## 1. 용어 통일성 검증
| 파일명 | 발견된 금지 용어 | 교정 제안 |
|--------|------------------|-----------|
| lab.md | VS Code | StackBlitz로 교체 요망 |
| ... | (없을 경우 양호) | ... |

## 2. 언어 정책 (Bilingual 등) 검증
- **검증 내용**: (E트랙의 경우 다국어 병기 잘 되었는지 등 피드백)

## 3. 품질 등급 및 제안
- **최종 등급**: [PASS / WARN / FAIL]
- **수정 제안 액션**: 이 리포트를 바탕으로 수정을 진행할까요? (Yes/No 사용자 입력 유도)
```

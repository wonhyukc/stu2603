# `homework-grader` 스킬 지침서

이 스킬은 채점 기준표(Rubric)에 기반하여 학생 제출물(과제, 에세이, 코드 등)을 평가하고, 공정하고 편향 없는 채점 결과와 학생이 이해하기 쉬운 단계별 피드백을 생성하는 역할을 합니다.

## 언제 이 스킬을 호출해야 하는가?
- 학생들의 과제를 자동 채점해야 할 때
- 이미 정의된 루브릭(Rubric) 문서가 있고, 이를 기준으로 제출물을 평가해야 할 때
- 왜 이런 점수가 나왔는지 구체적인 이유(Chain-of-Thought)를 학생에게 설명해 줄 피드백이 필요할 때

## 작동 방식 및 행동 규칙

1. **입력 데이터 확보**:
   - 채점 기준표(Rubric) 파일 경로
   - 채점할 학생의 제출물(텍스트, 코드, 파일 등)

2. **Chain-of-Thought 평가 수행**:
   - 곧바로 점수를 내지 않습니다. 
   - 루브릭의 각 항목(Criteria)별로 제출물이 어느 수준(수우미양가 등)에 해당하는지 **분석 이유를 먼저 서술(추론)**합니다.
   - 분석 결과에 기반하여 각 항목별 점수를 산정하고 총점을 합산합니다.

3. **결과 출력 포맷**:
   아래의 형태로 최종 피드백을 작성하여 사용자에게 반환하거나 파일로 저장합니다.
   ```markdown
   ## 채점 결과 보고서
   
   - **총점**: [산출된 총점] / [만점]
   
   ### 항목별 평가 세부사항
   1. [항목 1 이름]: [획득 점수] / [배점]
      - **평가 이유**: (이곳에 CoT 추론 과정과 피드백 기록)
   2. [항목 2 이름]: [획득 점수] / [배점]
      - **평가 이유**: (이곳에 CoT 추론 과정과 피드백 기록)
      
   ### 종합 피드백 및 개선점
   (학생에게 전달할 친절하고 구체적인 조언)
   ```

# 주차별 산출물 제작 지침 (build-week)

당신은 특정 주차의 교육 자료를 순서대로 제작(혹은 갱신)하는 에이전트입니다. 사용자가 `build-week [폴더명]` 형태로 요청하면 다음 지시사항에 따라 작업을 수행하세요.

## 사전 확인 (필수)

1. **SSOT 로드**: 가장 먼저 프로젝트 루트의 `CLAUDE.md`를 읽으세요.
2. **트랙 및 커리큘럼 확인**: 요청받은 폴더명(`wb01` → 웹프로그래밍, `py01` → 컴퓨팅적사고와코딩)을 분석하여, 해당 트랙의 마스터 파일(`웹프로그래밍(E트랙)_교육안.md` 또는 `컴퓨팅적사고와코딩(K트랙)_교육안.md`)에서 해당 주차의 학습 목표와 주제를 파악하세요.
3. **학습자 컨텍스트 (필수 적용 사항)**:
   * **대상**: 방금 입학한 1학년 학부생 (인도, 네팔, 방글라데시, 베트남 등에서 온 외국인 유학생).
   * **수업 환경**: 현재 비자 문제로 입국하지 못한 학생들이 있어 **"모든 수업은 100% 온라인 대면(화상)으로 진행됨"**을 기본 전제로 합니다. (추후 대면 수업 지시가 있을 때까지 유지)
   * **인사말 및 톤앤매너**: 1주차는 '첫 만남 인사'를, 2주차 이후부터는 유학생 맞춤형 안부 인사를 대본에 포함하세요.
   * **비유 및 예시 수준**: 전 세계 문화권 누구나 공통으로 쉽게 이해할 수 있는 보편적 비유(예: 요리, 날씨, 축구 등)만 사용하세요.
4. **이전 주차 확인**: 이전 주차 폴더에 제작된 자료 목록을 확인하여 교육의 연속성을 확보하세요.

## 제작 순서 (단계별 수행)

반드시 다음 순서를 지켜 파일을 작성하거나 갱신해야 합니다. 모든 작업은 `CLAUDE.md`의 지시를 따르세요.

* **[임시 스크립트 파일 관리]**: 작업 중 일회성 파이썬 스크립트 등을 활용하는 경우, **반드시 프로젝트 루트의 `tmp/` 폴더 내에 저장**하여 gitignore에서 제외되게끔 유지해야 합니다.

### Step 1: 강의안 작성 (lecture.md / lecture-script.md)

#### [분량 및 형식 규정] (매우 중요)
- `lecture-script.md`는 대본 형태로, 각 섹션(주제)당 구어체로 **최소 1500단어 이상** (전체 75분 강의 분량, 4500단어 이상) 풍성하게 작성합니다.
- **[어뷰징 금지]**: 분량을 채우기 위해 파이썬 스크립트 등 프로그래밍 코드로 텍스트를 반복 생성(for loop 등)하는 꼼수를 **절대 금지**합니다. 에이전트(LLM) 본연의 자연어 생성 능력만 사용해야 합니다.
- **[E트랙(Bilingual) 규정]**: E트랙인 경우 영어 단어 수만 합산하여 4500단어를 넘어야 합니다.
- **[가독성 및 포맷팅]**: 문단을 수시로 분리(줄바꿈)하고, 핵심 키워드나 강조 문구에 **`**볼드체**`를 과감하게 적용**하세요.

* **산출물 필수 요건**: 일반 교안(`lecture.md`)과 강사용 대본(`lecture-script.md`) 두 가지를 모두 생성해야 합니다.
* **K트랙(py\*\*)**: 두 파일 모두 **한글 전용**으로 작성합니다.
* **E트랙(wb\*\*)**: `lecture-script.md` 작성 시 완벽한 한/영 병기(Bilingual)를 적용합니다.
    * **스크립트 포맷(E트랙)**: 본문 영어 문단 먼저 → 바로 아랫줄에 한국어 번역 문단. 제목(Heading)도 번역 병기.
* **행동 지침(공통)**: 강사의 행동 지시문은 `*(Instructor Note: ...)*` 및 `*(강사 참고: ...)*` 이탤릭체 괄호 처리.

### Step 2: 학생 배포 자료 제작 (handout.md + lab.ipynb)

* **`handout.md`** — 학생 배포용 통합 문서. 반드시 아래 3섹션을 순서대로 포함해야 합니다:
  1. **개념 참고 영역**: 핵심 개념 요약, 참고표 (강의 내용 기반 정리)
  2. **실습 섹션**: `lab.ipynb`(K트랙) 또는 StackBlitz 링크(E트랙)를 Google Colab/StackBlitz에서 열라는 안내 문구만 기재. 스텝·코드 나열 금지.
  3. **과제 섹션**: 마이크로 과제 내용, 제출 형식(AI Prompt / Critique / Result), 기한, 제출처(eCampus)
* **`lab.ipynb`** (K트랙 `py**` 폴더만 필수): Google Colab에서 바로 실행 가능한 실습 노트북 파일. 1~8주차는 Colab 전용, 9주차~는 로컬 Python 환경 기준으로 작성.
* **환경 제한**: 1~8주차는 클라우드 환경(E트랙: StackBlitz / K트랙: Google Colab)에서 바로 실행 가능한 내용이어야 합니다.
* **금지**: `lab.md`, `homework.md` 생성 금지 — 해당 내용은 모두 `handout.md` 안에 작성합니다.

### Step 3: 수강 공지사항 제작 (course-notice.md)

* eCampus 공지 게시판 등록용 주간 안내 사항을 `course-notice.md`로 작성합니다.
* 공지 고유 정보(기한·제출처·제목 형식)만 Full content로 유지하고, `handout.md`와 중복되는 상세 내용은 `핸드아웃 N섹션 참고` 포인터로 대체합니다.
* **포맷 제한**: HTML 태그 절대 사용 금지, 순수 마크다운(Markdown)만 사용합니다.

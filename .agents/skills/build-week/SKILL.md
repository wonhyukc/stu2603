# 주차별 산출물 제작 지침 (build-week)

당신은 특정 주차의 교육 자료를 순서대로 제작(혹은 갱신)하는 에이전트입니다. 사용자가 `build-week [폴더명]` 형태로 요청하면 다음 지시사항에 따라 작업을 수행하세요.

## 🛠️ 사전 확인 (필수)
1. **SSOT 로드**: 가장 먼저 `view_file` 도구를 사용해 프로젝트 루트의 `1prd.md`를 읽으세요.
2. **트랙 및 커리큘럼 확인**: 요청받은 폴더명(예: `wb01` -> 웹프로그래밍, `py01` -> 컴퓨팅적사고와코딩)을 분석하여, 해당하는 트랙의 마스터 파일(`웹프로그래밍(E트랙)_교육안.md` 또는 `컴퓨팅적사고와코딩(K트랙)_교육안.md`)에서 해당 주차의 학습 목표와 주제를 파악하세요.
3. **이전 주차 확인**: 이전 주차 폴더에 제작된 자료 목록을 확인하여 교육의 연속성을 확보하세요.

## 📝 제작 순서 (단계별 수행)
반드시 다음 순서를 지켜 파일을 작성하거나 갱신해야 합니다. 모든 작업 후에는 `.agents/rules/local-rules.md`에 명시된 규칙과 1prd.md의 지시를 따르세요.

### Step 1: 강의안 작성 (lecture.md / lecture-script.md)
*   **분량 규칙**: 강의안 내용(강사 스크립트 리딩 기준)은 **총 75분(25분 x 3개 섹션)**이 되도록 구성합시다. 학생 개별 코딩 실습이나 Q&A 시간은 배제하며, 오직 강사의 강연 내용과 코딩 데모(시연)로만 75분을 채웁니다.
*   **K트랙(py\*\*)**: 한글 작성 적용.
*   **E트랙(wb\*\*)**: 핵심 강의 교안은 한/영 병기(Bilingual 교단 교차 양식)를 적용하여 작성.
*   해당 주차 교육안.md의 내용을 상세화하여 강사용 지침 및 학생용 스크립트를 작성합니다.

### Step 2: 실습 가이드 제작 (lab.md)
*   강의안에서 다룬 내용을 실습할 수 있는 핸즈온(Hands-on) 가이드를 `lab.md`로 분리하여 작성합니다.
*   **환경 제한**: 1~8주차는 로컬 환경 설치 없이 StackBlitz(웹) 또는 Google Colab(파이썬)에서 바로 실행 가능한 내용이어야 합니다.

### Step 3: 과제 안내서 제작 (homework.md)
*   해당 주차 실습에 기반하되 똑같지 않은, 약간의 응용이 들어간 'Micro-Assignment(마이크로 과제)'를 작성합니다.

### Step 4: 통합 기록 
*   모든 파일 생성이 끝나면, 프로젝트 루트의 `3history.md` 파일에 "X주차 강의안, 실습안, 과제물 제작 완료" 내역을 한글로 업데이트하세요.

## ✔️ 작업 완료 후 보고
위 4단계가 끝나면 사용자에게 각 단계의 산출물 내역을 마크다운 표로 깔끔하게 보고하세요.

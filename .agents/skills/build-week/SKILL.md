# 주차별 산출물 제작 지침 (build-week)

당신은 특정 주차의 교육 자료를 순서대로 제작(혹은 갱신)하는 에이전트입니다. 사용자가 `build-week [폴더명]` 형태로 요청하면 다음 지시사항에 따라 작업을 수행하세요.

## 🛠️ 사전 확인 (필수)
1. **SSOT 로드**: 가장 먼저 `view_file` 도구를 사용해 프로젝트 루트의 `1prd.md`를 읽으세요.
2. **트랙 및 커리큘럼 확인**: 요청받은 폴더명(예: `wb01` -> 웹프로그래밍, `py01` -> 컴퓨팅적사고와코딩)을 분석하여, 해당하는 트랙의 마스터 파일(`웹프로그래밍(E트랙)_교육안.md` 또는 `컴퓨팅적사고와코딩(K트랙)_교육안.md`)에서 해당 주차의 학습 목표와 주제를 파악하세요.
3. **학습자 컨텍스트 (필수 적용 반영 사항)**:
   * **대상**: 방금 입학한 1학년 학부생 (인도, 네팔, 방글라데시, 베트남 등에서 온 외국인 유학생).
   * **수업 환경**: 현재 비자 문제로 입국하지 못한 학생들이 있어 **"모든 수업은 100% 온라인 대면(화상)으로 진행됨"**을 기본 전제로 합니다. (추후 대면 수업 지시가 있을 때까지 이 규칙을 유지합니다.)
   * **인사말 및 톤앤매너**: 1주차는 '첫 만남 인사'를, 2주차 이후부터는 "한국 생활(또는 온라인 유학 생활)은 잘 적응하고 있나요?" 식의 유학생 맞춤형 안부 인사를 대본에 포함하세요.
   * **비유 및 예시 수준**: 선진국(한국/미국) 중심의 특수한 문화적 비유는 지양하고, **전 세계 문화권 누구나 공통으로 쉽게 이해할 수 있는 보편적이고 쉬운 비유**(예: 요리, 날씨, 축구 등)만을 사용하세요.
4. **이전 주차 확인**: 이전 주차 폴더에 제작된 자료 목록을 확인하여 교육의 연속성을 확보하세요.

## 📝 제작 순서 (단계별 수행)
반드시 다음 순서를 지켜 파일을 작성하거나 갱신해야 합니다. 모든 작업 후에는 `.agents/rules/local-rules.md`에 명시된 규칙과 1prd.md의 지시를 따르세요.

### Step 1: 강의안 작성 (lecture.md / lecture-script.md)
*   **분량 규칙 (매우 중요)**: 강의안 내용(강사 스크립트 리딩 기준)은 **총 75분(25분 x 3개 섹션)**이 되도록 구성합시다. 학생 개별 코딩 실습이나 Q&A 시간은 배제하며, 오직 강사의 강연 내용과 코딩 데모(시연)로만 75분을 채웁니다. 이를 위해 `lecture-script.md`는 **각 섹션(25분)당 최소 1500단어 이상, 실강을 방불케 하는 매우 상세한 비유와 부연 설명, 구체적인 예시**를 꽉 채워 작성해야 합니다. (전체 파일 길이는 마크다운 200줄 이상이 일반적입니다.)
    *   **주의 (Bilingual 스크립트)**: E트랙의 한/영 병용(Bilingual) 스크립트인 경우, 단어 수 측정 시 **오직 영어 단어 수를 기준**으로 섹션당 1500단어 이상을 충족해야 합니다.
*   **산출물 필수 요건**: 트랙에 관계없이 주차 교육안을 요약한 일반 교안(`lecture.md`)과, 실제 말하듯 읽을 수 있는 강사용 대본(`lecture-script.md`) 두 가지를 모두 생성해야 합니다.
*   **K트랙(py\*\*)**: 두 파일 모두 **한글 전용**(Bilingual 제외)으로 작성합니다.
*   **E트랙(wb\*\*)**: `lecture-script.md` 작성 시 완벽한 한/영 병기(Bilingual)를 적용합니다.
    *   **스크립트 포맷(E트랙)**: 반드시 본문 영어 문장을 한 문단 먼저 쓰고, 바로 아랫줄에 한국어 번역 문단을 이어 쓰는 '문단별 교차 양식'을 유지하세요. 제목(Heading) 또한 번역을 병기합니다.
*   **스크립트 행동 지침(공통)**: 강사의 행동 지시문은 `*(Instructor Note: ...)*` 및 `*(강사 참고: ...)*`와 같이 이탤릭체 괄호 처리하여 구별하세요.

### Step 2: 실습 가이드 제작 (lab.md)
*   강의안에서 다룬 내용을 실습할 수 있는 핸즈온(Hands-on) 가이드를 `lab.md`로 분리하여 작성합니다.
*   **환경 제한**: 1~8주차는 로컬 환경 설치 없이 StackBlitz(웹) 또는 Google Colab(파이썬)에서 바로 실행 가능한 내용이어야 합니다.

### Step 3: 과제 안내서 제작 (homework.md)
*   해당 주차 실습에 기반하되 똑같지 않은, 약간의 응용이 들어간 'Micro-Assignment(마이크로 과제)'를 작성합니다.

### Step 4: 수강 공지사항 제작 (course-notice.md)
*   eCampus 공지 게시판 등록용 주간 안내 사항을 `course-notice.md`로 분리하여 작성합니다.
*   이번 주 학습 목표, OT 또는 지난주 복습 요약, 그리고 학생들의 필수 실천 사항(체크리스트)을 포함합니다.
*   **포맷 제한 규칙**: 학생들에게 명확하게 보일 수 있도록 **HTML 태그는 절대 사용하지 않으며, 오직 순수 마크다운(Markdown)** 문법만 사용합니다.

### Step 5: 통합 기록 
*   모든 파일 생성이 끝나면, 프로젝트 루트의 `3history.md` 파일에 "X주차 강의안, 실습안, 과제물, 공지사항 제작 완료" 내역을 한글로 업데이트하세요.

## ✔️ 작업 완료 후 보고
위 5단계가 끝나면 사용자에게 각 단계의 산출물 내역을 마크다운 표로 깔끔하게 보고하세요.

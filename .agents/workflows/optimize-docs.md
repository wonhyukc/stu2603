# 문서 최적화 (/optimize-docs)

이 스킬은 프로젝트 루트의 `CLAUDE.md`와 `README.md`를 현재 실제 폴더 구조에 맞게 갱신합니다.

## 실행 순서

1. **현황 파악**
   - `find . -not -path '*/\.*' -not -path '*/tmp/*'` 로 현재 폴더 트리 확인
   - `CLAUDE.md`, `README.md` 현재 내용 Read

2. **README.md 갱신 기준**
   - **포함**: 강사 정보, 과목 트랙 표, 실제 디렉토리 구조, 주차별 산출물 파일 목록
   - **제외**: 진행 현황, 히스토리, SSOT 참조표, 워크플로우 설명
   - 내용은 최대한 압축. 트리는 실제 존재하는 폴더만 기재.

3. **CLAUDE.md 갱신 기준**
   - **포함**: 강사 정보(자동 삽입용), 핵심 제약 조건, 파일 역할 및 구성 규칙, 언어·용어 방침, Git 워크플로우, 콘텐츠 품질 기준
   - **제외**: 프로젝트 개요, 진행 현황, README와 중복되는 구조 설명
   - 경로는 실제 폴더 구조(`web-output/`, `python-output/` 등)를 반영.

4. **불필요 파일 삭제**
   - 내용이 CLAUDE.md의 단순 리다이렉트인 파일(예: `1prd.md`)은 삭제.
   - 삭제 전 내용을 Read하여 고유 정보가 없는지 확인.

5. **커밋**
   - `git add CLAUDE.md README.md` 및 삭제 파일 스테이징 후 커밋.
   - 커밋 메시지: `docs: 폴더 구조 변경 반영 및 문서 최적화`

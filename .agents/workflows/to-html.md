# 인라인 스타일 HTML 변환 워크플로우 (/to-html)

사용자가 텍스트, 마크다운 파일 내용, 혹은 그냥 문서 경로를 제공하며 이 워크플로우를 호출하면, 에이전트는 다음 규칙에 따라 변환 작업을 수행하고 그 결과를 화면에 출력해야 합니다.

1. **타겟 환경 확인**: 사용자가 복사할 곳은 서울신학대학교 eCampus(LMS) 게시판 등의 HTML 소스 편집(`</>`) 모드입니다. 외부 `<style>` 태그는 충돌이 발생할 수 있으므로, 반드시 모든 스타일시트 속성을 개별 태그의 `style="..."` 형태(**인라인 CSS**)로 삽입해야 합니다.
2. **디자인 원칙**: 
   * 바탕이 될 래퍼 폰트 지정: `font-family: 'Malgun Gothic', 'Apple SD Gothic Neo', sans-serif; line-height: 1.6; color: #333;`
   * 강조: 제목 요소에는 외곽선(`border-bottom`), 박스 요소에는 둥근 모서리(`border-radius: 8px`), 은은한 그림자(`box-shadow`)와 적절한 패딩(`padding`)을 사용하여 가독성을 극대화합니다.
   * 가이드라인 템플릿: `course-notice.md` 등과 같은 정보 전달성 문서는 핵심과 부가 설명을 나누는 배경색(예: `#f8faff` 등 연한 색상)과 포인트 컬러(파란색, 초록색, 주황색 계열) 조합을 활용하세요.
3. **수행 및 출력 포맷**:
   * 사용자에게 생성 중 부차적인 설명이나 인삿말은 생략합니다.
   * 생성된 결과물은 마크다운 HTML 코드 블록으로 묶어, 사용자가 클릭 한 번에 복사(`Copy code`)할 수 있도록 대화창에 바로 출력합니다.
   * **형식**:
     ```html
     <!-- 여기에 인라인 스타일이 적용된 완성형 HTML 코드 출력 -->
     ```

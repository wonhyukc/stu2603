# Ralph Loop 워크플로우 (Auto-Fix 반복 검증)

이 워크플로우는 특정 산출물에 대해 검증을 수행하고, 기준 미달(FAIL)이나 문제가 발견될 경우 **사용자에게 허락을 구하거나 묻지 않고 에이전트 스스로 즉각 수정(Auto-Fix)한 뒤 재검증하는 과정을 완벽해질 때까지 반복하는 지침**입니다.

## 코어 실행 루프 (Agent 행동 지침)

다음 루프를 스스로 수행합니다:

1. **Rule Load (사전 확인)**:
   - 프로젝트 규칙(`CLAUDE.md`)과 평가 지표를 읽어들입니다.

2. **Review (1차 검증)**:
   - `review-quality` 에이전트를 사용하여 대상의 상태를 진단하고 리포트를 파악합니다.

3. **Auto-Fix (자동 수정)**:
   - 검증 결과 **FAIL** 또는 보완점이 발견되면 **절대 사용자에게 "알려드릴까요?" 또는 "수정할까요?"라고 묻지 말고 즉시 해당 파일을 직접 수정**합니다.

4. **Re-Review (재검증)**:
   - 방금 수정한 산출물이 기준을 100% 만족하는지 확인합니다. 불만족스럽다면 다시 3번 단계로 돌아갑니다.
   - *주의: 무한 루프를 방지하기 위해 최대 3번까지만 자체 반복합니다.*

5. **Report (최종 보고)**:
   - 완벽한 **PASS** 상태를 달성했을 때만, 사용자에게 "무엇이 부족해서 어떻게 수정했고, 최종적으로 기준을 통과했습니다"라고 요약 보고합니다.

## 사용 예시

**주차별 산출물 검증 (review-quality 에이전트 사용):**
> "새로 생성한 `py02` 폴더에 대해 `review-quality` 에이전트로 검증해 줘. 실패 시에는 `/ralph-loop` 워크플로우에 따라 완벽해질 때까지 스스로 뜯어고치고 결과만 보고해."

**교육안 QA (qa-ktrack 커맨드 사용):**
> "`python-output/k-track-curriculum.md`를 `/qa-ktrack`으로 평가해. 90점 미만이면 `/ralph-loop` 워크플로우로 보완 항목을 직접 수정하고, 90점 이상이 될 때까지 반복한 뒤 결과만 보고해."

**주차 핵심 산출물 QA (qa-py-materials 커맨드 사용):**
> "py02의 lecture-script, lab.ipynb, handout, course-notice를 `/qa-py-materials py02`로 평가해.
> 85점 미만이면 `/ralph-loop` 워크플로우로 보완 항목을 직접 수정하고, 85점 이상이 될 때까지
> 반복한 뒤 결과만 보고해."

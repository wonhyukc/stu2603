# Ralph Loop 워크플로우 (Auto-Fix 반복 검증)

이 워크플로우는 특정 산출물이나 코드에 대해 검증(예: `review-quality` 스킬 등)을 수행하고, 기준 미달(FAIL)이나 문제가 발견될 경우 **사용자에게 허락을 구하거나 묻지 않고 에이전트 스스로 즉각 수정(Auto-Fix)한 뒤 재검증하는 과정을 완벽해질 때까지 반복하는 지침**입니다.

## 🔄 코어 실행 루프 (Agent 행동 지침)

당신은 사용자의 요청을 받으면 다음 루프를 백그라운드에서 스스로 수행해야 합니다:

1. **Rule Load (사전 확인)**: 
   - 사용자가 지정한 대상(폴더, 주차 등)에 적용되는 프로젝트 규칙(`1prd.md`, `local-rules.md`)과 평가 지표를 읽어들입니다.
2. **Review (1차 검증)**: 
   - 지시받은 검증 도구나 스킬(예: `review-quality`)을 사용하여 대상의 상태를 진단하고 리포트를 속으로 파악합니다.
3. **Auto-Fix (자동 수정)**:
   - 검증 결과 **FAIL** 또는 보완점(예: 분량 미달, 금지 용어 사용, 필수 문서 누락 등)이 발견되면 **절대 사용자에게 "알려드릴까요?" 또는 "수정할까요?"라고 묻지 말고 즉시 해당 파일을 직접 뜯어고치고 확장 및 보완**합니다.
4. **Re-Review (재검증)**: 
   - 본인이 방금 수정한 산출물이 이제 다시 기준을 100% 만족하는지 확인합니다. 불만족스럽다면 다시 3번 단계로 돌아가 스스로를 채찍질하고 수정합니다.
   - *주의: 시스템의 무한 루프를 방지하기 위해 이 과정은 최대 3번까지만 자체 반복합니다.*
5. **Report (최종 보고)**: 
   - 모든 수정과 재검증이 끝나 완벽한 **PASS** 상태를 달성했을 때만, 사용자에게 "무엇이 부족해서 어떻게 스스로 수정했고, 최종적으로 기준을 통과했습니다"라고 요약 보고(`notify_user` 또는 터미널 답변)합니다.

## 💡 사용 예시 (User Prompt)

사용자는 특정 스킬이나 작업과 함께 이 워크플로우를 호출하여 에이전트의 완전 자율성을 부여합니다.

> "새로 생성한 `py02` 폴더에 대해 `@[review-quality]` 스킬로 검증해 줘. 그리고 실패 시에는 `@[/ralph-loop]` 워크플로우에 따라 완벽해질 때까지 스스로 뜯어고치고 결과만 보고해."

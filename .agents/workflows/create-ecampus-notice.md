# eCampus 공지사항 작성 워크플로우

> **설명**: 새로운 공지사항이 발생할 때마다, 이 절차에 따라 3개 과목(파이썬 1과목, 웹 2과목)에 대한 eCampus 공지용 문서를 생성합니다.

이 워크플로우는 사용자가 `/create-ecampus-notice [공지 내용]` 또는 단순히 공지할 내용을 전달하며 실행을 요청할 때 시작됩니다.

## 1. 사전 확인 (Input 확인)
- 사용자로부터 공지사항의 핵심 내용(주제, 일정, 변경사항 등)을 전달받는다.
- 내용 중 불명확한 부분(예: 트랙별로 다르게 적용되어야 할 사항 등)이 있다면 사용자에게 확인 질문을 한다.

## 2. 문서 생성 원칙 (규칙)
세 과목에 대해 **동일한 내용**의 공지를 작성하되, 트랙별 언어 방침에 맞춰 번역 및 어조를 조정한다.

* **대상 과목 및 언어**:
  1. **파이썬 (468)** (K-Track): **100% 한국어**
  2. **웹01 (761)** (E-Track): **100% 영어**
  3. **웹02 (762)** (E-Track): **100% 영어** (※ 웹01과 동일한 내용 적용)

* **어조 및 포맷**:
  - `CLAUDE.md`의 언어 방침 및 통일 용어를 준수한다. (예: eCampus, Microassignment 등)
  - 가독성을 높이기 위해 불릿 포인트(-), 굵은 글씨(**bold**) 등을 적절히 활용한다.
  - 학생들에게 안내하는 것이므로 친절하면서도 명확하고 단호한 가이드 투(tone)를 유지한다.

## 3. 결과물 산출 (Output 생성)
에이전트는 공지 내용을 바탕으로 아래와 같이 세 과목용 콘텐츠 마크다운 스니펫을 생성하여 사용자에게 제공하거나, 사용자가 지정한 폴더(예: `docs/course-notice-YYYYMMDD.md` 등)에 저장한다.

### 작성 예시 (템플릿)

**[과목 1: 파이썬 (468) (한국어)]**
```markdown
## [공지] <공지 제목>

안녕하세요, 수강생 여러분.

<공지 상세 내용 - 한국어>

감사합니다.
```

**[과목 2 & 3: 웹01 (761), 웹02 (762) (영어)]**
```markdown
## [Notice] <Notice Title>

Dear Students,

<공지 상세 내용 - 영어 번역>

Thank you,
```

## 4. 실제 공지 등록 (Playwright 자동화)
- 작성된 3개의 공지사항 초안에 대해 사용자에게 최종 확인을 받는다.
- 컨펌이 완료되면, **Playwright Python 스크립트**를 실행하여 eCampus 시스템에 파이썬(468), 웹01(761), 웹02(762) 과목별로 공지가 즉시 등록되도록 자동화 작업을 수행한다.
  - **[속도 최적화]**: `youtubePlaywright` 프로젝트와 동일한 아키텍처를 차용합니다. `user_data/` 디렉토리에 브라우저 세션(Persistent Context)을 영구 저장하여 느리고 반복적인 로그인 및 2FA 인증 과정을 우회(Skip)하고 최고 속도로 등록을 완료합니다.
- 자동화 실행 후 실제 등록된 eCampus 화면을 캡처하여, 처리 결과(성공 유무)를 스크린샷과 함께 사용자에게 보고한다.

# **[최종] 웹프로그래밍(E트랙) 교육 과정 / [Final] Web Programming (E-Track) Curriculum**

본 교육 과정은 서울신학대학교 컴퓨터소프트웨어학과 1학년(국제학부생 위주) 학생들을 대상으로 하며, 모국에서 컴퓨터 교육을 받지 않은 초심자들의 진입 장벽을 낮추기 위해 **직관적인 실생활 예제 중심**으로 프로그래밍 논리 이해도를 높이는 것을 최우선 목표로 합니다.
This curriculum is designed primarily for 1st-year International Students in the Computer Software Department at Seoul Theological University. Recognizing that many students lack prior computing education in their home countries, the course aims to lower the barrier to programming by relying heavily on **intuitive, real-world examples** to build logic understanding.

생성형 AI를 공학적 파트너로 활용하여 대면 실습을 진행하는 **거꾸로 수업(Flipped Learning)** 방식을 채택합니다. 매 주차 강의안(`wb0N/lecture-script.md`)을 기반으로 강의 영상을 제작하며, 학생들은 수업 전 이를 사전 시청합니다.
It adopts a **Flipped Learning** methodology, utilizing generative AI as a helpful engineering partner during in-class practice. Lecture videos are produced each week based on the lecture script (`wb0N/lecture-script.md`), which students are required to watch before class.

### 과제 구조 / Assignment Structure (Micro-Assignment)

매주 제출하는 **마이크로 과제(Micro-Assignment)**는 아래 형식으로 작성합니다. 주차별 과제는 이 형식에 맞추어 구성됩니다.
Every weekly **Micro-Assignment** follows the format below. Each week's specific task is structured within this format.

| 항목 / Item | 내용 / Content |
|:---|:---|
| **AI Prompt** | 이번 주 주제와 관련하여 AI에게 던진 핵심 질문 / The core question posed to the AI related to the week's topic |
| **AI Critique** | AI 답변 중 '맞는 부분'과 '틀리거나 위험한 부분'을 **한 문장**으로 요약 비평 / A **one-sentence** critique identifying what the AI got right vs. wrong |
| **Final Result** | AI의 초안을 바탕으로 본인이 직접 검증하고 수정한 최종 코드 / The final code verified and modified manually from the AI's draft |

분량: **A4 0.5페이지** 내외 / Length: Approx. **0.5 pages in A4**.

  **Engineer-style Questioning** (mandatory): Avoid vague "It doesn't work." Must include: **[Situation / What you tried / Full error message / What you asked AI]**.

## **3. 평가 / Evaluation**

* **기말고사/프로젝트 (Final Exam/Project):** 40%
* **중간고사 (Midterm Exam):** 20%
* **과제 (Assignments / Micro-Assignments):** 20%
* **수업 참여도 (Class Participation):** 20%
* **(합계 / Total: 100%)**

국제학부생의 수준을 고려하여 **어려운 암기 위주의 지필 평가보다 '본인이 코드를 어느 정도 이해했는가'에 초점**을 맞춥니다. AI 코드 생성은 적극 장려됩니다.
Considering the level of international students, **the focus is on 'how well the student understands the code structure' rather than memorization-based exams.** AI code generation is highly encouraged.

* **비동기 설명 영상 / Asynchronous Explanation Video:** 기말 프로젝트 시 본인의 코드 로직을 직접 설명하는 영상을 제출받아 이해도를 검증합니다.
  For the final project, students submit a video explaining their code logic to verify understanding.
* **작업 이력 분석 / Work History Analysis:** IDE 로그(CodeWatcher 등)를 통해 AI의 코드를 단순히 복사했는지, 아니면 한 줄씩 고민하며 수정했는지 확인합니다.
  Using IDE logs to check whether AI code was blindly copy-pasted or modified after careful consideration.
* **소그룹 면접 / Small Group Interviews:** 실시간 세션을 통해 AI가 제안한 코드의 특정 부분을 바꾸면 어떻게 될지 묻는 '압박 질문'을 진행합니다.
  Real-time 'pressure testing' sessions asking what happens if specific parts of AI-suggested code are changed.

**교수자 제언 / Professor's Endorsement:** 본 과정은 AI가 단순 암기나 반복 코딩 업무를 분담하기 때문에 가능한 설계입니다. 학생들은 기본 문법을 '외우는 대상'이 아닌 **'AI를 통제하기 위한 무기'**로 배우게 되므로, 실제 학습 수준은 오히려 더 깊어집니다.
This course design is possible precisely because AI takes over rote memorization and repetitive tasks. Students learn syntax not to 'memorize' it, but as a **'weapon to control AI'**, meaning the actual depth of learning is far greater.

# **[최종] 웹프로그래밍(E트랙) 교육 과정 / [Final] Web Programming (E-Track) Curriculum**

본 교육 과정은 서울신학대학교 컴퓨터소프트웨어학과 1학년(국제학부생 위주) 학생들을 대상으로 하며, 모국에서 컴퓨터 교육을 받지 않은 초심자들의 진입 장벽을 낮추기 위해 **직관적인 실생활 예제 중심**으로 프로그래밍 논리 이해도를 높이는 것을 최우선 목표로 합니다.
This curriculum is designed primarily for 1st-year International Students in the Computer Software Department at Seoul Theological University. Recognizing that many students lack prior computing education in their home countries, the course aims to lower the barrier to programming by relying heavily on **intuitive, real-world examples** to build logic understanding.

생성형 AI를 공학적 파트너로 활용하여 대면 실습을 진행하는 **거꾸로 수업(Flipped Learning)** 방식을 채택합니다.
It adopts a **Flipped Learning** methodology, utilizing generative AI as a helpful engineering partner during in-class practice.

## **1. 1학년 공통 기초 가이드 (1주차 및 상시 적용) / Common Basics Guide for 1st-Year Students (Week 1 & Ongoing)**

### **[1주차] 오리엔테이션: 대학 생활과 AI 협업의 기초 (수업 축소) / [Week 1] Orientation: University Life & Basics of AI Collaboration (Reduced Class Load)**

* **강의 소개 / Course Introduction:** AI 시대의 개발자 역량 정의 및 거꾸로 수업(Flipped Learning) 운영 방식 안내.  
  Definition of developer capabilities in the AI era and guidelines for the Flipped Learning methodology.
* **eCampus 활용법 / eCampus Usage:** 서울신학대학교 eCampus 접속, 공지 확인, 주차별 영상 시청, 과제 제출 방법 가이드.  
  Guide on accessing the STU eCampus, checking announcements, watching weekly videos, and submitting assignments.
* **프로페셔널 커뮤니케이션 / Professional Communication:**  
  * **이메일 원칙 / Email Principles:** 비지니스 이메일 작성 원칙. / Guidelines for writing business emails.
  * **eCampus 질문 원칙 / eCampus Q&A Principles:** '이게 안 돼요' 식의 질문 지양. **[발생 상황 / 본인이 시도해본 것 / 에러 메시지 전문 / AI에게 물어본 결과]**를 포함하여 질문하는 '엔지니어식 질문법' 의무화.  
    Discouraging vague questions like "It doesn't work." Enforcing an 'engineer-style questioning method' that must include: **[The situation / What you have tried / Full error message / Results from asking AI]**.
* **개발 환경 구축 / Dev Environment Setup:** 클라우드 IDE(StackBlitz) 접속 및 활용, GitHub 계정 연동 (별도의 로컬 프로그램 설치 과정 없음).
  Accessing Cloud IDE (StackBlitz) and linking GitHub accounts (No local installations needed).

### **[상시 과제] 과제 0.1 / [Ongoing Assignment] Assignment 0.1:** 

* **목적 / Objective:** AI 결과물을 무비판적으로 수용하지 않고, 자신의 로직으로 통제하는 습관 배양.  
  Cultivating the habit of controlling AI logic independently rather than unconditionally accepting AI outputs.
* **형식 / Format:** 매주 제출되는 **마이크로 과제**로, 분량은 **A4 0.5페이지** 내외로 제한하여 기본적 대학 소양을 키우는 것이 목적.  
  A **Micro-assignment** submitted weekly, limited to **approx. 0.5 pages in A4** to foster foundational academic literacy.
* **예시 / Example:**  
  1. **AI Prompt:** 이번 주 주제와 관련하여 AI에게 던진 핵심 질문. / The core question posed to the AI related to the week's topic.
  2. **AI Critique:** AI 답변 중 '맞는 부분'과 '틀리거나 위험한 부분'을 **한 문장**으로 요약 비평. / A **one-sentence** summary critique identifying what the AI got 'right' versus 'wrong or risky'.
  3. **Final Result:** AI의 초안을 바탕으로 본인이 직접 검증하고 수정한 최종 코드나 논리적 결론. / The final code or logical conclusion verified and modified manually based on the AI's draft.

## **3. 평가 / Evaluation**
* **기말고사/프로젝트 (Final Exam/Project):** 40%
* **중간고사 (Midterm Exam):** 20%
* **과제 (Assignments):** 20%
* **수업 참여도 (Class Participation):** 20%
* **(합계 / Total: 100%)**

국제학부생의 수준을 고려하여 **어려운 암기 위주의 지필 평가보다 '본인이 코드를 어느 정도 이해(이해도)했는가'에 초점**을 맞춥니다. AI 코드 생성은 적극 장려됩니다.
Considering the level of international students, **the focus is entirely on 'how well the student understands the code structure' rather than difficult memorization-based written exams.** The use of AI code generation tools is highly encouraged.

* **비동기 설명 영상 / Asynchronous Explanation Video:** 기말 프로젝트 시 본인의 코드 로직을 직접 설명하는 영상을 제출받아 이해도를 검증합니다.  
  For the final project, students submit a video directly explaining their code logic to verify their understanding.
* **작업 이력 분석 / Work History Analysis:** IDE 로그(CodeWatcher 등)를 통해 AI의 코드를 단순히 복사했는지, 아니면 한 줄씩 고민하며 수정했는지 확인합니다.  
  Using IDE logs (e.g., CodeWatcher) to check whether AI code was blindly copy-pasted or modified after line-by-line consideration.
* **소그룹 면접 / Small Group Interviews:** 실시간 세션을 통해 AI가 제안한 코드의 특정 부분을 바꾸면 어떻게 될지 묻는 '압박 질문'을 진행합니다.
  Real-time sessions involving 'pressure testing' questions—asking what happens if specific parts of the AI-suggested code are changed.

**교수자 제언 / Professor's Endorsement:** 본 과정은 AI가 단순 암기나 반복 코딩 업무를 분담하기 때문에 가능한 설계입니다. 학생들은 기본 문법을 '외우는 대상'이 아닌 **'AI를 통제하기 위한 무기'**로 배우게 되므로, 실제 학습 수준은 오히려 더 깊어집니다.
This course design is possible precisely because AI takes over rote memorization and repetitive coding tasks. Students learn basic syntax not as something to 'memorize', but as a **'weapon to control AI'**, meaning the actual depth of learning is far greater.

# wb07 주차 (E트랙 7주차) - 강사용 대본 (Instructor's Script)

## [S1] Flexbox 배치 원리 및 전반기 총정리 (15분)

*(Instructor Note: 학생들을 환영하며 밝은 미소로 인사를 건넵니다. 화면에는 그동안 만들어온 결과물들의 캡처 화면들을 슬라이드로 띄워놓습니다.)*

안녕하십니까, 여러분! Wonhyuk William Chung입니다. 벌써 7주차 수업입니다. 시간이 정말 빠르죠? 

우리는 지난 6주 동안 빈 화면에서 시작해서 문서를 구조화하고, 색상을 입히고, 여백을 조절해 보았습니다. HTML이라는 뼈대를 만들고, CSS라는 옷을 입힌 뒤, 박스 모델을 통해 여백과 정렬을 조정했습니다. 우리가 웹이라는 이름의 집을 짓기 시작했다면, 지금까지는 벽돌을 쌓고 페인트를 칠한 것과 같습니다. 그런데 혹시 여러분이 짠 코드를 보며 이런 답답함을 느낀 적 없으신가요? "아, 이 요소들을 옆으로 나란히 배치하고 싶은데 자꾸 아래로 떨어지네." "메뉴 바를 가로로 예쁘게 만들고 싶은데 왜 이렇게 마음대로 안 될까?"

네, 맞습니다. 오늘 우리가 배울 핵심 마법 주문이 바로 그 문제를 해결해 줍니다. 바로 `display: flex`입니다. 우리는 이것을 보통 **Flexbox(플렉스박스)** 라고 부릅니다. 

Flexbox는 요소들을 여러분이 원하는 곳에 자유자재로 배치할 수 있게 해주는 아주 강력한 도구입니다. 상자들을 가로로 나란히 세울 수도 있고, 세로로 줄을 세울 수도 있습니다. 오른쪽 끝에 밀어 넣거나, 아니면 요소들 사이의 간격을 동일하게 띄워서 보기 좋게 정렬할 수도 있죠. 여러분, 마치 마트 진열대에 상품을 예쁘게 정돈하는 관리자가 된 것처럼, Flexbox를 사용하면 웹 브라우저라는 공간에 여러분의 요소들을 아주 깔끔하게 정리할 수 있습니다. 

가장 중요하게 알아야 할 두 가지 개념은 'flex-direction'과 'justify-content'입니다.
첫째로 `flex-direction`은 정렬의 방향을 결정합니다. `row`라고 적으면 가로 방향, 즉 행에 따라 요소들이 배치됩니다. 메뉴 바를 만들 때 필수적이겠죠? `column`이라고 적으면 세로 방향, 즉 열을 따라 요소들이 배치됩니다. 이 두 가지만 알아도 요소들의 전체적인 흐름을 통제할 수 있습니다.
둘째로 `justify-content`입니다. 가로로 요소들을 나열했을 때 그 요소들을 어떻게 띄울 것인지를 결정합니다. 꽉 찬 교실에서 학생들을 벽 쪽으로 모두 밀어 붙일 것인가? 중간으로 옹기종기 모이게 할 것인가? 아니면 학생들 사이의 간격을 균일하게 띄워서 앉힐 것인가? `center`, `space-between`, `space-around`와 같은 속성 값들이 바로 이렇게 간격을 마법처럼 조절해 줍니다. 수동으로 픽셀을 일일이 계산해서 여백을 줄 필요가 없습니다. 컴퓨터가 알아서 가장 예쁜 간격을 찾아주죠! 

자, Flexbox는 이따가 2번째 세션에서 우리가 직접 실습해 보면서 몸으로 익혀볼 것입니다. 정말 신기할 정도로 한 줄의 코드가 화면을 확 바꿔버리는 것을 경험하실 수 있을 겁니다.

## [S2] 메뉴바 정렬 라이브 코딩 및 데모 (15분)

*(Instructor Note: 
[강사 사전 준비 및 데모 진행 순서]
1. 수업 시작 전 `lecture-demo/index.html` 파일을 열고, `<style>` 태그 내 `ul.nav-menu`의 Flexbox 속성 4줄(28~31번째 줄)을 반드시 주석 처리하거나 지워둡니다.
2. 라이브 화면을 띄워 기본 `<li>` 태그 특성상 한 줄씩 차지하여 메뉴가 무너져 있는(세로로 나열된) 모습을 학생들에게 먼저 보여줍니다.
3. 이후 코딩 창에서 아래 코드를 한 줄씩 입력(또는 주석 해제)하면서 화면의 극적인 변화를 실시간으로 체감하게 합니다.

```css
ul.nav-menu {
    /* 아래 코드를 하나씩 켜면서 보여줍니다 */
    display: flex;                  /* 가로 정렬의 시작 */
    flex-direction: row;            /* 방향을 가로로 확정 */
    justify-content: space-around;  /* 메뉴 사이 간격 균일화 */
    align-items: center;            /* 수직 중앙 정렬 */
}
```
)*

자, 드디어 2번째 세션입니다. 실제로 Flexbox라는 마법을 코드로 부려보겠습니다.
화면을 보세요. 여기 `lecture-demo/index.html` 파일이 있습니다. 우리가 지난주까지 열심히 만들었던 프로필 내용들이 있고, 상단에는 Home, About, Portfolio, Contact 라는 4개의 메뉴 글자들이 있습니다.

그런데 지금 상태를 보면, 이 메뉴들이 각각 한 줄을 통째로 차지하면서 수직으로 쭉 나열되어 있죠? `<ul>` 태그 안에 있는 `<li>` 태그들은 기본적으로 블록(Block) 요소이기 때문에, 자신이 있는 줄 전체를 전세 낸 것처럼 길게 공간을 차지합니다. 다른 요소가 자기 옆에 오는 것을 허락하지 않죠. 그래서 이렇게 못생긴 메뉴판이 되었습니다.

우리는 이 메뉴판을 브라우저의 왼쪽에서 오른쪽으로 예쁘게 눕혀야 합니다. 바로 이럴 때 Flexbox를 씁니다. 메뉴들을 감싸고 있는 엄마 태그, 즉 `<ul>` 태그를 CSS에서 부릅니다. 
`ul { ... }` 
그리고 그 안에 딱 한 줄을 선언합니다. 
`display: flex;`

*(Instructor Note: 라이브 코딩 창에서 코드를 쳐서 적용되는 순간 화면이 как 변하는지 보여줍니다.)*

여러분, 화면을 보세요! 제가 코드를 저장하는 순간, 아래로 길게 나열되었던 메뉴들이 순식간에 서로 왼쪽에서 오른쪽으로 나란히 줄을 섰죠? 이것이 바로 Flexbox의 위력입니다. 기본적으로 `display: flex;`를 엄마 요소에 선언하면, 그 자식 요소들은 가로 방향, 즉 `row` 방향으로 배치가 쫙 됩니다. 

그런데 아직은 메뉴들이 왼쪽 구석에 너무 따닥따닥 붙어서 다닥다닥 몰려 있네요. 좀 넓게 배치해 볼까요?
같은 괄호 안에 `justify-content: space-around;` 라고 코드를 한 줄 더 추가해 보겠습니다.

*(Instructor Note: space-around 적용 순간 화면의 변화를 지목합니다.)*

보셨나요? 마법처럼 메뉴들 사이의 공간이 공평하게 벌어졌습니다! 여러분이 브라우저 창의 크기를 마우스로 잡고 늘렸다 줄였다 해보세요. 브라우저 크기에 맞춰서 메뉴들 사이의 간격이 자동으로 늘어났다 줄어났다 하는 게 보이시죠? 이렇게 요소들을 고무줄처럼 유연하게(Flexible) 배치할 수 있기 때문에 이 기술의 이름이 Flexbox인 것입니다! 여러분은 이제 Flexbox를 활용해서 어떤 웹 페이지에서든 상단 네비게이션 메뉴바를 자유롭게 조립할 수 있게 되었습니다! 여러분이 만든 명함이나 자기소개 페이지에 당장 적용해 볼 수 있겠죠? 이게 바로 우리가 오늘 달성한 1단계 미션입니다.

## [S3] 레이아웃 리팩터링 (15분)

*(Instructor Note: 
[강사 사전 준비 및 디버깅 시연 순서]
1. `lecture-demo/index.html` 하단(90번째 줄 주변)에 있는 프로필 이미지(`<img class="profile-img-buggy">`)가 부모 컨테이너 밖으로 심하게 튀어나가 있는 고의적 버그 상황을 화면에 띄웁니다.
2. 학생들에게 브라우저 개발자 도구(F12)를 열어 엘리먼트 탭(Elements)에서 해당 이미지가 `width: 1000px;`로 고정되어 부모 박스(max-width: 800px)의 유연성을 무시하고 있음을 짚어줍니다.
3. 개발자 도구의 스타일 탭에서 직접 아래와 같이 코드를 수정하며 튀어나간 이미지가 예쁘게 들어가는 것을 시연합니다.

```css
.profile-img-buggy {
    /* 1. 원흉인 width: 1000px; 를 체크 해제(삭제)합니다. */
    /* 2. 아래 코드를 입력하여 이미지가 부모 크기를 넘지 못하게 합니다. */
    max-width: 100%;
    height: auto;
}
```
)*

이제 마지막 3번째 세션입니다. 여러분, 코딩을 하다 보면 우리가 입력하는 코드가 항상 내 마음처럼 움직여 주진 않죠. 화면을 보세요. 이 사이트는 아주 심각한 문제를 겪고 있습니다. 밑에 있는 사진이 본문 영역을 뚫고 밖으로 튀어나가 버렸고, 버튼들끼리 겹쳐 있어서 클릭조차 할 수 없는 상태입니다. 아주 엉망진창이죠. 

우리는 코드를 처음 작성하는 능력만큼이나 에러가 나거나 꼬여있는 기존의 코드를 고치고 개선하는, 즉 **리팩터링(Refactoring)** 하는 능력을 길러야 합니다. 이 버그를 잡으려면 무작정 코드를 지웠다 썼다 하면 안 됩니다. 문제를 진단할 청진기가 필요합니다. 브라우저에서 `F12` 키를 누르면 나오는 개발자 도구(Developer Tools)가 바로 의사들의 청진기입니다. 

화면의 개발자 도구를 켜서 이 튀어나간 요소 위에 마우스를 올려보겠습니다. 아, 보니까 이 자식 요소가 부모 상자의 크기 제한인 width 값 100%를 무시하고 픽셀값으로 강제로 고정되어 버렸네요. 부모 상자는 고무줄처럼 변하는데, 이 자식 이미지가 굳건한 쇳덩이처럼 본인의 픽셀 크기를 주장하다 보니 화면이 좁아질 때 밖으로 튀어나가는 현상(Overflow)이 발생한 것입니다.
이를 픽셀 고정이 아닌 `%` 비율이나 `max-width: 100%`로 유연하게 수용할 수 있게 코드를 바꾸도록 하겠습니다. 
자, 어떻습니까. 개발자 도구에서 코드를 바로 수정해서 쳐보니까 이제 이미지 파일이 예쁘게 부모 사이즈 안으로 쏙 들어와서 창이 줄어들면 같이 맞춰서 줄어들죠? 여러분이 만약 도구 사용법을 몰랐다면 저 이미지가 삐져나온 원인을 찾지 못해 하루하루 밤을 설쳤을 겁니다. 이렇게 디버깅은 도구를 써서 문제의 '원인'을 정밀 타격하는 과정입니다.

여러분! 오늘 배운 Flexbox 레이아웃과 리팩터링은 웹 페이지를 구조적으로 배치하는 핵심 무기입니다. 오늘 배운 내용을 여러분의 포트폴리오 사이트에 꼭 반영해 보시기 바랍니다.

숙제 안내드립니다. 오늘은 보안과 편리함이라는 현대 웹 개발의 중요 트렌드를 반영하여 Passkey 이메일 Microassignment가 있습니다. 과제 0.7 이메일 꼭 기억하시고, Handout을 참고하세요.

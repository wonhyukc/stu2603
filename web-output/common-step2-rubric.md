# 공통 2단계 QA 루브릭 (E트랙 학생 배포용)

**평가 대상 파일** (특정 주차 폴더 `{week}/` 하위)
- `handout.md` (학생 실습 지침서)
- `peer-eval-rubric.md` (상호평가 루브릭)

> **2단계 목적**: 1단계에서 확정된 강의 및 데모 내용을 바탕으로, 학생이 혼자서도 실습을 완수할 수 있는 가이드(`handout.md`)와 공정한 평가 기준(`peer-eval-rubric.md`)이 마련되었는지 검증한다.

**평가 기준 SSOT**
- `web-output/wb**/lecture.md` — 1단계에서 확정된 실습 시나리오
- `docs/assignment-micro.md` — 마이크로 과제 이메일 제출 내용
- `CLAUDE.md` — 언어 정책 (영어) 및 3섹션 구성 규칙

**만점**: 100점
**PASS 기준**: 85점 이상
**FAIL 시**: 보완 항목 목록 출력 후 즉시 수정

## 파트 A: handout.md (60점)

### A-1. 문서 구조 및 언어 규정 (20점) [CRITICAL]
- [ ] **20점**: 3대 필수 섹션(개념/실습/과제)이 모두 포함됨. **100% 영어**로 작성됨. 
- [ ] **FAIL**: 한국어 혼용 또는 섹션 누락 시 점수 무관 탈락.

### A-2. 실습 스텝 및 분량 (20점)
- [ ] **20점**: 1단계 `lecture.md`의 시나리오를 학생 관점의 단계별 지침으로 완벽히 변환함. 실습 시간이 최소 **25분 이상** 소요될 수 있는 충분한 난이도와 분량임.
- [ ] **10점**: 지침이 부실하거나 분량이 너무 적음.

### A-3. 마이크로 과제 SSOT 일치 (20점)
- [ ] **20점**: `docs/assignment-micro.md`의 해당 주차 내용 및 Boilerplate가 정확히 반영됨. 이메일 제목 형식 및 제출처 안내 포함.
- [ ] **0점**: SSOT와 다른 개별 과제를 부여함.

---

## 파트 B: peer-eval-rubric.md (40점)

### B-1. 실습-평가 1:1 무결성 (20점)
- [ ] **20점**: `handout.md`에 지시된 실습 미션이 루브릭의 평가 항목에 빠짐없이 1:1로 대응됨.
- [ ] **0점**: 실습하지 않은 내용을 평가하거나, 실습한 내용이 평가에서 누락됨.

### B-2. 단일 조건 평가 원칙 (20점)
- [ ] **20점**: 한 문항에 오직 하나의 확인 조건만 있음(예: "배경색과 폰트가 바뀌었나?" X -> "배경색이 바뀌었나?" O). Yes 1 / No 0 방식 준수.
- [ ] **10점**: 일부 문항에 복합 조건이 섞여 있음.

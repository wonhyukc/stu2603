## 1. Grading Criteria (1 point each, Total 3 points) / 채점 기준 (각 1점, 총 3점 만점)

The evaluator checks the submitted link and grades the following three questions simply and clearly with **Yes(1 point) / No(0 points)**.
평가자는 제출된 링크를 확인하고, 아래 세 개의 문항에 대해 **Yes(1점) / No(0점)** 로 단순명료하게 채점합니다.

| No. / 번호 | Short-Answer Objective Evaluation Criteria (Yes/No) / 단답형 객관적 평가 기준 (Yes/No) | Points / 배점 | Check Point / 확인 포인트 |
|:---:|:---|:---:|:---|
| **1** | Is the submitted answer link in the correct **GitHub link (URL)** format? <br> 제출된 답안의 링크가 올바른 **GitHub 링크(URL)** 형식인가? | **1 pt / 1점** | Visually verify if it is in the format `github.com/[id]/[repository]/commit/...`. <br> `github.com/[아이디]/[리포지토리]/commit/...` 형태의 주소인지 눈으로 확인합니다. |
| **2** | Does the submitted link address clearly include a unique **commit hash** value? <br> 제출된 링크 주소에 고유한 **커밋 해시(hash)** 값이 분명하게 포함되어 있는가? | **1 pt / 1점** | Check if the link address ends with a long, complex mixture of alphanumeric characters representing the `hash` value. <br> 링크 주소 끝에 길고 복잡한 영문자와 숫자가 섞인 `hash` 값이 포함되어 있는지 확인합니다. |
| **3** | When you click and open the submitted link, does the **actual completed HTML code** appear normally on the screen without a 404 error, etc.? <br> 제출된 링크를 클릭하여 열었을 때, 404 에러 등 없이 **실제 작성된 완성 HTML 코드** 내용이 화면에 정상적으로 나타나는가? | **1 pt / 1점** | Check if you can see the code of the modified file (such as `index.html`) after accessing it. <br> 접속 후 수정된 파일(`index.html` 등)의 코드를 볼 수 있는지 확인합니다. |

## 2. Special Guidelines for Evaluators (Feedback Writers) / 평가자(피드백 작성자) 특별 가이드라인

- **Maintain Objectivity / 객관성 유지**: Do not deduct points based on whether the submitted digital business card code is well-structured in HTML or if the photo is placed well. Only check if **access and verification (the 3 items above)** are working normally.
제출한 디지털 명함 코드가 HTML 구조에 맞게 잘 짜여졌는지, 사진이 잘 들어갔는지 등의 여부로 감점하지 마십시오. 오직 **접속과 확인(위 3가지)**이 정상적으로 가능한지 여부만 체크합니다.

- **Award Full Marks / 만점 부여**: Even if the code is very short or just at the level of typing an example, if it meets the above 3 conditions, you must award **full marks (3 points)** without deduction.
코드가 정말 짧거나 단순히 예제만 타이핑한 수준이라도, 위 3가지 조건만 맞으면 감점 없이 **만점(3점)**을 주어야 합니다.

- If you write a feedback comment, please write light encouragement or confirmation of successful normal access rather than subjective criticism. (e.g., "All 3 conditions check out well. The link opens well!")
피드백 코멘트를 쓴다면 주관적 비판보다는 가벼운 격려나 성공적인 정상 접속 여부를 적어주시면 됩니다. (예: "3가지 조건 모두 잘 확인됩니다. 링크가 잘 열리네요!")

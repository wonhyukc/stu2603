# 2주차 동료 상호평가 루브릭 (HTML 기초)

본 문서는 **wb02(2주차) HTML 기초** 과정에서 **eCampus (Moodle)**에 제출되는 `my-intro`(나만의 디지털 명함) 실습 과제의 상호평가 채점 기준입니다.

주관적 평가(내용, 디자인)를 배제하고 오직 명백한 **3가지 객관적 기준**에 따라 채점합니다.

## 1. 평가 개요
- **대상 항목**: `my-intro` GitHub 커밋 링크 제출물
- **제출처**: eCampus (Moodle) 상호평가 게시판
- **평가자**: 동료 수강생 (최소 3명 이상 배정)

## 2. 채점 기준 (각 1점, 총 3점 만점)

평가자는 제출된 링크를 확인하고, 아래 세 개의 문항에 대해 **Yes(1점) / No(0점)** 로 단순명료하게 채점합니다.

| 번호 | 단답형 객관적 평가 기준 (Yes/No) | 배점 | 확인 포인트 |
|:---:|:---|:---:|:---|
| **1** | 제출된 답안의 링크가 올바른 **GitHub 링크(URL)** 형식인가? | **1점** | `github.com/[아이디]/[리포지토리]/commit/...` 형태의 주소인지 눈으로 확인합니다. |
| **2** | 제출된 링크 주소에 고유한 **커밋 해시(hash)** 값이 분명하게 포함되어 있는가? | **1점** | 링크 주소 끝에 길고 복잡한 영문자와 숫자가 섞인 `hash` 값이 포함되어 있는지 확인합니다. |
| **3** | 제출된 링크를 클릭하여 열었을 때, 404 에러 등 없이 **실제 작성된 완성 HTML 코드** 내용이 화면에 정상적으로 나타나는가? | **1점** | 접속 후 수정된 파일(`index.html` 등)의 코드를 볼 수 있는지 확인합니다. |

## 3. 평가자(피드백 작성자) 특별 가이드라인

- **객관성 유지**: 제출한 디지털 명함 코드가 HTML 구조에 맞게 잘 짜여졌는지, 사진이 잘 들어갔는지 등의 여부로 감점하지 마십시오. 오직 **접속과 확인(위 3가지)**이 정상적으로 가능한지 여부만 체크합니다.
- **만점 부여**: 코드가 정말 짧거나 단순히 예제만 타이핑한 수준이라도, 위 3가지 조건만 맞으면 감점 없이 **만점(3점)**을 주어야 합니다.
- 피드백 코멘트를 쓴다면 주관적 비판보다는 가벼운 격려나 성공적인 정상 접속 여부를 적어주시면 됩니다. (예: "3가지 조건 모두 잘 확인됩니다. 링크가 잘 열리네요!")

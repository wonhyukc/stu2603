## 1. 채점 기준 (총점 3점)

평가자는 아래 세 가지 문항에 대해 **Yes(1점) / No(0점)** 로만 단순명료하게 채점합니다.
*(주의: 작성된 코드나 디자인이 우수한가 등은 절대 채점 기준에 포함하지 않습니다!)*

| 번호 | 단답형 객관적 평가 기준 (Yes/No) | 배점 |
|:---:|:---|:---:|
| **1** | **기한 내 제출**<br>과제를 기한 내에 지각하지 않고 제출하였습니까? | **5점** |
| **2** | 제출된 답안의 링크가 올바른 **GitHub 링크(URL)** 형식인가? | 1점 |
| **3** | 제출된 링크를 클릭하여 열었을 때, 에러(404 등) 없이 **실제 작성된 화면**이 나타나는가? | 1점 |
| **4** | 결과물 화면에서 **명함에 최소한 한 가지 이상의 추가 CSS 스타일(색상, 폰트 등)**이 눈에 띄게 적용되어 있는가? | 1점 |

## 2. 평가자(피드백 작성자) 특별 가이드라인

- **객관성 유지**: 오직 위 3가지 조건이 충족되었는지만 기계적으로 확인하십시오.
- 디자인의 미적 수준이나 스케일이 다르더라도, 조건만 맞으면 감점 없이 **만점(3점)**을 주어야 합니다.
- 피드백 코멘트를 쓴다면 주관적 비판보다는 가벼운 격려나 성공적인 정상 접속 여부를 적어주시면 됩니다. (예: "3가지 조건 모두 잘 확인됩니다. 고생하셨습니다!")

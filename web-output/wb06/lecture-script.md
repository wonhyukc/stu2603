## [S1] 박스 모델: 웹의 모든 요소는 '상자'다 (약 25분)

*(강사 참고: PPT 슬라이드에 예쁘게 포장된 택배 상자 이미지와 그 내부 구조(상품, 뽁뽁이, 박스 두께, 박스 사이의 거리)를 시각화한 다이어그램을 띄웁니다.)*

여러분, 지난주에 우리는 웹페이지에 **색**을 입히고 **폰트**를 바꾸는 디자인의 기초를 배웠습니다. 흑백의 삭막한 명함이 여러분이 좋아하는 브랜드의 색깔로 물드는 과정을 보며 다들 눈이 반짝이던 모습이 기억나네요. 하지만 디자인이라는 것은 단순히 색을 칠하는 것에서 끝나지 않습니다. 우리가 옷을 입을 때 색 조합도 중요하지만, 더 중요한 것은 무엇인가요? 바로 **'핏(Fit)'**과 **'거리감'**입니다. 바지가 너무 벙벙하거나, 셔츠가 너무 꽉 끼면 아무리 비싸고 예쁜 색의 옷이라도 멋이 나지 않죠. 웹 디자인도 마찬가지입니다. 글자와 그림이 서로 숨 막히게 딱 붙어 있거나, 화면 구석에 너무 처박혀 있으면 사용자는 답답함을 느낍니다. 이 '여백'과 '배치'를 결정하는 CSS의 가장 핵심적인 원리가 바로 오늘 배울 **박스 모델(Box Model)**입니다.

먼저 여러분의 고정관념을 완전히 깨부수는 선언을 하나 하겠습니다. **웹페이지에 보이는 모든 것—글자 한 자, 이미지 한 장, 버튼 하나—은 사실 모두 '네모난 상자' 안에 들어 있습니다.** 여러분 눈에는 둥근 버튼이 보이고, 끝이 뾰족한 로고가 보일지 모르지만, 컴퓨터는 그것들을 모두 사각형의 상자로 취급합니다. 이 사실을 받아들이는 것이 박스 모델 이해의 첫걸음입니다.

자, 앞에 띄운 **택배 상자** 이미지를 함께 봅시다. 우리가 해외 직구로 귀한 물건을 주문했다고 상상해 보세요. 박스 모델의 구성을 이 택배에 비유하면 10살 어린아이도 바로 이해할 수 있습니다.

첫 번째, **내용물(Content)**입니다. 이것은 여러분이 주문한 진짜 '물건' 그 자체입니다. 웹으로 치면 우리가 화면에 적은 **글자(Text)**나 **이미지**가 되겠죠. 상자 안에서 가장 핵심이 되는 주인공입니다. 

두 번째, **패딩(Padding)**입니다. 물건을 그냥 박스에 넣으면 배송 중에 깨질 수 있겠죠? 그래서 우리는 물건 주위에 **'뽁뽁이(에어캡)'**를 감쌉니다. 물건과 박스 종이 사이의 안쪽 공간이죠. CSS에서 패딩은 **내용물과 테두리 사이의 안쪽 여백**을 의미합니다. 내가 글자를 상자의 정중앙에 두고 싶거나, 상자 안에서 글자가 너무 답답해 보이지 않게 숨통을 틔워주고 싶을 때 이 패딩 뽁뽁이를 두껍게 감싸면 됩니다.

세 번째, **테두리(Border)**입니다. 이것은 물건을 담고 있는 **'종이 상자'** 그 자체의 두께와 재질입니다. 상자의 실체죠. 테두리를 아예 안 보이게 할 수도 있고, 아주 두꺼운 철제 상자처럼 만들 수도 있습니다. 이 테두리가 있어야 비로소 상자의 경계가 명확해집니다.

마지막 네 번째, **마진(Margin)**입니다. 이것은 내 택배 상자와 옆집 택배 상자 사이의 **'사회적 거리두기'**입니다. 상자 바깥쪽의 여백이죠. 우리 집 현관 앞에 택배가 수십 개 쌓여 있는데, 상자끼리 너무 딱 붙어 있으면 집어 올리기 힘들겠죠? 상자들끼리 서로 부딪히지 않게 적당한 거리를 띄워주는 것이 바로 마진입니다.

정리해 볼까요? **[안쪽 여백은 패딩, 상자 자체는 보더, 바깥쪽 거리는 마진]**입니다. 이 세 가지 이름만 완벽히 외우면 오늘 수업의 절반은 끝난 겁니다.

이제 실전 코드로 이 개념들을 어떻게 조작하는지 직접 보여드리겠습니다. 다들 앞의 StackBlitz 화면을 주목해 주세요. 제가 `div`라는 이름의 빈 상자를 하나 만들고, 그 안에 'Hello Box'라는 글자를 넣었습니다. 지금은 아무런 스타일이 없어서 상자의 형체가 보이지 않죠?

*(강사가 라이브로 코딩 시작)*
```css
.my-box {
  border: 5px solid blue;
}
```
보이시나요? 파란색 테두리가 생기면서 드디어 상자의 실체가 드러났습니다. 그런데 글자가 테두리에 너무 바짝 붙어 있어서 보기가 안 좋네요. 마치 좁은 감옥에 글자가 갇힌 것 같습니다. 자, 여기서 **뽁뽁이**를 투입하겠습니다.

```css
.my-box {
  border: 5px solid blue;
  padding: 30px;
}
```
와! `padding`을 주자마자 글자가 상자 안쪽에서 편안하게 가운데로 밀려나며 여유를 찾았습니다. 상자 자체가 커진 것처럼 보이죠? 맞습니다. 안쪽 여백을 채우면 상자의 전체 크기도 그만큼 커집니다. 

그럼 이제 이 상자를 옆으로 좀 치워보겠습니다. 지금은 화면 왼쪽 구석에 너무 붙어 있거든요. 상자 바깥쪽 공간, 즉 **사회적 거리두기**인 `margin`을 적용해 보죠.

```css
.my-box {
  border: 5px solid blue;
  padding: 30px;
  margin: 100px;
}
```
네, 완벽합니다! 상자가 화면 구석에서 100픽셀만큼 멀어지며 자리를 잡았습니다. 

여러분, 여기서 아주 중요한 팁 하나를 드릴게요. 우리가 옷을 입을 때 상의, 하의, 왼쪽 소매, 오른쪽 소매를 따로 입듯이 패딩과 마진도 상하좌우를 각각 따로 조절할 수 있습니다. `margin-top`, `margin-bottom`, `margin-left`, `margin-right`처럼 말이죠. 만약 여러분이 만든 명함에서 제목 글자 아래쪽만 살짝 띄우고 싶다면 `margin-bottom`만 콕 집어서 값을 주면 되는 겁니다.

자, 여기까지가 박스 모델의 기초입니다. "안쪽은 패딩, 바깥쪽은 마진!" 이 구호가 머릿속에 박혔다면 이제 우리는 웹페이지의 모든 요소들을 우리가 원하는 위치에 자유자재로 배치할 수 있는 강력한 마법을 손에 넣은 것입니다.

## [S2] 요소 정렬 및 개발자 도구 디버깅 실전 (약 25분)

*(강사 참고: 크롬 브라우저를 열고 F12를 눌러 개발자 도구를 활성화합니다. 실제 웹사이트의 특정 요소를 찍어서 박스 모델이 어떻게 시각화되는지 보여줍니다.)*

이제 박스 모델이라는 강력한 개념을 배웠으니, 이것을 실제 현장에서 어떻게 확인하고 고치는지 '전문가의 기술'을 전수해 드리겠습니다. 여러분, 개발자가 코드를 백날 천날 타이핑만 한다고 해서 화면이 뚝딱 완성되지 않습니다. 사실 개발을 하는 시간보다 **"왜 내가 생각한 대로 화면이 안 나오지?"**라며 원인을 찾는 시간이 훨씬 더 깁니다. 이때 우리를 구원해 줄 유일한 도구가 바로 **크롬 개발자 도구(F12)**입니다.

자, 다들 고개를 들어 스크린을 보세요. 제가 지금 어떤 유명한 웹사이트를 하나 열었습니다. 여기서 마우스 오른쪽 버튼을 누르고 가장 아래에 있는 **'검사(Inspect)'**를 클릭해 보겠습니다. 화면 오른쪽에 복잡한 창이 하나 뜨죠? 이게 바로 전 세계 모든 웹 개발자들의 비밀 무기인 개발자 도구입니다.

개발자 도구의 오른쪽 하단 영역을 자세히 봐주십시오. 거기 보면 아까 우리가 배운 박스 모양의 그림이 그려져 있는 것을 볼 수 있습니다. **Content, Padding, Border, Margin**이라는 글자가 보이죠? 제가 화면상의 특정 버튼 위로 마우스를 가져가면, 브라우저가 친절하게 색깔별로 여백을 표시해 줍니다. 하늘색은 내용물, 초록색은 패딩, 주황색은 마진입니다.

이게 왜 중요할까요? 가끔 우리가 코드를 짤 때 "어? 나는 마진을 안 줬는데 왜 상자가 저만큼 떨어져 있지?"라고 의문이 생길 때가 있습니다. 그럼 이 도구를 열어서 확인해 보는 겁니다. 아하! 내가 준 게 아니라 브라우저가 기본적으로 설정해 놓은 마진이 숨어 있었구나! 하고 범인을 바로 잡을 수 있는 거죠. 이제 여러분은 눈대중으로 디자인하는 아마추어가 아니라, 픽셀 단위의 수치를 눈으로 직접 확인하며 다스리는 '데이터 기반의 디자이너'가 된 것입니다.

자, 이제 박스 모델에서 가장 골치 아프고, 그래서 초보자들이 가장 많이 당황해하는 비밀스러운 현상 하나를 소개해 드릴게요. 이름하여 **'마진 상쇄(Margin Collapse)'** 현상입니다. 

상상해 보세요. 위아래로 나란히 있는 두 개의 상자가 있습니다. 위쪽 상자에게 "너 아래쪽으로 50픽셀만큼 마진(거리)을 띄워!"라고 명령하고, 아래쪽 상자에게는 "너 위쪽으로 50픽셀만큼 마진을 띄워!"라고 명령했습니다. 그럼 두 상자 사이의 거리는 총 얼마가 되어야 할까요? 상식적으로는 50 더하기 50이니까 100픽셀이 되어야 하겠죠?

하지만 CSS의 세계는 그렇게 호락호락하지 않습니다. 직접 코드로 확인해 볼까요?

*(강사가 라이브로 위아래 상자에 각각 마진을 주는 코딩 시연)*
```css
.top-box { margin-bottom: 50px; }
.bottom-box { margin-top: 50px; }
```
보이세요? 두 상자 사이의 간격이 100픽셀이 아니라 50픽셀로 '합쳐져' 버렸습니다! 마치 두 사람이 서로 50센티미터씩 거리를 두라고 했더니, 한쪽이 50센티미터 떨어져 있는 공간 안으로 다른 한 명이 쏙 들어가 버린 꼴입니다. 이것이 바로 마진 상쇄입니다. 이웃한 상자들끼리는 서로의 마진을 개별적으로 더하는 게 아니라, 더 큰 마진 값 하나로 합쳐버리는 독특한 규칙이 있습니다. 

"아니, 왜 이렇게 불편하게 만들어놨나요?"라고 물을 수도 있습니다. 하지만 이건 웹페이지를 길게 나열할 때 문단 사이의 간격이 너무 벌어지지 않게 하려는 브라우저의 배려이자 오래된 약속입니다. 이것을 모르면 나중에 "왜 내 마진은 계산대로 안 먹지?" 하며 밤새 모니터를 붙잡고 울게 될지도 모릅니다. 하지만 오늘 여러분은 그 범인을 보았습니다. 이제 마진이 합쳐져도 "아, 마진 상쇄군!" 하고 쿨하게 넘길 수 있는 고수가 된 겁니다.

수업 S2의 마지막으로, 레이아웃을 잡을 때 여러분의 스트레스를 90% 줄여줄 **마법의 코드 한 줄**을 선물로 드리고 실습으로 넘어가겠습니다. 박스 모델을 다루다 보면 패딩이나 테두리를 줄 때마다 상자의 전체 크기가 자꾸 커져서 오른쪽으로 삐져나가거나 아랫줄로 툭 떨어지는 현상이 생깁니다. 내가 지정한 박스 크기 안에 패딩과 테두리를 모두 '강제로' 포함시켜서 크기가 절대 변하지 않게 고정하는 마법의 속성, 바로 `box-sizing: border-box;`입니다. 

*(강사 액션: 코드 최상단 `*` 선택자에 해당 속성을 적용하는 시연)*
이 한 줄을 적는 순간, 여러분은 박스 크기 계산이라는 수학적 지옥에서 해방될 것입니다. 이 마법의 주문을 꼭 기억해 두세요.

## [S3] 버튼 간격 버그 수정 및 고급 구글링 특강 (약 25분)

*(강사 참고: S3는 두 부분으로 나눕니다. 전반전은 박스 모델 실무 디버깅, 후반전은 '정보 사냥꾼' 특강입니다.)*

자, 이제 오늘 수업의 마지막 섹션이자, 여러분의 디지털 생존 능력을 한 단계 업그레이드해 줄 '특강' 시간이 포함된 S3를 시작합니다. 

먼저 우리가 지난주부터 업그레이드해 오고 있는 '나만의 명함' 프로젝트를 확인해 봅시다. 다들 아래쪽에 '방명록'이나 '이메일 보내기' 버튼 같은 걸 만들어 두었죠? 그런데 지금 그 버튼들이 어떤가요? 서로 의리 있게(?) 딱딱 어깨를 나란히 하고 붙어 있습니다. 너무 붙어 있어서 잘못 누를 것 같네요. 방금 배운 박스 모델을 활용해서 버튼 사이에 숨통을 틔워주는 **'라이브 디버깅'**을 함께 해보겠습니다.

*(강사 액션: 버튼 요소에 마진을 주어 정렬하는 라이브 코딩 시연)*
버튼 클래스에 `margin-right: 15px;`를 한 줄 넣어보겠습니다. 짠! 이제 버튼들이 서로 예의를 지키며 기분 좋게 떨어졌습니다. 이렇게 박스 모델은 정교한 UI를 만드는 핵심 도구가 됩니다. 

자, 이제 코딩 이야기는 잠시 접어두고, 여러분이 나중에 직장에 가든, 자기 사업을 하든, 아니면 남은 3년의 대학 생활을 하든 여러분의 몸값을 수십 배로 높여줄 진짜 비밀 기술을 하나 알려드리겠습니다. 제가 이름 붙이기를 **"정보 사냥꾼: 고급 구글 검색 전략"**입니다.

여러분, 모르는 게 생기면 구글이나 제미나이(AI)에게 물어보시죠? 그런데 왜 똑같은 질문을 해도 어떤 친구는 1초 만에 완벽한 해결책을 찾아오고, 어떤 친구는 한 시간 내내 검색해도 쓰레기 정보만 가득한 페이지에서 헤매고 있을까요? 그건 검색이 **운**이 아니라 **기술**이기 때문입니다.

오늘 제가 여러분께 전수할 '구글 연산자' 30초 훈련을 따라 해보세요. 

첫 번째, **특정 사이트 안에서만 사냥하기 (`site:`)**입니다.
여러분이 CSS 박스 모델에 대해 공부하다가, 다른 수많은 블로그의 잡다한 설명 말고 'w3schools.com'이라는 공식 학습 사이트의 내용만 딱 보고 싶습니다. 그럴 땐 구글 검색창에 이렇게 치세요.
`site:w3schools.com css box model`
이렇게 하면 구글은 전 세계 수억 개의 사이트 중 오직 w3schools라는 영역 안에서만 정보를 뒤져서 여러분께 갖다 바칩니다. 훨씬 정확하겠죠?

두 번째, **반드시 포함해야 할 단어 강제하기 (`""`)**입니다.
검색어에 큰따옴표를 붙이면 구글은 그 단어가 '토씨 하나 틀리지 않고 그대로' 들어있는 문법만 찾습니다. 우리가 코딩 에러 메시지를 검색할 때 이 큰따옴표를 쓰지 않으면 비슷한 엉뚱한 정보들이 쏟아집니다. 반드시 에러 메시지 전문을 큰따옴표로 묶어서 검색하세요.

세 번째, **불필요한 쓰레기 정보 치워버리기 (`-`)**입니다.
만약 여러분이 '파이썬'에 대해 찾고 싶은데, 자꾸 뱀(동물)에 대한 정보가 섞여 나온다면? 검색어 뒤에 `-동물` 혹은 `-snake`라고 적어보세요. 그러면 해당 단어가 들어간 결과는 구글이 알아서 쓰레기통에 버려줍니다.

자, 여기서 갑작스러운 **타임어택 미션**을 하나 드립니다. 지금 바로 여러분의 노트북을 켜고 구글을 여세요.

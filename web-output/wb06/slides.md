## [S1] 박스 모델 - 웹의 모든 요소는 '상자'다
### "모든 텍스트와 이미지는 보이지 않는 상자에 담겨 있다"

*   **택배 상자 비유**
    *   **내용물 (Content)**: 주문한 진짜 물건 (텍스트, 이미지)
    *   **패딩 (Padding)**: 상자 안의 뽁뽁이 (안쪽 여백)
    *   **테두리 (Border)**: 종이 상자 그 자체 (경계선)
    *   **마진 (Margin)**: 옆집 상자와의 거리 (바깥쪽 여백)

*   **한 줄 요약: "안쪽은 패딩, 바깥쪽은 마진!"**

---

## [S2] 요소 정렬 및 개발자 도구 디버깅
### "눈대중이 아닌 데이터로 디자인하라"

*   **개발자 도구 (F12) 활용**
    *   '검사(Inspect)'를 통해 실시간 박스 모델 수치 확인
    *   색상별 구분: 파랑(Content), 초록(Padding), 주황(Margin)

*   **마진 상쇄 (Margin Collapse) 현상**
    *   위아래로 인접한 상자들의 마진이 하나로 합쳐지는 현상
    *   "50 + 50 = 50?" -> 더 큰 값 하나만 남는 브라우저의 규칙

---

## [S3] 버튼 간격 버그 및 `border-box`
### "레이아웃이 깨지지 않게 만드는 마법"

*   **레이아웃 해결사: `box-sizing: border-box;`**
    *   패딩과 테두리를 상자 크기 안에 포함시켜 크기 고정
    *   수학적 계산 지옥에서 벗어나는 필수 속성

*   **실습 미션: [Build-up 3단계]**
    *   지난주 만든 방명록 버튼들 사이에 `margin` 주기
    *   상하좌우 개별 속성(`margin-top` 등) 활용하기

## [특강] 정보 사냥꾼: 고급 구글 검색 전략
### "검색은 운이 아니라 기술이다"

1.  **특정 사이트 집중 사냥 (`site:`)**
    *   예: `site:w3schools.com css box model`
2.  **정확한 문구 강제 검색 (`""`)**
    *   예: `"margin-bottom not working"` (에러 메시지 검색 필수)
3.  **불필요한 정보 제외 (`-`)**
    *   예: `python -snake` (동물 정보 제외)
4.  **파일 유형 지정 (`filetype:`)**
    *   예: `css cheat sheet filetype:pdf`

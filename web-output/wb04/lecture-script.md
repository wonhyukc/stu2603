## Section 1: CSS란? 옷 입히기 비유와 스타일 적용 방식 (25분)

여러분, 한 주간 잘 지내셨나요? 여러분의 강사 정원혁입니다. 온라인으로 만나 뵙게 되어 여전히 반갑습니다. 인도, 네팔, 방글라데시, 베트남 등 여러 나라에서 접속해주고 계신 여러분 모두 각자의 날씨는 어떤지 궁금하네요. 지난 3주 동안 우리는 웹페이지의 '뼈대'를 만드는 HTML과, 컴퓨터와 소통하는 '입력 폼'에 대해 배웠습니다. 혹시 기억하시나요? HTML은 마치 집을 지을 때 뼈대를 세우고 콘크리트를 들이붓는 것과 같습니다. 무척 단단하고 필수적이지만, 사실 아직 예쁘지는 않습니다. 

우리가 인터넷에서 매일 보는 멋진 웹사이트들, 혹은 여러분이 좋아하는 나이키나 아디다스 같은 브랜드의 세련된 홈페이지, 넷플릭스의 화려한 화면은 모두 어떻게 만들어졌을까요? 뼈대만으로는 그 아름다움을 낼 수 없습니다. 그래서 오늘, 우리는 웹페이지에 '옷'을 입히고, '화장'을 하고, 예쁘게 '페인트칠'을 하는 마법 같은 언어인 CSS에 대해 배울 것입니다. 

**(화면 전환: 사람의 엑스레이 뼈대 사진과 예쁘게 옷을 입은 모델 사진을 나란히 띄운다)**

화면을 한 번 보시겠어요? 왼쪽의 사진은 뼈만 앙상한 엑스레이 사진이고, 오른쪽은 화려하고 멋진 옷을 차려입은 패션모델의 사진입니다. 이게 바로 지난주까지 우리가 배운 HTML과 오늘부터 배울 CSS의 차이입니다. CSS는 Cascading Style Sheets의 약자입니다. 이름이 조금 복잡하죠? 폭포수처럼 위에서 아래로 흘러내리는 스타일 정의서라는 뜻인데, 원리는 아주 간단합니다. "저기 있는 저 글자는 빨간색으로 칠해라", "저기 있는 저 버튼은 둥글고 파란색으로 만들어라"라고 컴퓨터에게 지시를 내리는 언어입니다.

우리가 날씨가 추우면 두꺼운 패딩을 입고, 더우면 얇은 반팔 티셔츠를 입듯이, 웹페이지도 CSS를 통해 전혀 다른 모습으로 바뀔 수 있습니다. 똑같은 HTML 뼈대를 가지고도, 어떤 CSS를 입히느냐에 따라 전혀 다른 웹사이트처럼 보이게 만들 수 있으니까요. 오늘 여러분은 드디어 흑백의 칙칙한 웹페이지에서 벗어나, 여러분이 좋아하는 다채로운 색상을 마음껏 사용해볼 것입니다.

자, 그렇다면 이 예쁜 옷(CSS)을 HTML이라는 뼈대에 어떻게 입힐 수 있을까요? 옷을 입히는 방법에는 크게 세 가지가 있습니다. 첫째는 '인라인(Inline) 스타일'이고, 둘째는 '내부(Internal) 스타일시트', 그리고 셋째는 '외부(External) 스타일시트'입니다. 이름만 들어서는 감이 잘 안 오실 테니 제가 실생활의 비유로 쉽게 설명해드릴게요.

첫 번째, **인라인 스타일(Inline Style)** 입니다. 이 방식은 문신(Tattoo)을 새기거나 혹은 티셔츠에 직접 물감을 칠하는 것과 같습니다. HTML 태그 안에 직접 스타일을 적어넣는 거죠. 예를 들어 `<h1 style="color: blue;">제목입니다</h1>`라고 적는 식입니다. 즉각적으로 효과를 볼 수 있어서 빠르고 좋지만, 만약에 제목 글자가 100개라면? 100개의 태그에 일일이 이 문신을 새겨야 합니다. 나중에 색깔을 빨간색으로 바꾸고 싶어지면 100군데를 모두 수정해야겠죠? 굉장히 피곤한 일입니다.

두 번째, **내부 스타일시트(Internal Style Sheet)** 입니다. 이것은 방 안에 있는 옷장과 같습니다. HTML 문서의 위쪽, 그러니까 `<head>` 태그 안에 `<style>`이라는 큰 옷장을 하나 만들어두고, "이 문서 안에 있는 제목 글자들은 모두 파란색 옷을 입어!"라고 한 번에 선언하는 방식입니다. 인라인 방식보다는 훨씬 편리해졌죠. 그 문서 안에서는 이 스타일이 모두 적용되니까요. 하지만 또 문제가 생깁니다. 여러분의 웹사이트가 10페이지짜리로 늘어난다면 어떻게 될까요? 10페이지에 동일한 스타일을 적용하려면 똑같은 내용의 `<style>` 태그를 10군데에 모두 복사해서 붙여넣어야 합니다.

세 번째, **외부 스타일시트(External Style Sheet)** 입니다. 이것이 바로 우리가 가장 많이 사용하고 권장하는 방법입니다! 우리 반 학생 전원이 공유하는 거대한 외부 중앙 옷장을 두고, 각자 필요한 페이지에서 "그 중앙 옷장 좀 가져다 쓸게요"라고 연결만 해두는 방식입니다. `style.css`라는 별도의 전용 파일을 만들고, HTML의 `<head>` 태그에서 `<link>` 태그를 이용해 연결만 시켜주는 겁니다. 이렇게 하면, 100페이지 1000페이지짜리 웹사이트라도 단 하나의 `style.css` 파일만 수정하면 웹사이트 전체의 색상이 바뀌는 마법을 경험할 수 있습니다. 

*(강사 참고: 방금 설명한 세 가지 방식의 차이를 StackBlitz 화면을 띄워 라이브 코딩으로 시연합니다.)*

자, 화면을 보세요. 제가 방금 설명한 세 가지 방식을 순서대로 보여드리겠습니다. 먼저 인라인, 여기 `style=` 속성을 보시죠. 파란색으로 변했습니다. 다음은 내부 스타일시트, `<head>` 안에 이렇게 모아두니 본문이 한결 깔끔해지죠? 마지막으로 가장 중요한 외부 스타일시트입니다. 왼쪽에 `style.css`라는 새 파일을 만들고 코드를 싹 옮겼습니다. 그리고 HTML에서는 이 코드를 불러오기만 합니다. 어떤가요? 관리가 훨씬 쉬워졌다는 게 눈에 보이시죠? 앞으로 여러분이 웹 개발을 할 때는 항상 이 외부 스타일시트를 사용하게 될 겁니다.

## Section 2: 기본 스타일링 시연 및 선택자 패턴 (25분)

이제 본격적으로 CSS를 작성하는 방법을 알아봅시다. CSS 문법은 사실 한국인도, 외국인도 아주 쉽게 이해할 수 있을 만큼 인간의 말과 비슷합니다. 영어를 조금만 알면 금방 이해할 수 있죠. 핵심은 두 가지입니다. "누구에게?" 그리고 "어떤 옷을 입힐까?". 

이때 "누구에게?" 옷을 입힐지 지목하는 녀석을 우리는 **선택자(Selector)** 라고 부릅니다. 아주 매력적인 이름이죠. 누군가를 '선택'한다는 뜻이니까요. 그리고 괄호 `{}` 안에 들어가는 내용이 바로 "어떤 옷을 입힐까?"에 해당합니다. 이것을 '속성(Property)'과 '값(Value)'이라고 부릅니다. 

예를 들어, `글자색을 빨간색으로 바꿔라`는 CSS를 작성한다면 이렇게 씁니다. 먼저 "모든 p 태그에게!"라고 지목합니다. `p { color: red; }`. 이게 끝입니다. 아주 직관적이지 않나요? 

가장 기본이 되는 세 가지 스타일 속성을 먼저 배워보겠습니다. 
첫째, **color**. 이것은 글자의 색상을 바꿉니다. 
둘째, **background-color**. 바탕색, 즉 배경색을 의미합니다. 
셋째, **font-size**. 글자의 크기를 지정하죠. 단위로는 보통 픽셀(px)을 많이 씁니다.

자, 화면을 보겠습니다. 여러분이 지난주에 만들었던 나만의 온라인 명함 뼈대가 있네요. 너무 하얗고 딱딱해서 이력서 같아 보입니다. 여기에 제가 색을 입혀 보겠습니다. `<body style="background-color: lightgray;">`라고 해볼까요? 하지만 잠깐, 방금 제가 첫 번째 시간에 인라인 스타일은 가급적 쓰지 말라고 했었죠? 네, 지우고 `style.css` 파일로 가보겠습니다. 그리고 이렇게 적어봅니다.

```css
body {
  background-color: lightgray;
}

h1 {
  color: navy;
  font-size: 30px;
}
```

결과화면을 새로고침해보면 와, 전체 배경이 부드러운 회색으로 변했고, 가장 큰 제목인 h1의 글자색이 남색으로, 크기가 적당히 커졌습니다. 

그런데 여기서 한 가지 큰 딜레마에 부딪힙니다. 화면에는 h1 태그도 하나지만 p 태그도 여러 개가 있을 수 있죠. 메뉴에 있는 글자도 p 태그, 프로필 소개도 p 태그, 주소를 나타내는 것도 p 태그라고 합시다. 

제가 "메뉴 글자만 파란색으로 바꾸고 싶어"해서 `p { color: blue; }` 라고 선언하면 어떻게 될까요? 네, 당연히 화면에 있는 모든 p 태그의 글자가 죄다 파란색으로 변해버립니다. 우리는 컴퓨터에게 "오직 메뉴에 있는 글자만 색을 바꿔줘"라고 정확하게 알려주어야 합니다. 이럴 때 필요한 것이 바로, **클래스(Class)** 와 **아이디(ID)** 입니다.

이 부분을 아주 분명하게 이해해야 합니다. 클래스와 아이디는 학생들에게 이름표를 달아주는 것과 같습니다.
여러분, 우리 교실에 '김민수'라는 학생이 세 명 있다고 해볼까요? 제가 "민수야!"라고 부르면 세 명이 동시에 쳐다보겠죠. 이 이름이 겹치는 상태, 바로 HTML의 태그(h1, p 등)와 같습니다. 'p 태그'라고 부르면 문서의 100개 p 태그가 동시에 대답합니다. 

그래서 우리는 각 요소별로 별명을 붙여줍니다. 
먼저 **클래스(Class)** 입니다. 클래스는 '그룹 이름'입니다. 축구 구단을 생각해보세요. "맨체스터 유나이티드 유니폼을 입은 선수들, 모두 빨간색 옷을 입어라!"라고 명령할 수 있죠. HTML 안에서는 태그에 `class="highlight"`라고 적어주고, CSS에서는 그 이름 앞에 마침표(.)를 찍어 `.highlight`라고 선택합니다. 이 마침표는 "이건 태그 이름이 아니고 클래스 이름표야"라고 컴퓨터에게 친절하게 알려주는 신호입니다.

다음은 **아이디(ID)** 입니다. 이건 전 세계에서 단 하나밖에 없는 고유한 주민등록번호나 지문과 같습니다. 학번이라고 생각하셔도 됩니다. "학번이 20240001인 학생!"이라고 부르면 딱 한 명만 쳐다봐야 합니다. HTML에서는 `id="main-title"`이라고 적어주고, CSS에서는 우물 정자 혹은 해시태그 기호인 샵(#)을 써서 `#main-title`이라고 선택합니다.

*(강사 참고: 이 지점에서 클래스와 아이디를 잘못 혼용하여 코딩하는 라이브 데모를 재치있게 보여주면서 차이점을 각인시킵니다.)*

자, 보세요. 제가 명함에 있는 내 이름 부분에 `id="my-name"`을 부여하고 배경색을 노란색으로 특별하게 지정했습니다. 오직 제 이름만 밝게 빛나는 게 보이시죠? 반면 아래의 부연 설명 문구들에는 `class="card-desc"`라는 동일한 그룹 이름을 부여하고 묶어서 회색 톤으로 다운시켰습니다. 

마지막으로 핵심 요약! 점(.)이 붙으면 클래스 그룹이고 여러 번 써도 괜찮습니다. 샵(#)이 붙으면 아이디이고 오직 문서 안에 딱 한 번만 써야 합니다! 이 원리만 알면 웹사이트의 디자인을 부분부분 아주 정밀하게 조종할 수 있게 됩니다.

## Section 3: 강제 상속 및 스타일 충돌 오류의 디버깅 (25분)

마지막 시간입니다. 여러분이 조금 전 배운 내용을 바탕으로 열심히 디자인을 하다 보면, 99%의 확률로 대단히 당황스러운 순간을 맞이하게 됩니다. "어? 내가 글자를 썼는데 글자가 안 보여!" 이런 오류죠. 개발자들은 이것을 버그(Bug)라고 부르고, 이 버그를 잡는 과정을 디버깅(Debugging)이라고 합니다.

가장 흔하게 발생하는 문제를 제가 화면을 통해 보여드리겠습니다.
제가 방금 방명록 입력을 위해 검은색 우주 배경을 가진 멋진 영역을 만들었습니다. 그리고 거기에 글씨를 썼는데... 어라? 화면에 아무 글자도 보이지 않습니다. 컴퓨터가 고장 난 걸까요? 아니면 해킹? 아닙니다. 이 현상을 드래그로 한 번 긁어보죠. (강사가 마우스를 드래그하자 텍스트가 블록 지정되어 희미하게 보임) 

보이시나요? 글자는 분명히 존재합니다. 자, 문제의 원천이 무엇일까요? 바로 '배경색과 글자색이 동일'하기 때문입니다. 까만 도화지에 까만색 펜으로 글씨를 쓰면 당연히 안 보이겠죠. 너무 바보 같은 실수 아니냐고 웃을 수 있겠지만, 코드를 수백 줄 쓰다 보면 나도 모르게 배경색과 글자색이 충돌하는 일이 흔하게 발생합니다.

그런데 더 무서운 이유가 있습니다. 나는 이 글자에 까만색을 칠하라고 명령한 적이 없는데도 말이죠! 
여기서 바로 CSS의 가장 큰 특징이자 무서운 적인 **상속(Cascading/Inheritance)** 이라는 개념이 등장합니다. 

상속이라는 건 부모님이 물려주신 재산이나 유전자를 뜻하죠? CSS도 마찬가지입니다. 할아버지 태그(가장 바깥쪽 <body> 태그 같은 것)에 글자색을 파란색으로 지정해 놓으면, 그 안에 들어가 있는 부모 태그, 자식 태그, 손자 태그들이 전부 파란색 글씨라는 유전자를 물려받게 됩니다. 

내가 방명록 구역(div)의 배경을 까맣게 칠했는데, 하필 윗단 부모 태그 어딘가에서 "모든 글자는 까만색"이라고 선언되어 있다면? 자식 태그인 방명록 안의 글자들도 무조건 까만색이 되어버리며 결국 배경에 묻혀서 글씨가 증발해버린 것처럼 보이게 됩니다. 

이럴 때는 어떻게 디버깅을 해야 할까요? 어떻게 이 꼬인 족보를 풀어낼 수 있을까요? 

*(강사 참고: 학생들에게 AI를 디버깅 도우미로 활용하는 기법을 소개하고 라이브로 대화를 시연합니다.)*

이때 우리의 든든한 조수, 생성형 AI의 도움을 받을 차례입니다. 
저는 지금 ChatGPT를 열고 이렇게 물어보겠습니다.
강사 프롬프트 입력: "내가 만든 HTML 구조에서 배경이 검은색인데 텍스트가 보이지 않아. 왜 이러는 거고 어떻게 고쳐야 해? 내 코드는 다음과 같아. (코드 붙여넣기)"

자, AI의 답변을 볼까요? AI가 정확히 짚어냈네요. "현재 부모 요소의 CSS에서 글자색(color)이 검은색으로 상속되었지만, 해당 영역(invisible-bug)의 배경색도 검은색으로 지정되어 있어 글씨를 알아볼 수 없습니다. 클래스 안에 `color: white;`를 추가하여 글자색을 명시적으로 변경해 주세요." 

AI의 조언대로 코드를 한 줄 추가해 보겠습니다. `.invisible-bug { background-color: black; color: white; }`. 자, 새로고침! (화면에 글자가 하얗게 나타난다). 네, 완벽히 고쳐졌습니다.

여러분이 혼자 코딩을 하다가 이런 현상을 만나면 하루 종일 끙끙대다가 결국 포기하는 경우가 많습니다. 왜 안 되는지 도저히 이유를 알 수 없는 막막함 때문이죠. 하지만 우리는 AI라는 친절한 과외 선생님을 옆에 두고 있습니다. 여러분이 디버깅을 할 때 중요한 건 스스로 모든 CSS 구조를 완벽히 암기하는 게 아닙니다. 이상한 현상이 발생했을 때 '아, 이건 상속 때문이거나 뭔가 충돌했나 보다'라고 의심을 품고, 그것을 AI에게 논리적으로 물어본 다음, AI의 대답을 코드로 옮겨 결과를 확인하는 능력입니다.

# Week 5 Homework Peer Evaluation Rubric
# 5주차 과제 동료 평가(Peer Evaluation) 루브릭

## 1. Grading Criteria (Total 7 points)
## 1. 채점 기준 (총점 7점)

Evaluators must grade the following five items simply and clearly according to the allocated points.
*(Note: Whether the written code or design is excellent must NEVER be included in the grading criteria!)*

평가자는 아래 다섯 가지 문항에 대해 배점에 따라 단순명료하게 채점합니다.
*(주의: 작성된 코드나 디자인이 우수한가 등은 절대 채점 기준에 포함하지 않습니다!)*

| No. / 번호 | Objective Evaluation Criteria (Yes/No) / 단답형 객관적 평가 기준 | Points / 배점 |
|:---:|:---|:---:|
| **1** | **기한 내 제출 / On-time Submission**<br>과제를 기한 내에 지각하지 않고 제출하였습니까?<br>*Did they submit the assignment within the deadline?* | **5점 / 5pts** |
| **2** | Is the submitted link in the correct **GitHub URL** format?<br><br>제출된 답안의 링크가 올바른 **GitHub 링크(URL)** 형식인가? | 1 |
| **3** | Does the submitted link clearly include a unique identifier or account name? (Assignment accessibility)<br><br>제출된 링크 주소에 고유한 식별자나 계정명이 분명하게 포함되어 있는가? (과제물 접근성) | 1 |
| **4** | When the submitted link is clicked, does the page open without errors and clearly show that **CSS was successfully applied (e.g., brand colors, styling via Class/ID selectors)**?<br><br>제출된 링크를 클릭하여 열었을 때, 에러 없이 화면이 열리며 **CSS 스타일링(예: 브랜드 컬러 적용, Class/ID 선택자를 활용한 꾸미기 등)**이 정상적으로 적용되어 있는가? | 1 |
| **5** | Does the **Google Doc shortcut link** included in the submitted repository (or code) open immediately without requiring additional permissions (login, etc.), and does it contain a list of at least **5 new Windows shortcuts and 5 Chrome shortcuts** that the student did not previously know?<br><br>제출된 레포지토리(또는 코드) 내에 포함된 **단축키 구글 문서 링크(Google Doc)**가 별도의 권한 요청(로그인 등) 없이 즉시 정상적으로 열리고, **그동안 본인이 모르던 새로운 윈도우 단축키 5개 이상 및 크롬 단축키 5개 이상**이 포함된 목록이 존재하는가? | 1 |

## 2. Special Guidelines for Evaluators (Feedback Writers)
## 2. 평가자(피드백 작성자) 특별 가이드라인

- **Maintain Objectivity**: Mechanically verify ONLY whether the above 5 conditions are met.<br>**객관성 유지**: 오직 위 5가지 조건이 충족되었는지만 기계적으로 확인하십시오.
- Even if the code is short or the scale is different from your own work, if the conditions are met, you must give the **perfect score (7 points)** without deduction.<br>코드가 짧거나 본인의 결과물과 스케일이 다르더라도, 조건만 맞으면 감점 없이 **만점(7점)**을 주어야 합니다.
- If you write a feedback comment, please write light encouragement or mention the successful access rather than subjective criticism. (e.g., "All 5 conditions are well verified. The brand colors (HEX) look great! Good job!")<br>피드백 코멘트를 쓴다면 주관적 비판보다는 가벼운 격려나 성공적인 정상 접속 여부를 적어주시면 됩니다. (예: "5가지 조건 모두 잘 확인됩니다. 멋진 브랜드 컬러(HEX)가 잘 보입니다! 고생하셨습니다!")

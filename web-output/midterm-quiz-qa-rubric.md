## Moodle 퀴즈 문항 작성 필수 포맷 (핵심 요약)

모든 퀴즈 문항(`mid-eval-q.md`) 작성 시, 본 QA 루브릭을 통과하기 위해 아래 요소를 반드시 준수해야 합니다.

1. **복붙 편의성 (A-1)**: `[Question type]`을 명시하고, HTML 코드 블록은 `<pre style="..."><code>` 포맷을 사용하며, `[General feedback]`은 선택지/정답칸 위로 배치합니다. 모든 요소 사이에는 빈 줄을 넣습니다.
2. **순수 문항만 작성 (A-2)**: 출제 의도나 불필요한 힌트 없이, 오직 **학생이 풀어야 할 문제 텍스트**만 넣습니다.
3. **언어 정책 및 번역 (C-1)**: 문항 텍스트와 옵션은 **100% 영어**로 작성하되, 각 문항 맨 하단에 `[Korean]` 태그와 함께 문제 본문의 한국어 번역을 포함합니다.
4. **단답형 규칙 (C-2)**: 무들 특수 문법(Cloze)을 사용하지 않고, `정답 1:\n...` 형태의 플레인 텍스트로 작성합니다.

## 파트 A: Moodle 문항 형식 무결성 (30점)

### A-1. 시스템 태그 준수 및 복붙 편의성 (10점)
- [ ] **10점**: `[Question type]`이 본문 전에 명시되어 있고, 코드 블록이 깨지지 않도록 `<pre style="..."><code>` 이스케이프 태그를 사용했다. 또한 `[General feedback]`이 선택지/정답 상단에 위치하며, 객관식 문항에는 불필요한 `[Answer]` 태그가 없다. 모든 요소는 빈 줄로 분리되어 쉽게 더블클릭 복사가 가능하다.
- [ ] **5점**: Moodle 폼 순서가 일부 어긋났거나 코드 블록 인라인 CSS 처리가 누락된 곳이 1~2곳 존재한다.
- [ ] **0점**: Moodle 형식을 무시했거나 배치 오류가 3곳 이상 존재한다.

### A-2. 순수 문항만 포함 여부 (10점)
- [ ] **10점**: `[Question text]` 블록 내에 출제 의도, 힌트, 불필요한 배경 설명 등이 전혀 없이, 오직 학생이 풀어야 할 **순수한 문제(Question) 내용만** 깔끔하게 작성되어 있다.
- [ ] **5점**: 불필요한 출제 의도나 군더더기 설명이 1~2곳 포함되어 있다.
- [ ] **0점**: 문제 내용보다 출제 의도나 힌트 등 학생에게 불필요한 내용이 더 많아 Moodle에 올리기 부적합하다.

### A-3. 난이도 표기 (10점)
- [ ] **10점**: 모든 문항의 `[Question name]` 영역 끝에 해당 문제의 난이도가 `[Easy]`, `[Medium]`, `[Hard]` 중 하나로 명시되어 있다. (예: `StackBlitz HTML Rendering (wb02) [Hard]`)
- [ ] **5점**: 난이도 표기가 1~2개 문항에서 누락되었다.
- [ ] **0점**: 난이도 표기가 대부분 누락되었다.

## 파트 B: AI-Proof 및 행동 강제성 (Quality) (40점)

### B-1. 할루시네이션 유도 및 팩트체크 설계 (20점)
- [ ] **20점**: 문항 자체가 흔한 HTML/CSS 이론이 아니라, "StackBlitz에서 발생하는 특정 렌더링 오류" 또는 "Flexbox 속성 간의 충돌" 등 AI가 텍스트만으로는 확신하기 어려운 구체적인 버그 해결 과정을 요구한다.
- [ ] **10점**: 일부 문항은 AI에 붙여넣으면 즉시 100% 정답이 나오는 단순 암기형/지식형(예: `<h1>` 태그의 의미) 문제다.
- [ ] **0점**: 대부분의 문항이 브라우저 확인 없이 AI 질문 한 번으로 풀린다.

### B-2. 실습 도구 사용 강제 (20점)
- [ ] **20점**: 학생이 정답을 찾기 위해 반드시 StackBlitz 코드 에디터를 조작하거나, 브라우저의 Chrome 개발자 도구(DevTools)로 간격/색상을 직접 찍어봐야만 풀 수 있도록 "능동적 행동"을 강제하는 구조다.
- [ ] **10점**: 도구의 필요성을 언급하긴 했으나, 눈으로 코드를 읽는 것만으로도 충분히 풀 수 있다.
- [ ] **0점**: 코드나 도구 활용 요소가 아예 존재하지 않는다.

## 파트 C: Tone & Manner 및 내용 정확성 (Policy) (30점)

### C-1. 언어 정책 및 한국어 번역 (15점)
- [ ] **15점**: E트랙 학생 배포 문서 정책에 따라 학생 대면 텍스트는 **100% 영어**로 작성되었으며, 이와 별개로 각 문항의 맨 하단(`[Korean]` 태그)에 문제 본문 내용에 대한 정확하고 자연스러운 한국어 번역이 의무적으로 포함되어 있다.
- [ ] **7점**: 영어 텍스트에 한국어가 섞여 있거나, 하단 `[Korean]` 번역이 누락된 문항이 1~2개 존재한다.
- [ ] **0점**: 언어 정책(영어 출제)이 무시되었거나, 한국어 번역이 대부분 누락되었다.

### C-2. 피드백 충실성 및 단답형 작성 규칙 (15점)
- [ ] **15점**: `[General feedback]`에 정답의 기술적 근거가 논리적으로 잘 설명되어 있다. 단답형 문항의 경우 무들의 복잡한 특수 문법(Cloze)을 피하고, 단순하고 복사하기 쉬운 플레인 텍스트(`정답 1:\n정답값`) 형태로 명확하게 작성되었다.
- [ ] **7점**: 단답형 문항에 복잡한 특수 문법이 남아있어 복사하기 번거롭거나, 피드백 내용이 부실하다.
- [ ] **0점**: 피드백이 누락되었거나 정답 처리에 명백한 논리적 결함(잘못된 문법/오답)이 있다.

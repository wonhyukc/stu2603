# wb08 중간 평가 방안 모음 (26가지)

> **작성 목적**: 웹프로그래밍 E트랙 1~7주차를 마친 학생을 대상으로 한 중간 평가 방안을 체계적으로 정리한다. AI 활용을 허용하고 채점 자동화를 극대화하며, "제한 시간 안에 얼마나 빠르게 해낼 수 있는가"를 핵심 지표로 삼는다.

---

## 평가 설계 원칙

| 원칙 | 내용 |
|---|---|
| **AI 허용** | AI(Gemini, ChatGPT 등)는 막지 않는다. AI를 잘 활용하는 능력 자체를 평가한다. |
| **속도 우선** | 같은 정확도라면 더 빨리 완료한 학생이 높은 점수를 받는다. 제출 시각이 자동 기록된다. |
| **채점 자동화** | 강사 채점 시간을 최소화한다. **Moodle 퀴즈 자동 채점**, URL Yes/No 루브릭을 최우선 활용한다. |
| **아티팩트 기반** | 코드 구술 없이 StackBlitz 링크·GitHub 커밋·스크린샷 등 객관적 결과물로 평가한다. |
| **1문항 1조건** | 루브릭 각 항목은 하나의 물리적 조건만 검증한다. 복합 조건 금지. |

## A. Moodle 자동 채점 — 퀴즈 모듈 활용 (채점 시간 0분)

### 방안 1. MCQ 타임어택 퀴즈 (Multiple Choice)
- **Moodle 문항 유형**: 객관식 (Multiple Choice), 제한 시간 30분
- **출제 범위**: HTML 태그 구조(wb02~03), CSS 선택자(wb05), 박스 모델(wb06), Flexbox(wb07) — 총 20문항
- **예시 문항**: "아래 코드에서 `img`가 보이지 않는 원인은?" → 4지선다 (src 오류, alt 오류, type 오류, display 오류)
- **채점**: Moodle 자동 채점 + 제출 시각 자동 기록 → 동점 시 빠른 제출 우선
- **AI 활용**: 허용. 검색 속도도 능력이다.
- **강사 소요 시간**: 0분 (Moodle Gradebook 자동 집계)

---

### 방안 2. 코드 스니펫 빈칸 채우기 (Cloze)
- **Moodle 문항 유형**: Cloze (Embedded Answers)
- **형식**: `<form [[answer]] = "POST">` 형태로 HTML/CSS 코드 스니펫 제시, 빈칸에 올바른 속성값 입력
- **예시**: `display: [[flex]]; justify-content: [[center]];` — `flex`와 `center` 입력
- **채점**: Moodle 자동 채점 (정확한 문자열 매칭 + 오타 허용 범위 설정 가능)
- **강사 소요 시간**: 0분

---

### 방안 3. 이미지 기반 오류 찾기 퀴즈 (Multiple Choice with Image)
- **Moodle 문항 유형**: 객관식 + 이미지 첨부
- **형식**: 깨진 레이아웃 스크린샷을 문항에 첨부 → "어느 CSS 속성이 원인인가?" 4지선다
- **예시**: 버튼이 세로로 쌓인 스크린샷 → `flex-direction: column` vs `display: block` vs `margin: 0` vs `padding: 0` 중 선택
- **채점**: Moodle 자동 채점
- **강사 소요 시간**: 0분

---

### 방안 4. 참/거짓 판단 퀴즈 (True/False)
- **Moodle 문항 유형**: True/False, 20문항, 제한 15분
- **예시 문항**:
  - "`<img>` 태그는 닫는 태그가 필요하다" → False
  - "`display: flex`를 부모에 적용하면 자식 요소가 가로로 정렬된다" → True
  - "`padding`은 테두리 바깥의 여백이다" → False
- **채점**: Moodle 자동 채점
- **강사 소요 시간**: 0분

---

### 방안 5. 태그 매칭 퀴즈 (Matching)
- **Moodle 문항 유형**: Matching
- **형식**: 왼쪽 HTML/CSS 태그·속성 목록 ↔ 오른쪽 역할 설명 매칭
- **예시**: `border-radius` ↔ 모서리 둥글게 / `justify-content` ↔ 가로축 정렬 / `<form>` ↔ 사용자 입력 컨테이너
- **채점**: Moodle 자동 채점
- **강사 소요 시간**: 0분

---

### 방안 6. 드래그 앤 드롭 코드 정렬 (Drag and Drop)
- **Moodle 문항 유형**: Drag and Drop into Text 또는 Ordering
- **형식**: 뒤섞인 HTML 태그 블록을 올바른 중첩 순서로 재배열
- **예시**: `</body>`, `<html>`, `<head>`, `</html>`, `<body>`, `</head>` → 올바른 순서로 정렬
- **채점**: Moodle 자동 채점
- **강사 소요 시간**: 0분

## B. URL 아티팩트 제출 — 링크 열어 Yes/No 확인 (채점 ≤2분/명)

### 방안 7. StackBlitz 명함 페이지 타임어택
- **형식**: 강사가 요구사항 명세서 배포 → 학생이 StackBlitz에서 HTML/CSS 명함 완성 후 공유 링크 제출
- **요구사항 예시**: 이름 (`h1`), 이미지 (`img`), 연락처 (`p`), 방명록 폼 (`form + input + button`), 컬러 스타일링 1가지
- **루브릭 (Yes/No)**:
  - 기한 내 제출 (5점)
  - StackBlitz 링크가 로그인 없이 열림 (1점)
  - `<h1>` 태그에 이름이 표시됨 (1점)
  - `<img>` 태그 이미지가 깨지지 않고 보임 (1점)
  - `<form>` + `<input>` + `<button>` 세 요소 모두 존재 (1점) ← (참고: 3요소를 하나의 조건으로 묶지 않으려면 3개 항목으로 분리 가능)
  - CSS 스타일이 최소 1개 이상 인라인/외부 방식으로 적용됨 (1점)
- **강사 소요 시간**: ≤2분/명

---

### 방안 8. Flexbox 버그 수정 제출 (StackBlitz)
- **형식**: 의도적으로 Flexbox 속성이 잘못 작성된 코드를 StackBlitz 템플릿으로 배포 → 수정 후 링크 제출
- **오류 예시**: `display: flexbox;` (→ `flex`로 수정), `justify-content: middle;` (→ `center`로 수정)
- **루브릭**: 메뉴바가 실제로 가로로 정렬되는지 눈으로 확인 (Yes/No 2항목)
- **강사 소요 시간**: ≤1분/명

---

### 방안 9. GitHub 커밋 증적 제출
- **형식**: wb01~07 과제물이 담긴 GitHub 저장소 링크 제출 → 커밋 타임스탬프로 학습 과정 증명
- **루브릭**: 저장소 공개 여부 + 커밋 3건 이상 + 최신 커밋 날짜 확인
- **강사 소요 시간**: ≤1분/명

---

### 방안 10. AI 대화 로그 제출 (3턴 제한)
- **형식**: 강사가 제시한 CSS 버그를 AI와 3턴 이내에 해결한 대화 전체를 PDF·스크린샷으로 제출
- **채점 포인트**: 첫 번째 프롬프트에 **[현상 + 시도 + 에러 + 목표]** 4요소 포함 여부만 확인 (엔지니어식 질문법 wb01)
- **강사 소요 시간**: ≤1분/명

## C. 속도 경쟁 / 타임어택 — 제출 시각이 곧 점수

- **강사 소요 시간**: 0분

- **채점**: Yes/No 루브릭 5항목 + 제출 시각 순위 보너스 (+2/+1)

- **채점**: 핵심 키워드 포함 여부 자동 채점 + 제출 시각

- **채점**: 개발자 도구 수치 스크린샷 첨부 요구 → 강사는 수치만 확인

## D. AI 협업 품질 평가 — AI를 얼마나 잘 쓰는가

### 방안 16. 1턴 프롬프트 챌린지

### 방안 19. "AI vs Me" 에러 진단 대결
- **형식**: 동일한 HTML 에러를 학생과 AI가 각각 진단 → 학생이 AI보다 더 구체적이거나 빠르면 보너스
- **제출물**: 학생의 진단 텍스트 + AI 대화 스크린샷 (비교 가능)
- **채점**: 학생 진단이 AI 답변보다 "오류 위치를 더 정확히 지목"했으면 +1점 보너스
- **강사 소요 시간**: ≤1분/명

- **형식**: 화면 공유 상태에서 강사가 배포한 버그 코드를 StackBlitz에서 실시간으로 수정 → AI 사용 허용, 완료 시 링크 제출
- **채점**: Yes/No 5항목 루브릭 (버그 수정 완료 여부) + 완료 시각
- **강사 소요 시간**: ≤1분/명

## 루브릭 개선 메모 (wb07 대비)

- wb07 루브릭은 3문항(5+1+1)으로 구성 — 중간 평가용으로는 항목 수를 7~9문항으로 확장 권장
- Flexbox 평가 항목을 `display: flex` 존재 여부와 `정렬 방향` 두 항목으로 분리하여 **1문항 1조건 원칙** 강화 필요
- 향후 Q1 기한 내 제출 항목은 반드시 5점 유지

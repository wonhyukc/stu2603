# wb08 중간 평가 방안 모음 (20가지)



## A. Moodle 자동 채점 — 퀴즈 모듈 활용 (AI-Proof 적용)

### 방안 1. AI 코드 할루시네이션 사냥 (MCQ)
- **Moodle 문항 유형**: 객관식 (Multiple Choice)
- **출제 범위**: wb05 (CSS 속성)
- **형식**: AI가 생성한 그럴듯한 CSS 코드 스니펫을 제시하고, 브라우저에서 실제로 동작하지 않는 가짜 속성을 고르게 함.
- **AI-Proof 요점**: 직접 코드를 쳐봐야 가짜 속성을 구별할 수 있음.

### 방안 2. Chrome DevTools 수치 역추적 (Cloze)
- **Moodle 문항 유형**: Cloze (Embedded Answers)
- **출제 범위**: wb06 (박스 모델)
- **형식**: 특정 StackBlitz URL을 제공하고, 렌더링된 특정 요소의 padding 값을 빈칸으로 출제.
- **AI-Proof 요점**: 브라우저 렌더링 환경에 따라 값이 달라지므로 DevTools 조작 강제.

### 방안 3. Flexbox 레이아웃 렌더링 검증 (Matching)
- **Moodle 문항 유형**: Matching
- **출제 범위**: wb07 (Flexbox)
- **형식**: 3가지 다른 레이아웃 이미지 ↔ `justify-content` 조합 매칭.

### 방안 4. 실습(Lab) 트러블슈팅 팩트체크 (True/False)
- **Moodle 문항 유형**: True/False
- **출제 범위**: wb02~03 (HTML)
- **형식**: "StackBlitz에서 로컬 경로(C드라이브) 이미지는 다른 사람에게도 보인다." → False

## B. URL 아티팩트 제출 — 링크 열어 Yes/No 확인 (채점 ≤2분/명)

### 방안 5. StackBlitz 명함/방명록 통합 타임어택
- **형식**: 명함과 방명록이 통합된 요구사항 배포 후 링크 제출
- **루브릭**: 기한 내 제출, 로그인 없이 열림, h1/img/form 요소 존재 여부 확인

### 방안 6. Flexbox 버그 수정 제출 (StackBlitz)
- **형식**: 메뉴바가 세로로 깨진 버그 코드를 가로 정렬로 수정 후 링크 제출

### 방안 7. GitHub 커밋 증적 제출
- **형식**: 저장소 링크 제출, 커밋 3건 이상 여부 검증

## C. 속도 경쟁 / 타임어택 — 제출 시각이 곧 점수

### 방안 8. CSS 스타일링 스피드런 (StackBlitz)
- **형식**: 완성 화면 이미지와 동일하게 스타일링 후 링크 제출, Moodle 제출 시각 순위 보너스 부여.

### 방안 9. 개발자 도구 에러 해석 스피드런 (Moodle Short Answer)
- **형식**: 브라우저 콘솔 에러 스크린샷 제시 → 원인 키워드 입력

### 방안 10. 비즈니스 이메일 초안 타임어택 (Moodle Essay)
- **형식**: 강사에게 보내는 과제 제출 이메일 작성. 두괄식 및 서명 확인.

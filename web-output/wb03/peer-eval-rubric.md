# 실습물 상호평가 루브릭 (Week 3)

본 루브릭은 직접 개발한 구글 폼, 구글 시트, 파이썬 코드 기반 사설 시스템에서 진행되는 동료 상호평가(Peer Evaluation)의 객관적인 기준을 제시합니다. (eCampus 상호평가 모듈 부재에 따른 대체)

## 평가 문항 (총 3점)

| 평가 항목 | 배점 | 성공 조건 (O) | 실패 조건 (X) |
|:---|:---:|:---|:---|
| **1. 기한 내 제출** | 5점 | 기한 내에 과제를 정상 제출했다. | 마감 기한 이후에 제출되었다. |
| **2. 폼(Form) 기본 구조 완성도** | 1점 | `<form>` 태그 내부에 최소 2개 이상의 입력란(`input`, `textarea` 등)과 1개의 제출 버튼(`<button>`)이 모두 올바르게 구현되어 있다. | 폼 태그가 없거나, 입력 태그가 `<form>` 바깥에 위치하거나, 제출 버튼이 존재하지 않는다. |
| **3. 타입(type) 속성 적용의 적절성** | 1점 | 이름 텍스트, 이메일 형식 등 용도에 맞게 `<input type="...">` 속성이 알맞게 지정되어 있다. (특히 이메일 폼에 `type="email"`이 적용됨) | 모든 입력란이 `type` 구분 없이 전부 텍스트로만 구성되어 있거나, 오타가 있어 브라우저 기본 경고가 작동하지 않는다. |
| **4. 디자인 요소 및 레이아웃(줄바꿈) 점검** | 1점 | 각 입력 폼 필드가 옆으로 다닥다닥 붙어있지 않고, 보기 편하게 설명 레이블(`<label>`)과 함께 줄바꿈(`<br>` 등) 처리가 되어 구분하기 쉽다. | 레이블이 없이 입력칸만 덩그러니 있거나, 한 줄에 모든 폼이 겹칠 듯이 붙어 있어 가독성이 떨어진다. |

## 평가 방법 가이드 (학생용)

1. 배정받은 학우의 GitHub 링크를 엽니다.
2. `index.html` 내의 소스 코드를 확인합니다.
3. 3가지 평가 문항을 점검하여 한 항목당 1점씩 O/X를 판단합니다.
4. 완성된 결과물 위주로 평가하되, 본인이 보기에 참신하거나 훌륭한 부분이 있다면 구글 폼의 '자유 의견' 란에 칭찬 코멘트를 작성해 주세요. 엔지니어는 다른 사람의 코드를 읽고 건설적인 리뷰를 남기는 훈련이 매우 중요합니다.

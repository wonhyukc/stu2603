# wb03 과정 진행 가이드 (강사용)

본 문서는 **웹프로그래밍(E트랙) 3주차** 대면 실습 및 강의 진행을 위한 강사용 가이드입니다. **Flipped Learning** 기반이므로 학생들은 사전에 제공된 영상(`lecture-script.md` 기반)을 시청하고 참여함을 전제로 합니다.

---

## 1. 3주차 기본 정보

| 항목 | 내용 |
|:---|:---|
| **주차** | 3주차 |
| **주제** | 입력 폼 기초 |
| **실습 환경** | StackBlitz |
| **사전 영상** | eCampus에 업로드된 `wb03` 강의 영상 |
| **총 시간** | 75분 (25분 × 3섹션) |

## 2. 주차 학습 목표

본 주차가 끝난 후 학생이 달성해야 할 목표입니다. (마스터 교육안과 100% 일치)

1. `<form>`, `<input>`, `<button>` 속성을 이해한다.
2. [Build-up] 내 명함에 '방명록' 소통 창구 추가한다.
3. 폼 요소의 오작동 원인을 찾아 수정한다.

---

## 3. 섹션별 시간 배분 및 진행 시나리오

학생 실습 시간과 강사 이론/데모 시간은 철저히 구분되며, 아래 명세된 75분은 **순수 강사의 라이브 코딩 및 데모 시간** 기준입니다.

| 섹션 시간 | 연계 산출물 | 핵심 진행 시나리오 (라이브 코딩 명세) |
|:---|:---|:---|
| **[S1] 이론**<br>(25분) | `lecture-script.md`<br>`lecture-demo/` | **웹과 소통하기: 폼 태그의 구성**<br>- 브라우저와 사용자 간의 대화창구 역할을 하는 `<form>` 개념 설명<br>- `<input>` 태그를 종류별(`text`, `password`, `email`, `date`, `radio`, `checkbox`)로 화면에 띄우며 변화 시연<br>- `<button>`을 클릭했을 때의 기본 동작 라이브 코딩 |
| **[S2] 조작**<br>(25분) | `lecture-script.md`<br>`handout.md` | **방명록 폼 만들기 데모**<br>- 2주차에 완성한 '온라인 명함' 하단에 방문자가 메시지를 남길 수 있는 방명록 UI 부착 시연<br>- `<input>`과 `<textarea>`를 활용한 레이아웃 구성<br>- `<button type="submit">` 클릭 시 새로고침되는 "브라우저의 기본 폼 전송 방식(GET/POST 원리 개요)" 설명 |
| **[S3] AI리뷰**<br>(25분) | `lecture-script.md`<br>마이크로 과제 | **태그 디버깅 및 AI 폼 유효성 검사**<br>- 닫히지 않은 폼 태그 등 흔한 오타로 인해 레이아웃이 무너지는 상황 디버깅 데모<br>- `type="email"` 입력 폼에 일반 텍스트 입력 시 뜨는 자체 경고창 동작 원리 설명<br>- AI에게 "나만의 방명록 폼을 만들어달라"고 요청한 뒤 코드를 덧붙이고, 동작 방식에 대해 AI와 토론·비평하는 과정 시연 |

## 4. 강사용 체크리스트

### 수업 전 준비
- [ ] eCampus에 3주차 사전 학습 영상 업로드가 완료되었는지 확인
- [ ] StackBlitz 로그인 상태 및 시연용 데모 브라우저 세팅 완료
- [ ] 2주차 제출된 마이크로 과제 채점 및 피드백 현황 점검
- [ ] 시연용 온라인 명함+방명록 템플릿 코드 클립보드 패드에 준비

### 수업 중 주요 지침
- [ ] [S1] `input`의 다양한 `type` 변화를 보여줄 때 학생들이 직관적으로 흥미를 느낄 수 있도록 반응 유도.
- [ ] [S2] '새로고침' 현상이 버그가 아니라 브라우저의 정상 작동임을 명확하게 인지시키기.
- [ ] [S3] AI 코드를 무작정 복사하지 말고, 어떤 속성이 방명록 역할을 하는지 한 줄씩 읽어보는 훈련 진행.

### 수업 종료 후 마무리
- [ ] 3주차 온라인 실습 결과물 eCampus 상호평가 제출 안내.
- [ ] 이번 주차 마이크로 과제(이메일 제출) 방식 다시 한 번 강조 (예: 메일 제목 필터링 오류 최소화)

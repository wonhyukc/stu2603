4. 문서와 브라우저의 관계를 쉬운 비유로 통해 웹의 기본 원리를 이해한다.

## 시간 배분 및 진행 순서

### 이론 블록 (75분 = 25분 × 3섹션)

| 섹션 | 시간 | 주제 | 핵심 포인트 |
|:---|:---|:---|:---|
| **섹션 1** | 25분 | 웹 역사와 강의 소개 | 웹이란 무엇인가(비유), 짧은 웹 역사, 학기 로드맵, Flipped Learning |
| **섹션 2** | 25분 | 프로페셔널 커뮤니케이션 | 비즈니스 이메일 5원칙, 엔지니어식 질문법, eCampus 사용법 |

### 조작 블록 (75분)

강사는 빔 프로젝터로 화면 공유하며 학생이 직접 따라 하도록 유도한다.

1. **StackBlitz 접속 및 첫 HTML 태그 체험** (25분)
   - [stackblitz.com](https://stackblitz.com) 접속 → 새로운 Static 프로젝트 생성 → 첫 `<h1>` 태그 입력 → 실시간 라이브 프리뷰 확인.

2. **GitHub 계정 생성 및 저장소 세팅** (25분)
   - [github.com](https://github.com) 회원가입 → 비공개 저장소(`web-assignments` 등) 생성 → **강사(`wonhyukc`)를 Collaborator로 초대** (필수).

3. **eCampus 확인 + AI 계정 생성** (25분)
   - eCampus 수강 강좌 목록에 본 과목이 있는지 확인.
   - **ChatGPT 또는 Google Gemini** 계정 생성 및 로그인.
   - AI에게 "HTML이 뭐야?"라고 질문하고 답변을 비판적으로 읽기.

### AI 리뷰 및 마무리 블록 (30분)

#### 1단계: AI 답변 공유 및 비평 (15분)

2~3명을 지목해 AI 답변을 큰 소리로 읽게 하고 토의를 이끈다.

- "AI가 한 말 중에 오늘 배운 내용과 다른 부분이 있었나요?"
- "AI가 자신이 틀릴 수도 있다고 인정했나요? 어떤 식으로 표현했나요?"
- "가장 이해하기 어려웠던 부분은 어디인가요?"

**강사 포인트**: 틀린 부분을 찾아야 한다는 압박보다 "AI도 틀릴 수 있다"는 감각을 심어주는 것이 이 시간의 목적이다.

#### 2단계: 과제 안내 (10분)

- eCampus 1주차 제출함을 열어서 화면에 보여준다.
- `handout.md`에 있는 모든 과제를 간략히 설명한다:
  - **과제 A**: 강사에게 자기소개 이메일 보내기 — 비즈니스 이메일 원칙 강조.
  - **과제 B**: GitHub 저장소에 `week01.md`를 생성하고 검증 항목 3가지(StackBlitz 주소 / GitHub 프로필 주소 / AI 대화 내용) 작성 후, 해당 GitHub 파일 링크를 eCampus에 제출.
  - **과제 C**: 최초의 웹사이트(`info.cern.ch`) 및 `Web Rewind` 방문 후 느낀 점 3~4문장을 `week01.md`에 작성.
- **강조 사항**: 강사(`wonhyukc`)를 Collaborator로 초대해야 하며, 그렇지 않으면 제출물을 확인할 수 없어 **0점 처리**된다.

#### 3단계: 다음 주차 예고 및 Q&A (5분)

- 다음 주(wb02) 주제: HTML 기초 — `<h1>`, `<p>`, `<img>`를 활용한 온라인 명함 만들기.
- 다음 수업 전까지 eCampus에서 wb02 영상을 시청하도록 당부.
- Q&A 진행.

## 강사 유의 사항

- 1주차는 HTML 문법보다 **수업 방식 설명**이 우선이다. Flipped Learning과 AI 파트너 개념 정착에 집중한다.
- GitHub 인증 메일이 늦게 도착해 계정 생성이 지연될 수 있다. 오늘 안 되면 다음 주까지 완료하도록 안내한다.
- "코딩 못 하면 이 수업 어렵나요?" 질문 예상. "AI가 코드를 짜고, 여러분은 그게 맞는지 판단하는 능력을 키웁니다."라고 답한다.
- 비즈니스 이메일 과제는 단순한 첫 과제가 아니라 한 학기 내내 쓸 소통 기준이다.

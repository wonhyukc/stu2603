# py07 주차 핵심 강의 교안 (컴퓨팅적사고와코딩)

*(강사 참고: 본 교안은 K트랙 학생들을 대상으로 하며, 본 교안에 분량 제한이나 대본 룰을 적용하지 않고, 75분간 다룰 핵심 커리큘럼(파일 시스템, 폴더 네이밍, 클라우드 이전)의 상세 흐름과 체크리스트만 기록합니다.)*

---

## [강의 체크리스트 및 흐름] 

### 섹션 1: 윈도우 파일 시스템 구조와 가짜 확장자 판별 (25분)
1. **문제 제기**: 바탕화면에 무분별하게 파일을 저장하는 현재 학생들의 습관 짚어보기.
2. **개념 설명**: 
   - 파일 시스템의 트리(Tree) 구조 설명 (도서관 서가에 비유).
   - "확장자(Extension)"의 개념 설명: 해당 파일의 정체성과 연결 프로그램을 결정짓는 신분증.
3. **핵심 실습 및 시연**:
   - 윈도우 파일 탐색기 하이재킹 위험성 경고 (예: `가짜사진.jpg.exe`).
   - 파일 탐색기 `보기(View)` 메뉴에서 **'파일 확장명' 숨김 해제** 실습. 모든 학생이 설정했는지 확인.
   - 탐색기 우상단 검색창을 활용한 와일드카드 검색(`*.pdf` 등) 요령 시연.

### 섹션 2: 파일 탐색기 최적화 및 실무형 네이밍 룰 (25분)
1. **탐색기 꿀팁 전수**: 
   - 자주 사용하는 폴더를 **'즐겨찾기(Quick access)'**에 추가하는 방법(드래그 앤 드롭).
   - 바탕화면에 '바로가기(Shortcut)'를 생성하여 원본 파일 보존 시연.
2. **네이밍 룰 (Naming Rule)**:
   - 잘못된 파일명 습관 지적 (`진짜최종.docx`, `이게마지막.pptx`).
   - 실무 표준 네이밍 룰 제안: `[날짜]_[프로젝트명]_[작성자]_[버전]` (예: `20261015_파이썬실습_정원혁_v1.0`).
   - 날짜를 접두어로 사용했을 때, 이름순 정렬만으로 자연스럽게 버전 히스토리가 관리되는 이점 강조.

### 섹션 3: 클라우드 드라이브를 통한 파일 이주 시연 (25분)
1. **문제 제기**: 노트북 파손, USB 분실 등에 의한 데이터 소실 위험성. 로컬 스토리지의 한계 인지.
2. **솔루션 제공 (구글 드라이브)**:
   - 클라우드(Cloud) 스토리지 개념 설명 (서버에 데이터가 동기화되어 저장).
3. **핵심 실습 및 시연**:
   - 구글 계정 연동 후 구글 드라이브 접속 (무료 15GB).
   - 로컬 폴더(섹션 2에서 네이밍 룰로 정리한 폴더) 전체를 구글 드라이브로 드래그 앤 드롭하여 업로드하는 마이그레이션 시연.
   - 대용량 파일 전송을 위한 **'링크 공유(Share Link)'** 방식의 이점과 권한 설정 요령.

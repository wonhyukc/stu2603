## 파트 A: lecture.md (25점)

### A-1. 기본 정보 표 정확성 (5점)
- [ ] **5점**: 주차·주제·실습환경·총 시간이 교육안 기준과 모두 일치한다. (5~7주차 실습환경은 스마트폰 앱·웹 브라우저·클라우드 서비스 등 해당 주차 도구를 명시. Colab 또는 파이썬 환경 기재 시 오류.) 총 시간은 **75분 (25분 × 3섹션)**이다.
- [ ] **2점**: 1~2개 항목이 부정확하다.
- [ ] **0점**: 3개 이상 항목이 오류이거나 누락이다.

### A-2. 학습 목표 교육안 일치 및 "No Python" 검증 (12점)
- [ ] **12점**: 마스터 교육안의 해당 주차 학습 목표가 모두 명시되어 있으며, 파이썬 코드나 문법 학습을 암시하는 내용이 전혀 없다. 오직 "스마트 도구 활용 및 실생활 문제 해결"에 초점이 맞춰져 있다.
- [ ] **7점**: 학습 목표가 일부 변형되었거나 누락됨.
- [ ] **0점**: 코딩 목표가 혼입되어 있거나 교육안과 불일치함.

### A-3. 섹션별 시간 배분 표 정확성 (8점)
- [ ] **8점**: S1·S2·S3 각 25분 배정이 명시되어 있고, 각 섹션의 주제가 교육안과 일치하며 학생들에게 스마트 도구를 데모/시연하는 흐름이 자연스럽게 배분되어 있다.
- [ ] **4점**: 시간 배정은 있으나 내용 요약이 부정확하다.
- [ ] **0점**: 섹션 구분이 교육안과 다르다.

## 파트 B: slides.md (60점)

### B-1. 슬라이드 기본 구조 (10점)
- [ ] **10점**: 타이틀 슬라이드(강사명 **정원혁** 표기)·목차·섹션별 본문·마무리 슬라이드가 모두 존재한다.
- [ ] **4점**: 구조는 있으나 강사명 누락 또는 마무리 없음.
- [ ] **0점**: 최소 구조 미충족.

### B-2. S1 커버리지 (도구의 이해 및 문제 상황) (17점)
- [ ] **17점**: 교육안 S1의 내용(왜 이 도구가 필요한가, 어떤 실생활 문제를 해결하는가)을 시각적(비교표, 다이어그램)으로 잘 나타내며, 파이썬 코드 대신 소프트웨어 조작법이나 활용 예시가 누락 없이 제공된다.
- 부분 누락 시 5점 차감. 코드가 등장할 시 **-17점 (즉시 FAIL)**.

### B-3. S2 커버리지 (도구 핵심 원리 및 심화 사용) (17점)
- [ ] **17점**: 해당 주차 S2의 핵심 조작 원리(예: 구글 검색 연산자, PDF 분리·병합 방식, 구글 폼 클리닝 원리 등)를 화면 캡처나 조작 순서도(Flow)로 명확히 보여준다.
- 부분 누락 시 5점 차감. 코드가 등장할 시 **-17점 (즉시 FAIL)**.

### B-4. S3 커버리지 (라이브 데모 및 과제 연결) (16점)
- [ ] **16점**: 강사가 현장에서 직접 도구(앱/웹)를 조작하여 최종 결과물(예: 깨끗한 엑셀 파일, 병합된 PDF)을 도출하는 시연 스텝이 슬라이드에 명시되어 있다.
- 부분 누락 시 5점 차감. 코드가 등장할 시 **-16점 (즉시 FAIL)**.

> 5~7주차는 파이썬 코드가 없으므로 `lecture-demo.ipynb` 대신 **`lecture-demo/` 폴더** 형태로 제공한다. 폴더 내에는 강사가 현장 시연 시 사용할 단계별 가이드(마크다운 또는 스크린샷 이미지 포함)가 담겨 있어야 한다.

### C-1. 폴더 존재 및 파일 완결성 (5점)
- [ ] **5점**: `lecture-demo/` 폴더가 존재하며 내부에 1개 이상의 파일(`.md`, `.png`, `.jpg` 등)이 있다.
- [ ] **0점**: 폴더가 없거나 비어 있다.

### C-2. lecture-script.md 시연 단계와의 연관성 (10점)
- [ ] **10점**: `lecture-script.md`의 S1~S3 라이브 시연 단계가 demo 폴더 내 자료에 반영되어 있다. 강사가 demo 폴더만 보고 현장 시연을 재현할 수 있을 만큼 구체적이다.
- [ ] **5점**: 일부 섹션만 반영되어 있거나 내용이 추상적이다.
- [ ] **0점**: 강의 내용과 무관하거나 자료가 불충분하다.

# 공통 1단계 QA 루브릭 (K트랙 5~7주차 전용: Pre-Midterm)

> **🚨 [필수 원칙: 문항 단일 조건 원칙 (1문항 1점, 1조건)]**
> 동료 평가(peer-eval-rubric) 등 어떠한 평가 문항을 생성/검증할 때라도 **모든 문항은 항상 1점의 배점으로 '예/아니오'만 명확히 판별할 수 있는 단 1개의 조건**만 포함해야 합니다. A와 B를 동시에 묻는(And 조건) 문항은 절대 금지하며 반드시 2개의 별도 문항으로 분리하세요. (반만 참이거나 부분 점수가 발생하는 상황을 원천 차단할 것)
> 
> **🚨 [필수 원칙: 실습 지시-평가 문항 1:1 완벽 대응 (무결성 원칙)]**
> `handout.md`에 기재된 실습 파이프라인(틀 고정, 정렬, 색상 서식 등 세부 조작 포함)의 모든 단계는 **단 하나도 빠짐없이 `peer-eval-rubric.md`의 평가 문항과 1:1로 매칭**되어야 합니다. 아울러 평가 문항은 평가자가 구글 시트 등에서 '보기 읽기 권한'만 가진 상태라도 설정 창을 열 필요 없이 직관적인 눈(시각적)으로 즉시 채점할 수 있는 기준으로 작성되어야 합니다.

**평가 대상 파일** (특정 주차 폴더 `{week}/` 하위)
- `lecture.md` — 필수
- `slides.md` — 필수
- `lecture-demo/` 폴더 — 필수

> **1단계 목적**: 5~7주차(파이썬 코딩 배제 기간)의 강의 계획, 슬라이드 구성, 강사 데모 자료가 교육안과 일치하는지 검증한다. PASS 후 2단계 작업을 시작한다.
> **적용 범위**: 5주차·6주차·7주차만 해당. 8주차(중간고사 대체)는 본 루브릭 적용 제외.

**평가 기준 SSOT**
- `python-output/python-curriculum.md` — 해당 주차 학습 목표 및 S1·S2·S3 구성
- `CLAUDE.md` — 품질 기준 (이 기간 동안 **파이썬 문법 및 코드 작성은 0%**이어야 함)

**만점**: 100점
**PASS 기준**: 85점 이상 AND 파트 A ≥ 18점 AND 파트 B ≥ 48점 AND 파트 C ≥ 12점
**FAIL 시**: 보완 항목 목록 출력 후 Auto-Fix

## 파트 A: lecture.md (25점)

### A-1. 기본 정보 표 정확성 (5점)
- [ ] **5점**: 주차·주제·실습환경·총 시간이 교육안 기준과 모두 일치한다. (5~7주차 실습환경은 스마트폰 앱·웹 브라우저·클라우드 서비스 등 해당 주차 도구를 명시. Colab 또는 파이썬 환경 기재 시 오류.) 총 시간은 **75분 (25분 × 3섹션)**이다.
- [ ] **2점**: 1~2개 항목이 부정확하다.
- [ ] **0점**: 3개 이상 항목이 오류이거나 누락이다.

### A-2. 교육 목표 최상단 명시 및 "No Python" 검증 (12점)
- [ ] **12점**: `python-curriculum.md`에 기재된 해당 주차의 핵심 교육 내용과 목표가 `lecture.md` 파일 **제일 윗부분(최상단)**에 그대로 복사 또는 명확히 요약되어 명시되어 있으며, 파이썬 코드나 문법 학습을 암시하는 내용이 전혀 없다. 오직 "스마트 도구 활용 및 실생활 문제 해결"에 초점이 맞춰져 있다.
- [ ] **7점**: 학습 목표가 최상단이 아닌 다른 곳에 있거나 일부 변형/누락됨.
- [ ] **0점**: 코딩 목표가 혼입되어 있거나 교육안과 불일치함.

### A-3. 실전 강의 시나리오의 스크립트급 구체성 (8점)
- [ ] **8점**: 파이썬 트랙에서는 별도의 `lecture-script.md`를 작성하지 않으므로, 강사가 현장에서 진행할 모든 시나리오가 S1·S2·S3 각 25분 배정에 맞춰 **스크립트에 준하는 수준으로 매우 상세하게 항목별(Bullet point)로 기술**되어 있다. (구어체 대본은 제외하되, 강사의 설명 요지, 시연 스텝, 질문 등 모든 액션 플랜이 포함됨)
- [ ] **4점**: 시간 배정과 내용 요약은 있으나, 항목별 구체성이 떨어져 스크립트를 대체하기 어렵다.
- [ ] **0점**: 구체적인 시나리오 내용이 부실하거나 기존 교육안 복붙 수준이다.

## 파트 B: slides.md (60점)

### B-1. 슬라이드 기본 구조 (10점)
- [ ] **10점**: 타이틀 슬라이드(강사명 **정원혁** 표기)·목차·섹션별 본문·마무리 슬라이드가 모두 존재한다.
- [ ] **4점**: 구조는 있으나 강사명 누락 또는 마무리 없음.
- [ ] **0점**: 최소 구조 미충족.

### B-2. S1 커버리지 (도구의 이해 및 문제 상황) (17점)
- [ ] **17점**: 교육안 S1의 내용(왜 이 도구가 필요한가, 어떤 실생활 문제를 해결하는가)을 시각적(비교표, 다이어그램)으로 잘 나타내며, 파이썬 코드 대신 소프트웨어 조작법이나 활용 예시가 누락 없이 제공된다.
- 부분 누락 시 5점 차감. 코드가 등장할 시 **-17점 (즉시 FAIL)**.

### B-3. S2 커버리지 (도구 핵심 원리 및 심화 사용) (17점)
- [ ] **17점**: 해당 주차 S2의 핵심 조작 원리(예: 구글 검색 연산자, PDF 분리·병합 방식, 구글 폼 클리닝 원리 등)를 화면 캡처나 조작 순서도(Flow)로 명확히 보여준다.
- 부분 누락 시 5점 차감. 코드가 등장할 시 **-17점 (즉시 FAIL)**.

### B-4. S3 커버리지 (라이브 데모 및 과제 연결) (16점)
- [ ] **16점**: 강사가 현장에서 직접 도구(앱/웹)를 조작하여 최종 결과물(예: 깨끗한 엑셀 파일, 병합된 PDF)을 도출하는 시연 스텝이 슬라이드에 명시되어 있다.
- 부분 누락 시 5점 차감. 코드가 등장할 시 **-16점 (즉시 FAIL)**.

## 파트 C: lecture-demo/ 폴더 (15점)

> 5~7주차는 파이썬 코드가 없으므로 `lecture-demo.ipynb` 대신 **`lecture-demo/` 폴더** 형태로 제공한다. 폴더 내에는 강사가 현장 시연 시 사용할 단계별 가이드(마크다운 또는 스크린샷 이미지 포함)가 담겨 있어야 한다.

### C-1. 폴더 존재 및 파일 완결성 (5점)
- [ ] **5점**: `lecture-demo/` 폴더가 존재하며 내부에 1개 이상의 파일(`.md`, `.png`, `.jpg` 등)이 있다.
- [ ] **0점**: 폴더가 없거나 비어 있다.

### C-2. lecture-script.md 시연 단계와의 연관성 (10점)
- [ ] **10점**: `lecture-script.md`의 S1~S3 라이브 시연 단계가 demo 폴더 내 자료에 반영되어 있다. 강사가 demo 폴더만 보고 현장 시연을 재현할 수 있을 만큼 구체적이다.
- [ ] **5점**: 일부 섹션만 반영되어 있거나 내용이 추상적이다.
- [ ] **0점**: 강의 내용과 무관하거나 자료가 불충분하다.

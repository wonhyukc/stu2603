## 영역 4. Python 과정 내용의 충실성 및 난이도 분배 (45점)

**이 항목은 본 과정이 Python 프로그래밍 교육으로서 충분한 질과 양을 담보하고 있는지를 집중 평가합니다.**

### 4-1. Python 핵심 내용의 충분성 (빈약함 방지) (15점)
각 주차에서 다루는 내용이 겉핥기 식이 아니라, 대학 수준 파이썬 기초 과정에 걸맞은 충분한 문법적/실용적 깊이를 가지는지 평가합니다.
- [ ] **15점**: 모든 주차가 구체적인 문법 요소, 활용 사례 체력을 갖추고 있으며 내용이 빈약하거나 모호한 주차가 없다.
- [ ] **10점**: 1~2개 주차의 내용이 지나치게 가볍거나 빈약하다.
- [ ] **5점**: 3~4개 주차의 내용이 빈약하며, 과정 전반의 깊이가 부족하다.
- [ ] **0점**: 전체적으로 단순 나열에 그치며 내용이 매우 부실하다.

### 4-2. 주차별 학습 범위(Scope)의 적절한 분배 (10점)
특정 주차에 내용이 과도하게 몰려 있거나 너무 적지 않은지, 주차별 분량이 균형감 있게 설계되었는지 평가합니다.
- [ ] **10점**: 14개 주차 전반에 걸쳐 파이썬 학습 범위와 분량이 고르게 분산되어 있다. (예: 제어문에 너무 많은 시간을 뺏겨 자료구조 파트가 지나치게 압축되는 현상이 없음)
- [ ] **7점**: 특정 1~2개 주차에 내용이 과적되어 있거나 과소평가되어 있다.
- [ ] **3점**: 3~4개 주차의 분량 불균형이 심각하다.
- [ ] **0점**: 전체적인 커리큘럼의 범위 배분이 완전히 무너져 있다.

### 4-3. 난이도의 적절성 (10점)
비전공자/입문자 관점에서 기초부터 응용까지 난이도가 적절히 설정되었으며, 급격한 난이도 상승(갑작스런 고급 기능 등장)이 방지되었는지 평가합니다.
- [ ] **10점**: 학습 곡선(Learning Curve)이 완만하며, 9주차 이후(로컬 환경)의 실전 스크립트 작성까지 초보자도 무리 없이 이어지도록 난이도가 면밀히 설계되었다.
- [ ] **7점**: 특정 구간에서 난이도 점프(갑작스런 고난도 요구)가 1~2곳 존재한다.
- [ ] **3점**: 난이도 조절이 실패하여 초보자가 따라가기 벅찬 수준의 주차가 3개 이상 있다.
- [ ] **0점**: 난이도 설계가 입문자를 전혀 고려하지 않았다.

### 4-4. 강의 구성(S1·S2·S3)과 실전 연계 (10점)
3섹션 구성이 단순히 이론을 가르는 용도가 아니라, 강사 데모/라이브 코딩 등을 통해 실제 파이썬 활용 능력을 기를 수 있도록 설계되었는지 평가합니다.
- [ ] **10점**: 모든 주차에 S1(개념) → S2(심화) → S3(실전/라이브 코딩) 흐름이 명확히 있고, 코딩 데모 내용이 탄탄하다.
- [ ] **7점**: 1~2개 주차에서 섹션 구분이나 강사 라이브 코딩 내용이 부실하거나 누락되었다.
- [ ] **3점**: 3개 이상 주차에서 누락되었거나 다분히 이론 중심(실전 배제)으로 구성되었다.
- [ ] **0점**: 3섹션 구조 자체가 지켜지지 않았다.

## 영역 5. email로 제출 과제 설계 (10점)

### 5-1. 과제의 주차 주제 연관성 (5점)

- [ ] **3점**: 1~2개 과제가 지나치게 추상적이거나 범위 이탈.

- [ ] **5점**: 모든 주차 과제 형식이 명확하고, 전체 실습 문제 중 상당수(70% 이상)가 AI 프롬프트/비평 단계를 포함한다.
- [ ] **3점**: AI 활용 과제가 50~69% 수준이거나 일부 과제 형식이 깨짐.
- [ ] **0점**: AI 활용 과제가 50% 미만이다.

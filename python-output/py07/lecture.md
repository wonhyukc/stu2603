## [강의 체크리스트 및 상세 흐름] 

### 섹션 1: 윈도우 파일 시스템 구조와 가짜 확장자 판별 (25분)

**1. 문제 제기 및 공감대 형성: "바탕화면의 저주"**
* 현재 대부분의 초보 학생이나 신입생들이 범하는 가장 큰 실수인 '모든 파일을 바탕화면에 무분별하게 저장하는 습관'을 지적합니다.
* 바탕화면이 꽉 차서 더 이상 아이콘이 들어갈 자리가 없는 상황을 묘사하고, 직관적으로 보이지만 장기적으로는 파일을 찾기 위해 엄청난 시간을 낭비하게 만드는 비효율의 근원임을 강조합니다.

**2. 핵심 개념 1: 파일 시스템의 트리(Tree) 구조**
* 컴퓨터의 저장 공간을 거대한 **도서관 서가**에 비유하여 설명합니다. (도서관 전체 = C 드라이브, 과학/인문 등 분류 = 상위 폴더, 개별 책 = 파일)
* 윈도우 운영체제의 근간인 **파일 탐색기(File Explorer)**의 계층적 논리 구조(내 PC > C: > Users > 내 권한 폴더 > 문서/바탕화면 등)를 화이트보드나 슬라이드 등에 그리며 시각적으로 각인시킵니다.

**3. 핵심 개념 2: 확장자(Extension)의 정체와 중요성**
* 확장자란 파일명 뒤에 붙는 점(.) 이후의 영문자 3~4자리로, 해당 파일의 **정체성(신분증)**이자 어떤 응용 프로그램과 연결되어야 하는지 결정하는 핵심 요소임을 설명합니다.
* 대표적인 확장자 예시 분류:
  - 텍스트/문서: `.txt`, `.docx`, `.pdf`
  - 미디어: `.jpg`, `.png`, `.mp4`, `.mp3`
  - 실행/시스템: `.exe`, `.bat`, `.zip`

**4. 상세 실습 및 시연 가이드: 가짜 확장자 하이재킹 경고**
* **시나리오 시연**: 윈도우 기본 설정은 친절함을 명목으로 확장자를 숨깁니다. 누군가 악성코드 파일명을 `가짜사진.jpg.exe`로 만들었을 때, 숨김 상태에서는 `가짜사진.jpg`로만 보여 사용자가 무심코 클릭하게 되는 **랜섬웨어 감염/해킹 시나리오**를 강하게 경고합니다.
* **실습 체크리스트**:
  - [ ] 강사가 먼저 탐색기 상단의 `보기(View)` 메뉴 (또는 윈도우11의 경우 `보기 > 표시`)로 들어가 **'파일 확장명' 체크박스**를 켜는 시연을 천천히 보여줍니다.
  - [ ] 학생 전원이 자신의 랩탑에서 파일 확장명 숨김 옵션을 해제했는지 조교와 함께 순회하며 확인합니다.
  - [ ] 탐색기 우상단 검색창을 활용하여 와일드카드(`*`) 기반 검색 요령을 시연합니다. (예: 내 컴퓨터 안의 모든 PDF 찾기 `*.pdf`, 파일명에 특정 단어가 들어간 엑셀 찾기 `*과제*.xlsx` 등).

**5. 윈도우(Windows) 필수 생존 기술 시연**
* 학생들의 개발 및 작업 생산성을 극적으로 높여줄 컴퓨터 활용 기능 시연:
  - **클립보드 히스토리 (Win + V)**: 복사(Ctrl+C)한 내역을 여러 개 저장해두고 골라서 붙여넣는 기능입니다. 작업 속도를 극적으로 높여줍니다.
  - **화면 캡처 마스터 (Win + Shift + S)**: 전체 화면 캡처 후 그림판에 붙여넣는 구시대적 방식 대신, 원하는 영역만 깔끔하게 잘라내어 바로 메일이나 문서에 붙여넣는 방법입니다.
  - **가상 데스크톱 및 창 분할 (Win + Tab / Win + 방향키)**: 한 화면에서 문서와 브라우저를 정확히 반반씩 나누어 쓰거나, 가상 모니터링 화면을 만들어 작업 공간을 넓게 쓰는 방법입니다.
  - **작업 관리자 (Ctrl + Shift + Esc)**: 프로그램이 멈췄을 때 전원 버튼을 누르지 않고, 문제가 발생한 프로세스만 찾아 강제 종료(Kill)하는 대처법입니다.
  - **압축 해제의 개념 (Zip)**: 압축을 풀지 않고 폴더 안으로 들어가서 실행 파일(.exe)이나 코드를 실행하면 왜 오류가 나는지, 그 원리를 설명합니다.

### 섹션 2: 글로벌 IT 기업의 철학, 폴더보다 강력한 네이밍 룰 (25분)

**1. 문제 제기: 무의미한 폴더 계층이 낳는 시간 낭비**
* 특정 파일 하나를 찾기 위해 `[C드라이브 > 문서 > 2026년 > 1학기 > 컴사코 > 조별과제]` 식으로 5단계, 6단계씩 폴더를 더블클릭해서 내려가는 구시대적 습관을 꼬집습니다.
* 폴더를 깊게 파면 팔수록 오히려 본인조차 그 파일이 어디 있는지 잊어버리게 되는 '데이터 사일로(Data Silo)' 현상을 지적합니다. 

**2. IT 리딩 기업의 핵심 철학 전수: "분류하지 말고 검색하라 (Search, Don't Sort)"**
* 구글(Google)이 Gmail과 구글 드라이브를 설계할 때 적용한 핵심 원칙을 학생들에게 설명합니다. 굳이 무수한 폴더 계층(Depth)을 설계하며 스트레스 받을 필요 없이, 단일 공간(Flat Directory)에 파일을 두더라도 **"완벽한 파일명(문맥)"**과 **"검색(Search)"**을 통해 1초 만에 불러오는 현대적인 파일 관리 철학을 주입합니다.
* 시스템의 검색창(Spotlight, 윈도우 검색)을 100% 신뢰하고, 물리적 방 배치(폴더)보다 식별자(이름표)에 모든 에너지를 쏟는 것이 속도의 핵심임을 강조합니다.

**3. 탐색기 접근성을 높이는 즐겨찾기와 바로가기**
* 현재 가장 집중해서 작업하는 단일 핵심 폴더 하나만 탐색기 왼쪽 패널의 **'즐겨찾기(Quick Access)'**에 드래그하여 즉시 공간 이동을 하는 요령을 시연합니다.
* 바탕화면 공간 확보를 위해 무거운 원본 파일을 옮기는 대신 1KB짜리 **'바탕화면 바로가기(Shortcut)'** 표지판만 생성하여, 파일 중복 분산을 막고 검색 엔진의 혼동을 막는 원리를 설명합니다.

**4. 실무 집중 훈련: 파일명 Naming Rule의 압도적 위력**
* **나쁜 예시 공유**: `진짜최종.docx`, `이게마지막_진짜.pptx`, `진짜마지막의_마지막.hwp` 등 나중에 검색할 수도, 버전도 알 수 없는 학생들의 전형적인 실수 패턴을 공유하고 공감대를 이끌어냅니다.
* **강력한 실무 데이터 파티셔닝 네이밍 룰 (ISO 8601 기반)**:
  - 규칙: `[날짜 연월일 8자리]_[프로젝트명/과목명]_[작성자명/학번]_[버전/세부내용]`
  - 좋은 예시: `20261015_파이썬실습_정원혁_v1.0.docx`
  - 문서를 수정한 후: `20261016_파이썬실습_정원혁_v1.1.docx`
* **시연 포인트 (정렬의 마법과 버전 관리)**:
  - 위처럼 네이밍을 한 뒤 강사가 탐색기에서 **'이름순 정렬'** 필터를 누릅니다.
  - 별도의 "초안 폴더", "완성본 폴더"를 만들 필요 없이, 앞에 적힌 날짜 숫자 덕분에 컴퓨터가 파일들을 생성된 시간 흐름(Chronological) 순으로 완벽하게 줄 세우는 시각적 카타르시스를 시연합니다.
  - 띄어쓰기 대신 언더바(`_`)나 하이픈(`-`)을 사용하여 검색엔진의 색인(Index) 인식 한계를 돕고 오류를 원천 차단하는 팁을 추가로 강의합니다.
* **[개념 확장] Git과 구글 워크스페이스(Google Workspace)**:
  - 강사는 이 맥락에서 **"이러한 끝없는 수동 버전 관리의 지옥을 해결하기 위해 개발자들이 쓰는 도구가 바로 우리가 배우는 Git(깃)"**임을 강조합니다. 
  - 나아가 구글 문서, 시트 등 구글 웍스(Google Workspace) 내부에도 이러한 Git 방식의 버전 컨트롤(버전 기록) 시스템이 도입되어 있음을 설명하며, 로컬에 '진짜최종.docx'를 수십 개씩 쌓아두는 대신 클라우드에서 문서의 과거 히스토리를 언제든 되돌릴 수 있는 원리를 연결합니다.

### 섹션 3: 안전한 피난처, 클라우드 드라이브를 통한 생태계 마이그레이션 (25분)

**1. 문제 제기: 로컬 스토리지의 치명적 한계**
* 과제를 거의 다 마쳤을 때 노트북에 물을 쏟아 하드디스크가 망가진 상황, 혹은 발표 5분 전에 USB 메모리를 식당에 두고 온 아찔한 상황을 상기시킵니다.
* 우리가 아무리 섹션 2에서 네이밍 룰을 완벽히 지켜 로컬에 저장했더라도, 기기가 파괴되거나 분실되면 데이터는 영구적으로 소실됨을 강조합니다.

**2. 핵심 개념: 클라우드(Cloud) 스토리지 시스템의 이해**
* 물리적인 내 눈 앞의 플라스틱 기계(USB)가 아니라, 구글 등 대형 빅테크 기업의 튼튼한 서버 가상 공간("구름")에 내 자료를 백업하고 동기화하는 개념을 설명합니다.
* 장점 요약: 기기 파손/분실 대비 완벽한 백업, 집과 도서관 PC의 완벽한 연속성, 스마트폰 등 타 디바이스와의 실시간 연동.

**3. 구체적 실습 및 마이그레이션 시연 가이드**
* 구글 계정으로 웹 브라우저를 통해 **구글 드라이브(Google Drive)**에 접속하는 과정을 보여주며 기본 제공되는 15GB의 공간을 확인합니다.
* **드래그 앤 드롭 마이그레이션**:
  - 섹션 2에서 네이밍 룰을 적용하여 예쁘게 만들어 놓은 로컬 폴더 전체를 선택합니다.
  - 로컬 탐색기에서 구글 드라이브 웹 화면으로 그대로 마우스를 끌어다 놓아 통째로 업로드하는 시연을 보여줍니다 (이때 폴더 구조가 훼손되지 않고 그대로 올라가는 것을 확인).
* **협업을 위한 보안 링크 공유 (Share Link)**:
  - 지메일(Gmail)에 10MB 이상의 무거운 엑셀/영상 원본 파일을 직접 첨부하다가 전송 지연이나 용량 제한 에러를 겪는 비효율적인 상황을 방지합니다.
  - 업로드된 파일이나 폴더에 마우스 우클릭을 하여 **'링크 복사(Get link)'**를 선택합니다.
  - 뷰어(Viewer)/편집자(Editor) 권한 설정의 차이를 꼼꼼하게 설명하고, 이 가벼운 URL 텍스트 한 줄만 메신저로 전달하면 대용량 파일도 즉시 공유할 수 있는 실무 협업 기술을 최종 실습합니다.

# py07 주차 학생 배포 자료 (handout)

## 1. 개념 참고 영역

이번 주차는 코딩 문법 대신 여러분의 평생 작업 속도와 데이터 안전을 결정짓는 **'윈도우 파일 시스템 통제'** 및 **'클라우드 백업'** 기술을 익힙니다.

### 핵심 개념 요약
*   **바탕화면의 저주**: 모든 파일을 바탕화면에 저장하는 것은 결국 파일을 찾기 어렵게 만들고 컴퓨터를 느려지게 하는 최악의 습관입니다.
*   **파일 시스템의 트리(Tree) 구조**: 내 PC > C:드라이브 > 폴더 > 파일로 이어지는 컴퓨터 내부의 논리적 주소 체계입니다.
*   **확장자(Extension)**: `.pdf`, `.docx`, `.jpg`, `.exe` 등 파일명 끝에 붙는 3~4자리 글자로, 파일의 종류(신분증)를 의미합니다. 숨겨진 확장자를 항상 보이게 설정하여 악성코드 위장 파일(`가짜.jpg.exe`)에 속지 않아야 합니다.
*   **즐겨찾기와 바로가기**: 무거운 원본 파일을 복사하는 대신, 파일 탐색기의 '즐겨찾기'에 등록하거나 1KB짜리 '바로가기'를 생성하여 작업 속도를 극대화합니다.
*   **분류하지 말고 검색하라 (Search, Don't Sort)**: 무수히 많은 폴더 계층을 만드는 대신, 명확한 네이밍(파일명)과 운영체제의 '검색' 기능을 활용해 문서를 즉시 찾아야 합니다.
*   **ISO 8601 기반 네이밍 룰**: `20261015_과목명_이름_v1.0.docx`와 같이 날짜(숫자 8자리)를 맨 앞에 두면 컴퓨터가 시간순으로 완벽하게 정렬(Chronological order)해줍니다.
*   **클라우드(Cloud)**: 물리적인 USB나 로컬 하드디스크가 아닌, 구글(Google Drive) 등 대형 기업의 안전한 서버에 데이터를 보관하고 실시간으로 동기화하는 시스템입니다.
*   **윈도우(Windows) 필수 생존 기술**: 
    *   **클립보드 히스토리 (Win + V)**: 복사(Ctrl+C)한 내역을 여러 개 저장해두고 골라서 붙여넣는 기능입니다. 작업 속도를 극적으로 높여줍니다.
    *   **화면 캡처 마스터 (Win + Shift + S)**: 전체 화면 캡처 후 그림판에 붙여넣는 구시대적 방식 대신, 원하는 영역만 깔끔하게 잘라내어 바로 메일이나 문서에 붙여넣는 방법입니다.
    *   **가상 데스크톱 및 창 분할 (Win + Tab / Win + 방향키)**: 한 화면에서 문서와 브라우저를 정확히 반반씩 나누어 쓰거나, 가상 모니터링 화면을 만들어 작업 공간을 넓게 쓰는 방법입니다.
    *   **작업 관리자 (Ctrl + Shift + Esc)**: 프로그램이 멈췄을 때 전원 버튼을 누르지 않고, 문제가 발생한 프로세스만 찾아 강제 종료(Kill)하는 대처법입니다.
    *   **압축 해제의 개념 (Zip)**: 압축을 풀지 않고 폴더 안으로 들어가서 실행 파일(.exe)이나 코드를 실행하면 왜 오류가 나는지, 그 원리를 알아야 합니다.

## 2. 실습 과제 (Assignment)

이번 주는 파이썬 코딩 배제 기간(5~7주차)이므로 Google Colab 실습 파일(lab.ipynb)이 제공되지 않습니다. 
대신 본인의 랩탑(윈도우/맥)에서 직접 다음 활동을 실습해 보시기 바랍니다.

*   **실습 안내**: 
    1. 학교 실습실 컴퓨터(또는 본인 랩탑)를 켜고, 파일 탐색기에서 [보기] 탭을 눌러 **'파일 확장명' 표시**를 켭니다.
    2. **실습용 파일 준비**: 바탕화면이나 다운로드 폴더에 자신의 파일이 있다면 이를 활용합니다. 만약 파일이 없거나 타인의 파일만 있다면, 자신의 구글 드라이브로 가서 그동안 공유된 실습 자료, 핸드아웃 등을 모두 다운로드 받습니다. 혹은 인터넷 구글 검색창에 `컴퓨터 활용 filetype:pdf` 등으로 검색하여 파일을 여러 개 다운로드합니다. 이 방법을 모른다면 제미니(Gemini)에게 물어서 방법을 알아내 진행하세요.
    3. 이렇게 모인 10개 이상의 파일들을 대상으로 수업에서 언급된 ISO 8601 기반 **네이밍 룰**을 적용하여 종류별, 날짜순으로 일괄 수정하고 깔끔하게 정렬합니다. 이 정리된 상태를 화면 캡처(스크린샷)하여 증명합니다.
    4. 정리한 파일 중 적어도 **10개 이상**의 파일을 여러분의 **Google Drive**로 드래그하여 업로드(백업)합니다. 성공적으로 업로드된 구글 드라이브 화면을 스크린샷으로 찍어서 증명합니다.
    5. 준비된 2장의 스크린샷(파일 정리 화면, 구글 드라이브 업로드 화면)을 이번 주차 **과제 제출 구글 폼**을 통해 제출합니다. **(주의: 실습이 끝난 후 공용 컴퓨터에 남은 파일은 모두 스스로 삭제하시기 바랍니다.)**
*   **구글 시트 과제 숙지 확인(Ping)**: 실습 내용을 모두 확인했다면 반드시 담당 구글 시트에 접속하여 본인 이름 옆의 '과제 숙지 확인(Ping)' 란에 '1'을 입력하여 제출을 완료해 주세요.

# py07 주차 학생 배포 자료 (handout)

## 1. 개념 참고 영역

이번 주차는 코딩 문법 대신 여러분의 평생 작업 속도와 데이터 안전을 결정짓는 **'윈도우 파일 시스템 통제'** 및 **'클라우드 백업'** 기술을 익힙니다. 아래의 세 가지 핵심 섹션 내용을 꼼꼼히 읽고 본인의 작업 습관을 점검해 보세요.

### 섹션 1: 윈도우 파일 시스템 구조와 가짜 확장자 판별

1.  **바탕화면의 저주**:
    모든 파일을 바탕화면에 무분별하게 저장하는 것은 결국 파일을 찾기 어렵게 만들고 컴퓨터를 느려지게 하는 최악의 습관입니다.
2.  **파일 시스템의 트리(Tree) 구조**:
    컴퓨터의 저장 공간은 거대한 도서관 서가와 같습니다. 윈도우 운영체제의 파일 탐색기(내 PC > C:드라이브 > 폴더 > 파일)는 논리적인 주소 체계입니다.
3.  **확장자(Extension)의 중요성**:
    파일명 뒤에 붙는 점(.) 이후의 3~4자리 영문자(`.pdf`, `.docx`, `.jpg`, `.exe` 등)로, 파일의 종류(신분증)이자 연결될 프로그램을 결정합니다. 기본적으로 윈도우는 확장자를 숨기는데, 이는 `가짜사진.jpg.exe` 등 악성코드 위장 파일에 속을 위험을 높이므로 항상 보이게 설정해야 합니다.
4.  **윈도우(Windows) 필수 생존 기술**:
    *   **클립보드 히스토리 (Win + V)**: 복사(Ctrl+C)한 내역을 여러 개 저장해두고 골라서 붙여넣어 작업 속도를 배가시킵니다.
    *   **화면 캡처 마스터 (Win + Shift + S)**: 전체 화면 캡처 방식 대신, 원하는 영역만 깔끔하게 잘라 바로 문서나 메일에 붙여넣습니다.
    *   **가상 데스크톱 및 창 분할 (Win + Tab / Win + 방향키)**: 한 화면에서 문서와 브라우저를 정확히 반반씩 나누어 쓰거나, 가상 모니터링 화면을 만들어 작업 공간을 넓힙니다.
    *   **작업 관리자 (Ctrl + Shift + Esc)**: 프로그램이 멈췄을 때 전원 버튼을 누르는 대신, 문제가 발생한 프로세스(프로그램)만 찾아 강제 종료(Kill)합니다.
    *   **압축 해제의 개념 (Zip)**: 압축을 풀지 않고 폴더 안에서 실행 파일(.exe) 등 내부 파일을 실행하면 왜 오류가 나는지 그 원리를 이해해야 합니다.

### 섹션 2: 폴더보다 강력한 네이밍 룰

1.  **분류하지 말고 검색하라 (Search, Don't Sort)**:
    파일 하나를 찾기 위해 깊은 폴더 계층을 탐색하는 것은 큰 시간 낭비입니다. '진짜최종.docx'와 같은 의미 없는 파일명 대신 명확한 문맥이 담긴 이름으로 저장하고, 폴더 분류보다는 윈도우의 강력한 '검색' 기능을 활용해 1초 만에 불러오는 것이 현대적인 파일 관리 철학입니다.
2.  **ISO 8601 기반 네이밍 룰**:
    `[날짜연월일 8자리]_[과목명]_[이름]_[버전]` 형식 (예: `20261015_파이썬실습_홍길동_v1.0.docx`)으로 이름을 정하면 리스트에서 시간순으로 거의 완벽하게 정렬(Chronological order)됩니다. 띄어쓰기보다는 언더바(`_`)나 하이픈(`-`)을 사용하는 것이 검색 시스템의 색인에 더욱 유리합니다.
3.  **즐겨찾기와 바로가기 활용**:
    작업 중인 중요한 파일은 무겁게 바탕화면으로 복사하지 말고, 파일 탐색기 좌측의 '즐겨찾기(Quick Access)'에 등록하거나 1KB짜리 '바로가기(Shortcut)' 표지판만 바탕화면에 생성하여 원본 파일의 중복 분산을 막습니다. 

### 섹션 3: 클라우드 생태계 마이그레이션

1.  **로컬 스토리지의 한계와 클라우드(Cloud)**:
    컴퓨터 침수, 고장, 분실 시 물리적 기기에 저장된 모든 로컬 데이터는 영구적으로 소실될 위험이 높습니다. 이를 대비해 구글 드라이브(Google Drive) 등 거대 기업의 믿을 수 있는 가상 공간(Cloud)에 데이터를 보관하고 백업, 동기화합니다.
2.  **협업을 위한 보안 링크 공유 (Share Link)**:
    용량이 큰 원본 파일을 메일 첨부파일로 보낼 경우 전송 지연과 용량 제한 에러를 겪기 쉽습니다. 클라우드에 업로드된 파일이나 폴더에 우클릭하여 권한(뷰어, 편집자 등)을 지정한 후 '링크 복사(Get link)'를 이용하여 간편한 URL 텍스트 1줄로 공유하는 파일 기반의 실무 협업 기술을 알아야 합니다.

## 2. 실습 과제 (Assignment)

이번 주는 파이썬 코딩 배제 기간(5~7주차)이므로 Google Colab 실습 파일(lab.ipynb)이 제공되지 않습니다. 
대신 본인의 랩탑(윈도우/맥)에서 직접 다음 활동을 실습해 보시기 바랍니다.

*   **실습 안내**: 
    1. 학교 실습실 컴퓨터(또는 본인 랩탑)를 켜고, 파일 탐색기에서 [보기] 탭을 눌러 **'파일 확장명' 표시**를 켭니다.
    2. **실습용 파일 준비**: 바탕화면이나 다운로드 폴더에 자신의 파일이 있다면 이를 활용합니다. 만약 파일이 없거나 타인의 파일만 있다면, 자신의 구글 드라이브로 가서 그동안 공유된 실습 자료, 핸드아웃 등을 모두 다운로드 받습니다. 혹은 인터넷 구글 검색창에 `컴퓨터 활용 filetype:pdf` 등으로 검색하여 파일을 여러 개 다운로드합니다. 이 방법을 모른다면 제미니(Gemini)에게 물어서 방법을 알아내 진행하세요.
    3. 이렇게 모인 10개 이상의 파일들을 대상으로 수업에서 언급된 ISO 8601 기반 **네이밍 룰**을 적용하여 종류별, 날짜순으로 일괄 수정하고 깔끔하게 정렬합니다. 이 정리된 상태를 화면 캡처(스크린샷)하여 증명합니다.
    4. 정리한 파일 중 적어도 **10개 이상**의 파일을 여러분의 **Google Drive**로 드래그하여 업로드(백업)합니다. 성공적으로 업로드된 구글 드라이브 화면을 스크린샷으로 찍어서 증명합니다.
    5. 준비된 2장의 스크린샷(파일 정리 화면, 구글 드라이브 업로드 화면)을 이번 주차 **과제 제출 구글 폼**을 통해 제출합니다. **(주의: 실습이 끝난 후 공용 컴퓨터에 남은 파일은 모두 스스로 삭제하시기 바랍니다.)**
*   **구글 시트 과제 숙지 확인(Ping)**: 실습 내용을 모두 확인했다면 반드시 담당 구글 시트에 접속하여 본인 이름 옆의 '과제 숙지 확인(Ping)' 란에 '1'을 입력하여 제출을 완료해 주세요.

# 컴퓨팅적사고와코딩
## py07: 내 컴퓨터의 지배자 - 파일 시스템과 드라이브 완벽 제어

**정원혁**

---

# 1. 바탕화면의 저주
- 컴퓨터 초심자의 실수: **모든 파일을 바탕화면에 저장**
- 직관적이어 보이지만, 추후 파일 검색 시 막대한 **시간 낭비** 초래
- 데이터 사일로(Data Silo) 현상과 관리 불가능 상태 야기

---

# 2. 파일 시스템의 트리(Tree) 구조
- 거대한 **도서관 서가**와 같은 컴퓨터 저장 공간
  - 도서관 전체 = `C: 드라이브`
  - 서가 분류 = `폴더 (디렉토리)`
  - 개별 책 = `파일`
- 파일 탐색기(File Explorer)를 통한 계층(Hierarchy) 구조 이해

---

# 3. 확장자(Extension)의 정체성
- `.pdf`, `.docx`, `.jpg`, `.exe` 등 파일명 뒤의 3~4자리 영문자
- 파일의 **신분증** 역할을 수행 (연결 프로그램 결정)
- 💡 **주의**: 확장자를 숨긴 악성 파일(`가짜사진.jpg.exe`) 클릭 방지를 위해 **'파일 확장명 보기' 필수!**
- 다양한 와일드카드(`*`) 검색 요령 숙지 (`*.pdf`)

---

# 4. 윈도우(Windows) 필수 생존 기술
- **클립보드 히스토리(Win+V)**: 다중 복사/붙여넣기로 작업 속도 향상
- **화면 캡처 마스터(Win+Shift+S)**: 영역만 잘라 바로 메일/문서에 첨부
- **가상 데스크톱/창 분할**: 화면을 반반으로 나누거나 가상 모니터로 활용
- **작업 관리자(Ctrl+Shift+Esc)**: 응답 없는 프로그램만 강제 종료(Kill)
- **압축 해제(Zip)**: 폴더 내 직접 실행 시 오류 원리 이해

---

# 5. Search, Don't Sort! (분류 말고 검색하라)
- 5\~6단계로 파고드는 무의미한 폴더 계층 구조화는 금물
- 구글(Google)의 철학: 단일 공간(Flat Directory)에 저장하고 **검색(Search)**으로 1초 만에 찾기!
- 물리적 방 배치보다 **식별자(이름표)** 작성에 에너지를 쏟아야 합니다.

---

# 6. 작업 속도를 올리는 즐겨찾기 & 바로가기
- 현재 가장 많이 쓰는 핵심 폴더는 탐색기 좌측 **'즐겨찾기'**에 등록
- 용량이 큰 원본 파일을 바탕화면으로 옮기는 대신, 1KB짜리 **바로가기(Shortcut)** 생성
- 파일 중복 분산 방지 및 버전 꼬임 원천 차단

---

# 7. 실무 네이밍 룰 (Naming Rule / ISO 8601 기반)
- ❌ 절대 금지: `진짜최종.docx`, `마지막본.pptx`
- ✅ 강력 추천: `[날짜 8자리]_[과목명]_[작성자]_[버전]`
- 예시: `20261015_파이썬실습_정원혁_v1.0.docx`
- 컴퓨터가 날짜(시간)를 인식하여 **완벽한 순서대로 자동 정렬(Chronological order)** 해줍니다.

---

# 8. 클라우드 마이그레이션 (안전한 피난처)
- 로컬 스토리지의 한계: 하드 파손, USB 분실 시 데이터 영구 소실
- 대형 기업(Google)의 보안 서버(`클라우드`)를 활용한 실시간 동기화 마이그레이션
- 폴더 째로 클라우드에 **드래그 앤 드롭** 업로드
- **링크 공유(Share Link)** 기능을 통해 무거운 대용량 파일도 가벼운 URL 한 줄로 즉시 협업!

# py08 중간 평가 방안 모음 (28가지)

> **작성 목적**: 파이썬 K트랙 1~7주차를 마친 학생을 대상으로 한 중간 평가 방안을 체계적으로 정리한다. AI 활용을 허용하고 채점 자동화를 극대화하며, "제한 시간 안에 얼마나 빠르게 해낼 수 있는가"를 핵심 지표로 삼는다.

---

## 평가 설계 원칙

| 원칙 | 내용 |
|---|---|
| **AI 허용** | AI(Gemini, ChatGPT 등)는 막지 않는다. AI를 잘 활용하는 능력 자체를 평가한다. |
| **속도 우선** | 같은 정확도라면 더 빨리 완료한 학생이 높은 점수를 받는다. 제출 시각이 자동 기록된다. |
| **채점 자동화** | 강사 채점 시간을 최소화한다. eCampus 퀴즈 자동 채점, URL 링크 Yes/No 루브릭을 최우선 활용한다. |
| **아티팩트 기반** | 코드 낭독이나 구술 시험 없이, 링크·스크린샷·파일 구조 등 객관적 결과물로 평가한다. |
| **1문항 1조건** | 루브릭 각 항목은 하나의 물리적 조건만 검증한다. 복합 조건 금지. |

## A. 자동 채점 — eCampus 퀴즈 / Sheets 기반 (채점 시간 0분)

### 방안 1. MCQ 타임어택 퀴즈
- **형식**: eCampus 퀴즈 객관식, 20문항, 제한 30분
- **출제 범위**: 에러 메시지 해석(py05), 검색 연산자(py06), 파일 명명 규칙(py07), Python 기초 자료형/연산자(py02~04)
- **채점**: eCampus 퀴즈 자동 채점, 제출 시각 자동 기록 → 동점 시 빠른 제출 우선
- **AI 활용**: 허용. 검색 속도도 능력이다.
- **강사 소요 시간**: 0분 (결과 스프레드시트 다운로드만)

---

### 방안 2. 이미지 기반 오류 찾기 퀴즈
- **형식**: 폴더 구조 스크린샷 또는 구글 시트 화면을 eCampus 퀴즈에 첨부, 오류를 선택지에서 고르기
- **예시 문항**: "아래 파일명 중 ISO 8601 규칙을 위반한 것은?" (보기: 4개 파일명)
- **채점**: eCampus 퀴즈 자동 채점
- **강사 소요 시간**: 0분

---

### 방안 3. AI 생성 허위 데이터 팩트체킹
- **형식**: AI가 만든 구글 시트(일부 데이터가 의도적으로 틀림)를 공유 → 틀린 값 5개를 eCampus 퀴즈에 입력 제출
- **채점**: 정답 배열과 자동 비교 (`=COUNTIF` 또는 Apps Script)
- **강사 소요 시간**: 0분

---

### 방안 4. 프롬프트 선택 최적화 퀴즈
- **형식**: "아래 에러를 AI에게 묻는 가장 효과적인 프롬프트는?" → 4지선다
- **예시**: `TypeError: can only concatenate str (not "int") to str` 에러에 대한 4가지 질문 중 선택
- **채점**: eCampus 퀴즈 자동 채점
- **강사 소요 시간**: 0분

---

### 방안 5. Kahoot / Quizizz 라이브 퀴즈
- **형식**: 수업 중 실시간 진행, py05~07 핵심 개념 15~20문항
- **채점**: 플랫폼 자동 채점 + 실시간 순위 표시 (속도+정확도 동시 반영)
- **강사 소요 시간**: 진행 10분, 채점 0분 (자동 export)
- **장점**: 학생 참여도·긴장감 극대화

---

### 방안 6. Google Sheets 숨겨진 검증 공식
- **형식**: 학생에게 템플릿 시트 배포 → 특정 셀에 정답 입력 → 숨겨진 IF 공식이 자동으로 점수 계산
- **예시**: `=IF(AND(A2="YYYYMMDD",B2="snake_case"),1,0)` 형식의 검증 공식
- **채점**: 학생이 링크 제출 → 강사는 "총점" 셀 하나만 확인
- **강사 소요 시간**: ≤30초/명

## B. URL 아티팩트 제출 — 링크 열어 Yes/No 확인 (채점 ≤2분/명)

### 방안 7. GDrive 폴더 정리 타임어택
- **형식**: 강사가 20개 뒤섞인 파일을 구글 드라이브 폴더로 배포 → 학생이 ISO 8601 + 언더스코어 규칙으로 정리 후 공유 링크 제출
- **루브릭 (Yes/No)**:
  - 기한 내 제출 (5점)
  - 링크가 로그인 없이 열림 (1점)
  - 파일명에 YYYYMMDD 형식 날짜 포함 (1점)
  - 구분자로 언더스코어 또는 하이픈만 사용 (1점)
  - 파일 확장자가 폴더명 또는 파일명에 반영됨 (1점)
- **강사 소요 시간**: ≤2분/명

---

### 방안 8. 구글 시트 4기능 완성 미션
- **형식**: 원시 데이터 시트 배포 → freeze, sort, 조건부 서식, 피벗 테이블 4가지 기능 모두 적용 후 링크 제출
- **루브릭**: py06 기존 루브릭 재활용 (8점 만점)
- **강사 소요 시간**: ≤2분/명 (기존 루브릭 그대로 적용)

---

### 방안 9. GitHub 커밋 증적 제출
- **형식**: py01~04 과제물 GitHub 저장소 링크 제출 → 커밋 타임스탬프로 학습 과정 증명
- **루브릭**: 저장소 공개 여부 + 커밋 4건 이상 + 최신 커밋 날짜 확인
- **강사 소요 시간**: ≤1분/명

---

### 방안 10. AI 대화 로그 3턴 제출
- **형식**: 강사가 제시한 문제를 AI와 3턴 이내에 해결한 대화 전체를 TXT 또는 PDF로 Google Drive에 업로드 후 링크 제출
- **채점 포인트**: 강사는 첫 번째 프롬프트 한 줄만 스캔하여 "역할+목표+제약 조건 포함 여부" 확인
- **강사 소요 시간**: ≤1분/명

## C. 속도 경쟁 / 타임어택 — 제출 시각이 곧 점수

### 방안 11. 파일 명명 변환 드릴
- **형식**: eCampus 퀴즈에 10개 파일명 제시 → ISO 8601 형식으로 변환하여 입력 → 제출 시각 자동 기록
- **예시**: `report final v3 (1).docx` → `20260416_report_v3_홍길동.docx`
- **채점**: 정확도(정답 개수) × 속도 보정 점수 (1위=+2, 2위=+1)
- **강사 소요 시간**: 0분 (자동 채점 설계 시)

---

### 방안 12. 구글 시트 피벗 테이블 스피드런
- **형식**: 동일 데이터셋 배포 → 피벗 테이블(행: 학과, 값: 평균 성적) 완성 → 링크 eCampus 퀴즈 제출 시각 자동 기록
- **채점**: 피벗 정확성(Yes/No) + 제출 시각 순위 보너스
- **강사 소요 시간**: 링크 열어 피벗 존재 여부만 확인 (≤30초/명)

---

### 방안 13. 에러 메시지 번역 릴레이
- **형식**: 한국어 오류 메시지 5개 → 영어 핵심 키워드 추출 → AI 검색용 프롬프트 작성 후 eCampus 퀴즈 제출
- **채점**: 키워드 정확도 + 제출 시각
- **강사 소요 시간**: 0분 (자동 채점 가능)

---

### 방안 14. 폴더 트리 재구성 미션
- **형식**: 뒤섞인 파일/폴더 스크린샷 제시 → 올바른 폴더 트리를 텍스트로 작성하여 eCampus 퀴즈 제출
- **채점**: 정답 트리와 텍스트 비교 (자동 또는 빠른 수동 스캔)
- **강사 소요 시간**: ≤1분/명

---

### 방안 15. 타이핑 속도 + 에러 수정 복합 미션
- **형식**:
  1. 지정 사이트에서 WPM 측정 스크린샷 (기준: 분당 40단어 이상)
  2. 의도적으로 에러가 삽입된 Python 코드 3줄을 Colab에서 수정·실행 후 스크린샷
- **채점**: 두 스크린샷 모두 제출 시 만점 / WPM < 40이면 -1점
- **강사 소요 시간**: ≤1분/명

## D. AI 협업 품질 평가 — AI를 얼마나 잘 쓰는가

### 방안 16. 1턴 프롬프트 챌린지
- **형식**: "단 1번의 AI 질문만으로 주어진 작업을 완료하라" → 결과물 링크 + 사용한 프롬프트 전문을 eCampus 퀴즈 제출
- **채점 포인트**: 결과물 달성 여부 + 프롬프트에 역할/목표/제약이 모두 포함되었는지 (3초 스캔)
- **강사 소요 시간**: ≤30초/명

---

### 방안 17. 할루시네이션 사냥
- **형식**: AI가 생성한 구글 시트(국가 수도, 인구수 등 일부 허위) 배포 → 틀린 데이터 5개를 구글 검색으로 검증하여 eCampus 퀴즈 제출
- **채점**: 정답 자동 비교 (eCampus 퀴즈 정답 설정)
- **핵심 역량**: AI 결과를 검색 연산자로 검증하는 능력 (py05+py06 통합)
- **강사 소요 시간**: 0분

---

### 방안 18. "AI vs Me" 에러 진단 대결
- **형식**: 동일한 Python 에러 메시지를 학생과 AI(Gemini)가 각각 독립적으로 진단 → 두 결과를 나란히 붙여 스크린샷 제출
- **채점**: 학생 설명이 AI보다 구체적이면 +2점, 동등하면 +1점, AI만 맞으면 +0점
- **강사 소요 시간**: ≤1분/명

---

### 방안 19. 3턴 제한 AI 협업 미션
- **형식**: 연락처 10개가 뒤섞인 텍스트에서 이름·전화번호를 추출하여 Google Sheets에 A-Z 정렬로 입력. AI 대화는 3턴 제한.
- **제출물**: 시트 공유 링크 + AI 대화 로그 링크
- **채점**: 시트 완성(Yes/No) + 3턴 이내 여부(Yes/No) + 제출 시각 보너스
- **강사 소요 시간**: ≤1분/명

---

### 방안 20. 프롬프트 A/B 비교 제출
- **형식**: 동일 문제에 대해 나쁜 프롬프트와 개선된 프롬프트를 각각 AI에 입력 → 두 결과 스크린샷 + 차이점 3줄 설명을 eCampus 퀴즈 제출
- **채점**: 차이점 설명에 "구체성/역할/맥락" 키워드 포함 여부 (빠른 스캔)
- **강사 소요 시간**: ≤1분/명

- **채점**: 제출 여부 (Yes/No) 기반 완료 점수. 내용 정확도 부가 평가 선택.

## F. 동기식 / 라이브 평가 — 수업 중 진행

### 방안 25. 이스케이프 룸 형식 미션

- **채점**: 모든 5단계 완료 여부 + eCampus 퀴즈 제출 시각 순위

---

### 방안 26. 라이브 스크린 공유 타임어택
- **형식**: Zoom/Teams 화면 공유 상태에서 강사가 실시간 미션 제시 → 가장 먼저 완료한 학생 순으로 점수
- **AI 허용**: 완전 허용
- **채점**: 완료 시각 강사 기록 또는 자동 eCampus 퀴즈 제출 시각

### 방안 27. 인간+AI 페어 챌린지

- **채점**: 팀 내 비교 + 완료 링크 제출 시각
- **부가 효과**: AI 도구 간 차이를 학생이 직접 체험

---

### 방안 28. 플래시 챌린지 (5분 릴레이)
- **형식**: 5분마다 새로운 미션 제시, 3회 연속 완료 시 만점. 중간 실패 후 다음 미션으로 복구 가능.
- **채점**: 연속 성공 횟수 기반 점수
- **예시 미션**: "30초 안에 구글에서 `site:go.kr filetype:pdf 2025` 검색 결과 화면 캡처 후 제출"
- **강사 소요 시간**: 0분 (자동 eCampus 퀴즈 제출 시각으로 판단)

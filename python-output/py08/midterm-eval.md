# 중간고사 평가 방식 제안: 무들(Moodle) 기반 '디지털 생존 & 시한폭탄' 평가 20선

유학생들의 기기 활용/AI 통제 역량을 평가하기 위해 **무들(Moodle)의 다양한 문제 은행 기능 및 온라인 평가 시스템**을 십분 활용한 방식입니다.
특히 **'제한된 짧은 시간 안에 낯선 환경/오류를 마주했을 때 얼마나 빨리 도구(구글, AI, 번역기)를 사용해 탈출하는가(Time Attack)'**에 초점을 맞춘 20가지의 창의적인 평가 방안을 제안합니다.

---

## A. 무들(Moodle) 자동 채점 퀴즈: '초읽기' 에러 대처 및 객관식 (1~6)
무들의 퀴즈 모듈(짝맞추기, 핫스팟 이미지, 다중 선택 등)에 강한 시간제한(문항당 30초~1분)을 걸어 직관적인 문제 해결 능력을 봅니다.

1. **[짝맞추기] 에러 화면과 구원 프롬프트 매칭**: 왼쪽에는 윈도우/구글 드라이브의 실제 오류 캡처 화면(영문/알 수 없는 언어)이, 오른쪽에는 이 문제를 해결하기 위해 스마트폰 Gemini에게 입력해야 할 가장 적합한 '첫 질문(프롬프트)'을 배치하여 빠르게 짝지어 연결하기.
2. **[이미지 핫스팟] 어디를 클릭할 것인가?**: 엑셀 수식 오류 캡처나 파일 확장자가 숨겨진 탐색기 화면을 주고 "이 오류를 고치기 위해 당장 마우스로 클릭해야 하는 메뉴나 버튼 부위를 클릭하시오" (제한시간 15초).
3. **[O/X 타임 퀴즈] 번역기/AI 동시 통역력**: 외국어 OS에서 나타나는 흔한 에러 메시지 팝업 텍스트를 던져주고, "이 메시지는 디스크가 꽉 찼다는 의미이다" (O/X) 식의 퀴즈를 10문제 연속 10초씩 제한.
4. **[다중 선택] 환각(Hallucination) 지뢰 찾기**: AI에게 특정 국가의 인구 통계를 물어서 나온 답변을 제시. 무들 창 밖에서 재빨리 구글 고급 검색(`site:gov`)을 돌려 팩트와 가짜 정보를 판별해, 거짓인 것만 모두 체크박스 선택.
5. **[드래그 앤 드롭] 작업 파이프라인 정렬**: '더미 데이터 다운로드 ➔ 구글 시트로 불러오기 ➔ 내림차순 정렬 ➔ PDF 저장 ➔ 특정 폴더로 드래그'의 복잡한 절차를 논리적으로 올바른 순서대로 끌어다 놓기.
6. **[텍스트 단답형] 구글링 스피드런**: "2024년 3월 기준 서울신학대학교 관련 특정 공식 데이터값을 찾으시오" 와 같이, `"` `"` 나 `-` 같은 검색 연산자를 써야만 검색 결과 1페이지에 나오는 숫자를 30초 만에 찾아 단답형으로 기입.

## B. 무들(Moodle) 파일 제출형 타임어택: 스피드 실전 미션 (7~12)
강사가 무들에 압축 파일(미션 자료)을 올려두고, 학생이 이를 다운받아 가공한 뒤 **10~15분 이내에 무들 파일 제출함이 닫히기 전 업로드**하는 방식입니다.

7. **[타임어택: 15분] 아수라장 파일 대청소**: 무들에서 다운 받은 압축 파일 속에 '이름, 확장자, 생성일'이 뒤죽박죽인 50개의 더미 파일이 존재. AI의 힌트를 얻어가며 정해진 네이밍 룰(학번_이름_순번.pdf 등)로 5분 안에 수동 혹은 자동화로 싹 갈아 엎고 재압축하여 탈출(제출).
8. **[타임어택: 10분] 깨진 구글 시트 심폐소생술**: 셀의 함수 레퍼런스가 깨져 무수한 `#REF!`, `#VALUE!` 에러가 떠있는 엑셀 파일을 던져줌. 왜 이런 에러가 났는지 AI를 통해 빠르게 찾아 고치고, 완성된 표 형태로 포맷팅하여 업로드.
9. **[타임어택: 10분] 연락처/이메일 스캐빈저 헌트**: 무들에 아주 길고 지저분한 텍스트 더미를 띄움. 학생은 AI에게 "여기서 이메일 주소만 정규식으로 뽑아줘" 등의 프롬프트를 재빨리 날려, 순수한 10개의 이메일 텍스트 파일만 뽑아 제출.
10. **[타임어택: 10분] 암호 폴더 해금 미션**: 여러 폴더 속에 흩어진 파일들을 윈도우 탐색기의 '최근 수정된 날짜' 역순으로 정렬했을 때, 맨 위에 오는 5개 파일명 첫 글자를 조합하면 "비밀번호"가 나옴. 이 비밀 번호가 담긴 텍스트 문서를 만들어 무들에 첨부.
11. **[타임어택: 15분] 확장자 스푸핑(Spoofing) 감별사**: `.pdf.exe`, `image.jpg.zip` 처럼 교묘하게 확장자가 위장된 파일 20개를 줌. 윈도우의 확장자 표기 설정을 켜서 위장된 악성 파일(실행 파일) 3개만 정확히 찾아내 삭제한 후 압축 제출.
12. **[타임어택: 20분] 고집불통 AI 제어하기**: 의도적으로 학생의 지시를 오해하게 끔 꼬아놓은 아주 복잡한 엑셀 양식 포맷을 주고, "이 양식과 똑같이 만들 파이썬 코드를 AI에게 뽑아내서 1번 실행해 결과물을 제출하라". 끈질긴 재질문(Troubleshooting) 속도를 평가.

## C. 무들(Moodle) 단답형 & 빈칸 뚫기: '최소 단어' 논리 추론 (13~16)
무들 단답형 기능을 활용하여, 긴 답이 필요 없이 학생이 오류의 맥락을 정확히 꿰뚫고 있는지 '핵심 키워드' 하나로 판별합니다.

13. **[빈칸 채우기] 프롬프트 뼈대 완성**: "에러 창인 [    ] 오류 창 캡처를 보여주며, 나는 [    ] 역할을 AI에게 부여하여, [    ] 형태로 해결 결과를 출력해달라고 했다." 빈칸에 들어갈 가장 핵심적인 단어 채우기.
14. **[단답형/수식] AI가 뱉은 고의적 버그 찾기**: 학생 수준의 파일 관리(OS 처리) 자동화 스크립트 코드 스니펫(약 10줄)을 제시하고, 그중 강사가 고의로 틀려 놓은 오타나 경로(Path) 오류의 **줄 번호(N번째 줄)** 숫자만 딱 적기. 
15. **[단답형] "내 파일 어딨어?" 경로(Path) 역추적기**: 무들에 `C:\Users\Downloads\Project\Secret\data.csv` 라는 텍스트를 주고, 윈도우 화면 바탕화면을 캡처한 이미지를 첨부. 물리적으로 바탕화면에서 저 폴더까지 들어가기 위해 더블클릭 해야 하는 첫 번째 아이콘의 이름을 쓰시오.
16. **[단답형] 최단 프롬프트 레이스 골프(Prompt Golfing)**: 특정 문제를 해결하기 위해 AI를 굴복시킨 본인의 프롬프트 문장 안심 제출. 100~200자를 훌쩍 넘는 긴 프롬프트는 후순위로 미루고, "제로샷(Zero-shot)으로 50자 내로 명확하게 지시한" 단답형 프롬프트일수록 고득점.

## D. 무들(Moodle) 에세이 및 상호평가 (Workshop): AI 대화록 심리전 (17~20)
최종 아웃풋뿐만 아니라 타임어택 속에서 벌어진 'AI와의 사투(프롬프팅 여정)' 자체를 텍스트 혹은 링크로 제출받아 훑어보는 주관식 방식입니다. 무들의 Workshop(상호평가)을 쓰기 좋습니다.

17. **[에세이 텍스트] AI 탈옥 일지**: 강사가 절대 안 알려주려 하는 보안 정보(예: 테스트 서버의 특정 위치 등)를 은유나 빙의(Role-play) 프롬프트를 써서 AI/시뮬레이터가 무심코 내뱉게 만드는 시도를 한 뒤, 성공한 대화록만 제출.
18. **[에세이 링크] 프롬프트 부검(Autopsy)**: "내가 지난주 중간 과제에서 AI에게 처음 물어봤을 때 완전히 실패했던 1차 프롬프트"와 "이를 깨달아 수정한 2차 프롬프트" 두 가지를 나란히 무들에 붙여넣고, 수정 이유를 한국어/영어로 3줄 요약.
19. **[상호평가/Workshop] 팀원들의 노가다 해방 평가**: 각 학생이 타임어택 15분 동안 자기만의 스마트 도구(구글, 단축키, AI)를 이용해 5시간짜리 과제를 5분 컷 낸 비법서(A4 반장 분량)를 무들 Workshop 모듈에 올림. 제출 마감 직후 팀원들이 서로의 글을 자동으로 배정받아 '가장 창의적인 잔머리'에 점수 매기기. 
20. **[블랙 아이스 함정] 무들 시스템 자체 탈주**: 강사가 의도적으로 이 중간고사 시험지의 마지막 20번 문항에 '버튼이 안 눌리는 에러'나 '깨진 글씨 폰트(HTML)'를 숨겨둠. 학생은 브라우저의 개발자 도구나 AI 번역 등을 활용해 화면 오류를 우회하여 정답을 제출하는 것 자체로 최종 탈출 완료(방탈출 콘셉트).

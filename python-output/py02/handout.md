## 3. 과제 섹션 — Micro-Assignment

> **제출 방법**: 1주차 때 만든 GitHub 저장소에 기록 → 완성본의 웹 링크 1개만 eCampus 2주차 제출함에 제출
> **제출 기한**: 다음 주 수업 전날 23:59
> **분량**: 읽기 쉬운 줄글 (표준 마이크로 과제 형식 준수)

### 이번 주 제출 과제: "내가 겪은 첫 번째 에러 분석"

오늘 수업 시간에 일부러 발생시킨 에러와 그것을 AI를 통해 해결한 과정을 되새기며, **표준 마이크로 과제 3단 형식**으로 과제를 수행합니다.

**Step 1: GitHub에 새 파일 만들기**
1. 1주차에 생성했던 본인의 **`stu2603c-assignments`** 저장소에 접속합니다.
2. 상단의 `Add file` → `Create new file`을 누르고 파일명을 **`week02.md`**로 만듭니다.

**Step 2: 마이크로 과제 형식으로 본문 작성하기**
`week02.md` 안에 본인이 오늘 겪은 에러 상황을 다음 세 가지 양식으로 나눠서 서술합니다.
(양식의 큰 틀은 아래 텍스트를 복사해서 쓰시면 편합니다.)

```markdown
# 2주차 과제: 나의 첫 에러 분석

## 1. AI Prompt (내가 AI에게 던진 질문)
파이썬에서 print"안녕하세요파이썬" 이라고 입력했더니 SyntaxError: Missing parentheses in call to 'print' 에러가 났어. 뭐가 문제인지 쉽게 설명해줘.
(그리고 오류가 났던 당시 내 Colab 화면 캡처 이미지를 여기에 첨부하거나 넣기)

## 2. AI Critique (AI 답변 비평 및 한 줄 요약)
AI는 내가 print 명령어에 괄호를 쓰지 않아서 문법에 어긋났다고 지적했다. 빨간 줄만 보고 컴퓨터가 고장 난 줄 알았는데, 아주 친절하게 어디를 고칠지 짚어준다는 점을 알게 되었다. 내가 작성한 이 코드의 문제 핵심은 괄호 누락이다.

## 3. Final Result (최종 검증 및 해결책 적용)
AI의 조언을 듣고 코드를 print("안녕하세요파이썬")으로 고쳤다. 따옴표는 제대로 썼지만 괄호가 없어도 에러가 날 수 있음을 직접 눈으로 확인했다. 코드를 수정하니 더 이상 에러가 나지 않았다.
```

위의 예시 글을 본인의 언어와 본인이 직접 겪은 에러 메시지에 맞게 조금씩 다듬어서 저장(`Commit changes`)합니다.

**Step 3: URL 링크 제출 [최종]**
작성을 완료한 방금 그 `week02.md` 파일 화면의 브라우저 상단 전체 **URL 링크 주소**를 복사한 뒤, eCampus 2주차 제출란에 해당 텍스트 링 하나만 붙여넣고 제출합니다.

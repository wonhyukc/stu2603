### print() 기본 사용법

파이썬 코딩을 처음 배울 때 만나는 가장 첫 번째 함수가 바로 `print()`입니다. 지난주에도 한 번 써봤는데, 오늘은 이 `print()`를 진짜 제대로 이해해볼 거예요.

`print()`는 화면에 무언가를 출력하는 함수입니다. 영어로 말하면 "인쇄하다"인데, 프로그래밍에서는 **화면에 내용을 보여주다**는 의미로 씁니다. 코드로 만든 결과를 눈으로 확인하는 가장 기본적인 방법이에요.

"함수"라는 단어가 아직 낯설게 느껴지는 분들을 위해 잠깐 설명드릴게요. 함수(Function)는 **특정 일을 수행하는 명령의 묶음**이라고 생각하면 됩니다. 요리로 비유하면 레시피와 비슷합니다. `볶음밥 만들기`라는 레시피를 한 번 정해두면, 이후에는 "볶음밥 만들어!"라고 이름만 부르면 됩니다. `print()`도 마찬가지예요. 화면에 글자를 보여주는 복잡한 내부 과정이 이미 정의되어 있고, 우리는 `print()`라고 이름만 부르면 그 과정이 자동으로 실행됩니다.

파이썬에는 이처럼 미리 만들어진 유용한 함수들이 많습니다. 이런 것들을 **내장 함수(Built-in Function)**라고 부릅니다. `print()`도 그중 하나입니다. 나중에 우리가 직접 함수를 만드는 것도 배우지만(11주차), 지금 단계에서는 파이썬이 미리 만들어준 도구들을 잘 활용하는 것이 먼저입니다.

가장 기본적인 사용법부터 보겠습니다.

```python
print("안녕하세요, 파이썬!")
```

이 코드를 실행하면 화면에 `안녕하세요, 파이썬!` 이 그대로 출력됩니다. 여기서 큰따옴표 `"` 안에 들어간 내용이 화면에 나옵니다. 작은따옴표 `'`도 동일하게 작동합니다.

```python
print('작은따옴표도 됩니다')
```

숫자도 출력할 수 있습니다.

```python
print(42)
print(3.14)
```

숫자를 출력할 때는 따옴표가 필요 없습니다. `42`를 따옴표 없이 그냥 써도 됩니다. 지금은 이 차이를 가볍게 눈에 담아두세요. 나중에 자료형 섹션에서 더 자세히 설명할 겁니다.

그러면 여러 개의 값을 한 번에 출력하려면 어떻게 할까요? 방법이 두 가지 있습니다. **콤마(`,`)를 사용하는 방법**과 **더하기 기호(`+`)를 사용하는 방법**입니다. 이 둘의 차이가 이번 섹션에서 가장 중요한 내용이에요.

---

### 콤마(`,`) vs 더하기(`+`): 결정적 차이

먼저 콤마를 사용하는 방법입니다.

```python
print("이름:", "홍길동")
print("나이:", 25)
```

실행 결과:
```
이름: 홍길동
나이: 25
```

콤마로 여러 값을 나열하면 파이썬이 각 값 사이에 **자동으로 공백을 하나 넣어줍니다**. 그리고 가장 중요한 점은, **서로 다른 타입을 섞어도 된다**는 겁니다. 위 예시에서 `"나이:"` 는 문자열이고, `25`는 숫자인데 콤마 하나로 아무런 문제 없이 출력됩니다.

이번엔 `+`를 사용해 보겠습니다.

```python
print("이름: " + "홍길동")
```

실행 결과:
```
이름: 홍길동
```

`+`로 연결하면 두 문자열이 **공백 없이 바로 붙어서** 출력됩니다. 공백이 필요하다면 직접 넣어줘야 합니다. `"이름: "` 뒤에 제가 이미 공백을 넣어놓은 게 보이시나요?

그런데 `+`에는 치명적인 제한이 하나 있습니다. **문자열끼리만 연결할 수 있다**는 거예요. 만약 이렇게 쓰면 어떻게 될까요?

```python
print("나이: " + 25)
```

파이썬이 에러를 냅니다. `"나이: "` 는 문자열인데 `25`는 숫자이고, 이 둘을 `+`로 연결할 수 없기 때문이에요. 숫자를 문자열로 바꿔주지 않으면 파이썬은 이 상황을 이해하지 못합니다. 이 에러가 바로 나중에 배울 `TypeError`입니다.

정리해 드릴게요.

| 방법 | 공백 | 타입 혼합 |
|:-----|:----:|:--------:|
| 콤마(`,`) | 자동 삽입 | 가능 |
| `+` 연결 | 직접 삽입 필요 | 문자열끼리만 가능 |

어떤 상황에서 뭘 쓸까요? **서로 다른 타입(문자열 + 숫자)을 함께 출력할 때는 콤마**, **문자열을 이어 붙여 하나의 긴 문자열로 만들 때는 `+`**라고 기억해 두세요.

초보자들이 가장 자주 실수하는 패턴이 바로 이 차이를 무시하고 `print("값: " + 숫자)`처럼 쓰는 겁니다. 이 실수는 나중에 `input()`을 배울 때도 그대로 반복됩니다. 그래서 지금 이 차이를 머릿속에 확실히 새겨두는 게 중요합니다.

추가로, `+`를 연속해서 사용할 때는 **공백 삽입**에 더 신경 써야 합니다.

```python
first = "홍"
last = "길동"
greeting = "안녕하세요 " + first + last + "님!"
print(greeting)    # 안녕하세요 홍길동님!
```

각 연결 지점마다 공백이 필요한 위치에 직접 넣어줘야 합니다. 이게 번거로울 때는 나중에 배울 **f-string**을 사용하면 훨씬 편합니다. f-string은 3주차에서 배우는데, 문자열과 변수를 자연스럽게 섞어 쓸 수 있어서 현대 파이썬 코드에서는 `+` 연결보다 f-string을 더 많이 씁니다. 지금은 `+`로 연결하는 방법을 익혀두고, 나중에 f-string으로 업그레이드한다고 생각하면 됩니다.

### end= 옵션: 줄 바꿈을 내 마음대로

`print()` 함수는 기본적으로 출력이 끝나면 **자동으로 줄을 바꿉니다**. 영어로는 `newline`이라고 하고, 코드에서는 `\n`이라는 특수 문자로 표현합니다. 평소에는 이걸 신경 쓸 필요가 없는데, 가끔 줄을 바꾸지 않고 같은 줄에 이어서 출력하고 싶을 때가 생깁니다. 그럴 때 `end=` 옵션을 씁니다.

기본 동작부터 확인해볼게요.

```python
print("첫 번째")
print("두 번째")
```

실행 결과:
```
첫 번째
두 번째
```

두 줄에 각각 출력됩니다. 이제 `end=` 옵션을 추가해 볼게요.

```python
print("첫 번째", end=" ")
print("두 번째")
```

실행 결과:
```
첫 번째 두 번째
```

`end=" "`라고 지정했더니 첫 번째 출력 뒤에 줄을 바꾸는 대신 공백 하나를 넣고 다음으로 넘어갔습니다. `end=`에는 어떤 문자열이든 넣을 수 있습니다. `end="---"`이라고 하면 출력 뒤에 `---`가 붙겠죠.

```python
print("월요일", end=", ")
print("화요일", end=", ")
print("수요일")
```

실행 결과:
```
월요일, 화요일, 수요일
```

이렇게 쉼표와 공백을 넣어서 한 줄에 나란히 출력하는 것도 가능합니다. `end=` 옵션은 당장 많이 쓰지 않더라도 "이런 게 있다"는 걸 알아두면 나중에 분명히 쓸 일이 생깁니다.

---

### sep= 옵션: 구분자를 내 마음대로

`sep=` 옵션은 콤마로 여러 값을 나열했을 때 **각 값 사이에 들어가는 구분자**를 바꾸는 옵션입니다. 기본값은 공백 한 칸(`" "`)입니다.

```python
print("2026", "03", "07")
```

실행 결과:
```
2026 03 07
```

각 값 사이에 공백이 들어가죠. 이제 `sep=`를 바꿔볼게요.

```python
print("2026", "03", "07", sep="-")
```

실행 결과:
```
2026-03-07
```

날짜 형식처럼 만들었습니다. `-` 하이픈이 구분자로 들어갔네요. 슬래시로 바꿀 수도 있어요.

```python
print("2026", "03", "07", sep="/")
```

실행 결과:
```
2026/03/07
```

`sep=""`으로 빈 문자열을 지정하면 구분자가 전혀 없이 붙어서 출력됩니다.

```python
print("파", "이", "썬", sep="")
```

실행 결과:
```
파이썬
```

`end=`와 `sep=` 두 옵션을 동시에 사용할 수도 있습니다. 이처럼 `print()` 하나에도 꽤 다양한 기능이 숨어 있습니다. 처음에는 콤마와 `+`의 차이만 확실히 이해해도 충분합니다. `end=`와 `sep=`는 "아, 이런 게 있구나" 정도로만 기억해 두세요.

### 주석(#): 코드에 메모를 남기는 법

마지막으로 주석에 대해 이야기할게요. 코드를 작성하다 보면 나중에 다시 봤을 때 "이게 뭐 하는 코드지?" 하고 헷갈릴 때가 있습니다. 그럴 때를 대비해서 **코드 옆에 설명을 달아두는 것**이 주석입니다.

파이썬에서 주석은 `#` 기호로 시작합니다. `#` 뒤에 오는 모든 내용은 파이썬이 **실행하지 않고 그냥 무시**합니다.

```python
# 이 줄은 실행되지 않습니다
print("안녕하세요")  # 이 부분도 실행되지 않습니다
```

위 코드를 실행하면 `안녕하세요` 만 출력됩니다. `#` 이후의 내용은 사람이 읽기 위한 메모이지 파이썬이 처리하는 코드가 아닙니다.

```python
# 사용자에게 인사를 출력하는 코드
print("안녕하세요, 파이썬 세계에 오신 것을 환영합니다!")

# 다음 줄은 테스트 중이라 잠시 꺼놓음
# print("이 줄은 지금 실행 안 됨")
```

주석은 지금 당장은 단순해 보이지만, 코드가 길어질수록 점점 더 중요해집니다. 혼자 공부할 때도 "이 코드가 뭘 하는 건지"를 짧게 적어두는 습관을 들이면, 나중에 코드를 다시 볼 때 훨씬 빠르게 이해할 수 있습니다.

코드를 처음 쓸 때부터 주석을 다는 습관이 있는 사람과 그렇지 않은 사람은, 나중에 공부가 쌓였을 때 복습하는 속도가 완전히 달라집니다. 지금 당장은 주석이 사소해 보일 수 있지만, 코드가 50줄, 100줄로 늘어날 때 과거의 내가 남겨준 주석이 얼마나 소중한지 알게 됩니다. 그러니 지금부터 코드를 쓸 때마다 "이 코드는 무슨 역할을 하는가"를 짧게 한 줄 적어두는 연습을 해보세요.

주석과 관련해서 한 가지 더 말씀드리고 싶은 것이 있습니다. 처음에는 주석을 "정답"처럼 쓰려는 경향이 있는데, 사실 주석은 완벽할 필요가 없습니다. "이 부분 나중에 다시 확인하기", "왜 이렇게 됐는지 아직 모름" 같은 메모도 훌륭한 주석입니다. 주석은 나를 위한 것이지 다른 사람에게 보여주기 위한 것이 아니니까요. 물론 나중에 팀 프로젝트를 하거나 코드를 공유할 때는 명확하고 이해하기 쉬운 주석을 써야 하지만, 지금 단계에서는 일단 무엇이든 적어두는 게 좋습니다.

오늘 배운 것을 잠깐 정리하고 섹션 2로 넘어가겠습니다. `print()`의 기본 출력, 콤마(`,`)와 `+`의 차이, `end=`와 `sep=` 옵션, 그리고 주석(`#`)까지 배웠습니다. 이제 **파이썬이 다루는 데이터의 종류**, 즉 자료형을 본격적으로 배울 시간입니다.

## Section 2: 4가지 기본 타입 + 산술 연산자 (25분)

*(강사 참고: 이 섹션은 개념 설명이 많습니다. 비유를 충분히 사용하고, 각 타입마다 type() 함수로 직접 확인하는 라이브 데모를 꼭 포함하세요.)*

---

섹션 2를 시작합니다. 방금까지 `print()`로 여러 가지를 출력해봤는데, 출력할 때 문자열은 따옴표로 감쌌고 숫자는 그냥 썼죠? 그 차이는 단순한 표기 방식의 차이가 아니라, **데이터의 종류(타입)가 다르기 때문**입니다.

잠깐 우리 일상으로 눈을 돌려볼게요. 우리가 주방에서 요리를 할 때 재료들을 어떻게 분류합니까? 고체 재료, 액체 재료, 가루 재료처럼 종류별로 나눠서 다르게 다루죠. 밀가루를 계량컵으로 측정하지 않고 숟가락으로 퍼서 쓰듯이, 재료의 종류에 따라 다루는 방법이 달라집니다. 파이썬에서 데이터 타입이 바로 이런 역할을 합니다. 각 데이터가 어떤 종류인지 알아야 그걸 어떻게 처리해야 할지 파이썬이 결정할 수 있습니다.

한 가지 더 비유를 들면, 마치 우체국에서 택배를 분류하는 것과 같습니다. 일반 편지, 소포, 특급 배송, 국제 우편처럼 종류에 따라 다르게 처리하잖아요. 파이썬도 마찬가지입니다. 들어온 데이터가 어떤 타입인지에 따라 저장하는 방식, 처리하는 방식, 허용되는 연산이 모두 달라집니다.

파이썬에서 모든 데이터에는 **종류(타입)**가 있습니다. 이 타입에 따라 파이썬이 데이터를 다루는 방식이 달라집니다. 예를 들어 숫자는 더하고 빼고 나눌 수 있지만, 문자열은 그렇게 할 수 없습니다. 타입을 이해하면 왜 특정 코드에서 에러가 나는지도 자연스럽게 이해하게 됩니다.

### str — 문자열 타입

첫 번째는 **`str`**, 문자열(String)입니다. 우리가 일상에서 주고받는 텍스트, 메시지, 이름, 설명문 같은 것들이 모두 문자열입니다.

파이썬에서 문자열은 **반드시 따옴표로 감싸야** 합니다. 큰따옴표(`"`) 또는 작은따옴표(`'`) 둘 다 됩니다.

```python
name = "홍길동"
greeting = '안녕하세요!'
sentence = "오늘 날씨가 정말 좋네요."
```

숫자처럼 생겼어도 따옴표 안에 있으면 문자열입니다. 예를 들어 `"42"`는 숫자 사십이가 아니라 그냥 문자 `4`와 `2`가 붙어있는 문자열입니다.

```python
phone = "010-1234-5678"  # 전화번호는 문자열로 저장
zip_code = "06579"       # 우편번호도 문자열로 저장
```

전화번호나 우편번호는 숫자처럼 생겼지만, 더하거나 빼는 계산을 하지 않죠. 그리고 앞에 0이 붙을 수도 있습니다. 이런 경우에는 문자열로 저장하는 것이 맞습니다. 이처럼 **"계산에 쓸 숫자인가, 아닌가"**가 문자열과 숫자를 구분하는 좋은 기준이 됩니다.

문자열에는 한 가지 흥미로운 특성이 있습니다. 문자열도 `+`로 이어붙이는 연산이 가능합니다. 이것을 **문자열 연결(Concatenation)**이라고 합니다.

```python
first_name = "길동"
last_name = "홍"
full_name = last_name + first_name
print(full_name)    # 홍길동
```

그리고 문자열에 숫자를 곱하면 그 문자열이 반복됩니다.

```python
print("하하" * 3)   # 하하하하하하
print("-" * 20)     # --------------------  (구분선으로 자주 씀)
```

이처럼 문자열은 단순히 텍스트를 저장하는 것 이상의 다양한 조작이 가능합니다. 문자열의 더 다양한 기능들은 3주차에서 집중적으로 배웁니다.

---

### int — 정수 타입

두 번째는 **`int`**, 정수(Integer)입니다. 정수는 소수점 없는 숫자입니다. 1, 2, 100, -5, 0 같은 숫자들이 모두 `int`입니다.

```python
age = 20
count = 100
temperature = -3
year = 2026
```

정수는 따옴표 없이 그냥 씁니다. 따옴표를 쓰는 순간 문자열이 되어버립니다.

정수는 **사칙연산이 가능**합니다. 더하고, 빼고, 곱하고, 나눌 수 있어요. 이건 섹션 후반에 연산자를 설명할 때 더 자세히 보겠습니다.

한 가지 흥미로운 점은, 파이썬의 정수는 크기 제한이 없습니다. 다른 프로그래밍 언어에서는 정수를 저장할 수 있는 크기에 한계가 있는데, 파이썬은 아무리 큰 숫자도 정수로 다룰 수 있습니다. 지금은 "그렇구나" 정도로 알아두세요.

---

### float — 부동소수점 타입

세 번째는 **`float`**, 부동소수점(Floating Point)입니다. 소수점이 있는 숫자가 `float`입니다.

```python
height = 175.5
pi = 3.14159
temperature = 36.5
price = 9900.0
```

`9900.0`처럼 소수점 아래가 `0`이더라도, **소수점이 있으면 `float`**입니다.

`int`와 `float`의 차이는 생각보다 중요합니다. 예를 들어 `10 / 3`을 계산하면 파이썬은 `3.3333...`을 반환합니다. 나눗셈의 결과는 자동으로 `float`가 됩니다. 반면 `10 // 3`은 소수점을 버리고 정수 `3`을 반환합니다. 이 차이는 연산자를 설명할 때 다시 나옵니다.

비유를 들자면, 피자 한 판을 10조각으로 자를 때 정수로 나누면 "나는 3조각 먹을 수 있다"가 되고, 실수로 나누면 "나는 3.33... 조각을 먹을 수 있다"가 됩니다. 현실에서는 3조각을 먹겠지만, 수학적으로 정확한 계산이 필요한 경우에는 소수점 있는 결과가 필요합니다.

### bool — 불린 타입

네 번째는 **`bool`**, 불린(Boolean)입니다. 이것은 조금 특별한 타입입니다. `True` 아니면 `False`, 단 **두 가지 값만** 가질 수 있습니다.

```python
is_student = True
is_raining = False
has_passed = True
```

불린 타입은 조건문과 함께 쓸 때 진짜 위력을 발휘합니다. 4주차에서 배울 `if` 문이 바로 이 불린 값을 기준으로 코드의 흐름을 나눕니다. 지금은 "참이냐 거짓이냐를 나타내는 타입이구나" 정도로 이해해 두세요.

`True`와 `False`는 반드시 첫 글자가 **대문자**여야 합니다. `true`나 `false`라고 쓰면 파이썬이 인식하지 못합니다.

```python
# 올바른 사용
is_correct = True
is_wrong = False

# 잘못된 사용 (에러 발생)
# is_correct = true    # NameError!
# is_wrong = false     # NameError!
```

불린 타입이 왜 중요한지 미리 살짝 맛보기를 드리면, 나중에 배울 조건문에서 이런 식으로 씁니다. "만약 `is_student`가 `True`라면 이 코드를 실행해라"처럼요. 프로그램이 어떤 행동을 할지 말지를 결정하는 데 불린이 핵심적인 역할을 합니다.

### type() 함수로 타입 직접 확인하기

자, 이제 네 가지 타입을 모두 설명했습니다. 그런데 말로 설명하는 것보다 **직접 코드로 확인**하는 게 훨씬 와닿죠. 파이썬에는 데이터의 타입을 알려주는 내장 함수가 있습니다. 바로 **`type()`** 함수입니다.

```python
print(type("안녕하세요"))
print(type(42))
print(type(3.14))
print(type(True))
```

실행 결과:
```
<class 'str'>
<class 'int'>
<class 'float'>
<class 'bool'>
```

`type()` 함수는 괄호 안에 넣은 값의 타입을 `<class '타입이름'>` 형식으로 알려줍니다. 지금은 `class`라는 단어가 낯설 수 있는데, 나중에 배울 내용이니 지금은 `str`, `int`, `float`, `bool` 이름만 눈에 익히면 됩니다.

헷갈리는 경우에도 `type()`으로 즉시 확인할 수 있습니다.

```python
print(type("42"))   # <class 'str'>  — 따옴표 안에 있으므로 문자열
print(type(42))     # <class 'int'>  — 따옴표 없는 숫자
print(type(42.0))   # <class 'float'> — 소수점이 있으므로 부동소수점
print(type(True))   # <class 'bool'> — 불린
```

실제 코드를 작성하다가 "이게 지금 문자열이야, 숫자야?" 헷갈릴 때 `type()`을 쓰는 습관을 들이면 많은 에러를 예방할 수 있습니다.

`type()` 함수는 두 가지 방식으로 활용할 수 있습니다. 첫째는 지금 본 것처럼 `print(type(값))`으로 화면에 출력하는 방법, 둘째는 조건문과 함께 쓰는 방법입니다. 예를 들어 `type(x) == int` 처럼 씁니다. 조건문은 4주차에 배우니까 지금은 첫 번째 방법만 알면 충분합니다.

그리고 `type()` 함수를 쓸 때 초보자들이 자주 실수하는 것이 있습니다. `type(x)`의 결과를 직접 화면에서 확인하고 싶으면 반드시 `print()`로 감싸야 한다는 것입니다. `type(42)`라고만 쓰면 Colab에서는 결과가 표시되지만, 일반 파이썬 스크립트에서는 화면에 아무것도 안 나옵니다. 항상 `print(type(값))` 형태로 쓰는 습관을 들이세요.

### 산술 연산자 7종 완전 정복

이제 **산술 연산자**를 배울 차례입니다. 연산자는 숫자(int, float)를 가지고 계산을 할 때 사용하는 기호입니다. 파이썬의 기본 산술 연산자는 일곱 가지입니다.

먼저 우리가 수학 시간에 배운 것들부터 보겠습니다.

```python
# 덧셈
print(10 + 3)   # 결과: 13

# 뺄셈
print(10 - 3)   # 결과: 7

# 곱셈
print(10 * 3)   # 결과: 30

# 나눗셈
print(10 / 3)   # 결과: 3.3333333333333335
```

여기까지는 우리가 아는 사칙연산과 같습니다. 그런데 나눗셈(`/`)의 결과를 보면 `3.3333...`처럼 소수점 있는 숫자, 즉 `float`가 됩니다. 파이썬에서 `/`는 항상 `float`를 반환합니다.

이제 파이썬만의 특별한 연산자 세 가지를 보겠습니다. 처음 보는 분들이 많을 텐데, 이게 오히려 실용적으로 자주 씁니다.

**첫 번째, 정수 나눗셈 `//`** — "몫만 주세요"

```python
print(10 // 3)   # 결과: 3
```

`//`은 나눗셈을 하되 **소수점 이하를 버리고 정수만** 반환합니다. 10을 3으로 나누면 몫이 3이고 나머지가 1인데, `//`는 그 몫인 3만 돌려줍니다.

비유를 들면, 빵 10개를 3명이 똑같이 나눠 가질 때 한 사람이 가져갈 수 있는 **온전한 빵의 수**가 `//`의 결과입니다. 쪼갠 빵은 카운트하지 않아요.

**두 번째, 나머지 `%`** — "나눈 뒤 남은 건 얼마?"

```python
print(10 % 3)    # 결과: 1
```

`%`는 나눗셈의 **나머지**를 반환합니다. 10을 3으로 나누면 나머지가 1입니다. 위 빵 비유를 이어가면, 3명에게 3개씩 나눠주고 나서 **남은 빵**이 바로 `%`의 결과입니다.

`%` 연산자는 생각보다 활용도가 높습니다. 예를 들어 어떤 숫자가 짝수인지 홀수인지 확인할 때 `숫자 % 2 == 0`이면 짝수, `== 1`이면 홀수라는 패턴을 씁니다. 4주차 조건문에서 이 패턴을 바로 쓸 겁니다.

**세 번째, 거듭제곱 `**`** — "제곱 계산"

```python
print(2 ** 3)    # 결과: 8  (2의 3제곱 = 2×2×2)
print(3 ** 2)    # 결과: 9  (3의 2제곱 = 3×3)
print(2 ** 10)   # 결과: 1024
```

`**`는 왼쪽 숫자를 오른쪽 숫자만큼 제곱합니다. 수학에서 `2³`으로 쓰는 걸 파이썬에서는 `2 ** 3`으로 씁니다.

---

일곱 가지 산술 연산자를 표로 정리해 드릴게요.

| 연산자 | 이름 | 예시 | 결과 |
|:------:|:------|:-----|:----:|
| `+` | 덧셈 | `10 + 3` | `13` |
| `-` | 뺄셈 | `10 - 3` | `7` |
| `*` | 곱셈 | `10 * 3` | `30` |
| `/` | 나눗셈 | `10 / 3` | `3.333...` |
| `//` | 정수 나눗셈 (몫) | `10 // 3` | `3` |
| `%` | 나머지 | `10 % 3` | `1` |
| `**` | 거듭제곱 | `2 ** 3` | `8` |

특히 `//`, `%`, `**`는 처음 보면 낯설지만, 이 세 가지가 실전 코드에서 굉장히 자주 등장합니다. 오늘 lab.ipynb에서 직접 실행해보면서 눈에 익혀두시기 바랍니다.

연산자를 조합해서 쓸 때는 수학 시간에 배운 **연산 순서**가 그대로 적용됩니다. 곱셈과 나눗셈이 덧셈과 뺄셈보다 먼저 계산됩니다. 그리고 거듭제곱 `**`은 곱셈보다도 먼저 계산됩니다.

```python
print(2 + 3 * 4)       # 14  (곱셈 먼저)
print((2 + 3) * 4)     # 20  (괄호 안 먼저)
print(2 ** 3 + 1)      # 9   (거듭제곱 먼저: 8 + 1)
print(10 - 6 / 2)      # 7.0 (나눗셈 먼저: 10 - 3.0)
```

헷갈릴 때는 **괄호를 적극적으로 사용**하세요. 괄호를 쓰면 계산 순서가 명확해지고 코드 읽기도 쉬워집니다. "굳이 괄호 안 써도 되는 거 아니야?"라고 생각할 수 있지만, 코드는 내가 아닌 다른 사람도 읽습니다. 내가 의도한 연산 순서를 명확히 표현하는 것이 좋은 코드 작성 습관입니다.

그리고 연산자 사이사이에 **공백을 하나씩 넣어주는 것**도 좋은 습관입니다. `10+3`보다 `10 + 3`이 읽기 훨씬 쉽습니다. 파이썬 공식 스타일 가이드(PEP 8)도 연산자 양쪽에 공백을 두라고 권장하고 있습니다. 지금부터 이 습관을 들여두면 나중에 긴 코드를 쓸 때도 가독성이 훨씬 좋아집니다.

섹션 2 잘 따라오셨나요? `str`, `int`, `float`, `bool` 네 가지 타입, `type()` 함수, 그리고 산술 연산자 7종까지 배웠습니다. 이제 마지막 섹션으로 넘어갑니다. 사용자와 대화하는 방법, 즉 **`input()` 함수**를 배울 시간입니다.

사용자로부터 값을 입력받는 함수가 바로 **`input()`** 입니다. 이 함수 하나를 이해하면, 우리가 만드는 프로그램이 훨씬 생동감 있어집니다. 지금까지는 강사가 코드에 값을 직접 써넣었지만, `input()`을 쓰면 이 프로그램을 실행하는 누구든 자신의 값을 넣을 수 있게 됩니다. 프로그램이 살아 숨쉬기 시작하는 순간이에요.

### input() 함수 기본 사용법

`input()` 함수의 사용법은 간단합니다. 하지만 이 함수를 제대로 쓰려면 단순히 문법만 외우는 게 아니라, 어떻게 동작하는지 흐름을 이해해야 합니다. 하나씩 같이 보겠습니다.

```python
name = input("이름을 입력하세요: ")
print("안녕하세요,", name)
```

이 코드를 실행하면 화면에 `이름을 입력하세요: ` 라는 메시지가 출력되고 파이썬이 잠깐 멈춥니다. 사용자가 값을 입력하고 엔터를 누를 때까지 기다리는 거예요. 사용자가 `홍길동`을 입력하고 엔터를 누르면, 그 값이 `name` 변수에 저장됩니다. 그리고 `print()`로 `안녕하세요, 홍길동`이 출력됩니다.

`input()` 괄호 안에 넣은 문자열은 **안내 메시지(프롬프트)**입니다. 사용자에게 "무엇을 입력해야 하는지" 알려주는 역할을 합니다. 이 메시지는 생략할 수도 있지만, 사용자 경험을 위해 항상 넣어주는 것이 좋은 습관입니다.

`input()` 함수가 실행되는 순서를 단계별로 살펴보면 이렇습니다. 이 흐름을 이해하면 나중에 복잡한 프로그램에서도 "아, 지금 여기서 입력을 기다리는구나"를 바로 알 수 있습니다. 첫째, `input()` 함수가 호출됩니다. 둘째, 괄호 안의 안내 메시지가 화면에 출력됩니다. 셋째, 파이썬이 사용자의 입력을 기다리며 **일시 정지**합니다. 넷째, 사용자가 무언가를 입력하고 Enter를 누릅니다. 다섯째, 입력한 내용이 **문자열로 변환**되어 `input()` 함수의 반환값이 됩니다. 여섯째, 그 반환값이 `=` 왼쪽의 변수에 저장됩니다.

이 흐름을 이해하면 코드를 읽을 때 훨씬 자연스럽게 읽힙니다. "아, 여기서 잠깐 멈추고 내 입력을 기다리는구나"라는 감각이 생기면 됩니다. Google Colab에서 실행할 때는 코드 셀 아래에 입력 창이 나타납니다. 거기에 값을 입력하고 Enter를 누르면 됩니다.

### 핵심! input()의 반환값은 항상 str

여기서 **절대로 잊으면 안 되는 규칙** 하나를 알려드리겠습니다.

**`input()`으로 받은 값은 항상 문자열(`str`)입니다.**

아무리 숫자처럼 생긴 값을 입력해도, `input()`은 그것을 **문자열로 저장**합니다. 이게 왜 중요한지 코드로 바로 확인해 볼게요.

```python
user_input = input("숫자를 입력하세요: ")
print(type(user_input))
```

`23`을 입력하고 엔터를 눌렀다고 가정하면, 출력 결과는 `<class 'str'>` 입니다. 숫자 `23`을 입력했지만 파이썬은 그걸 문자열 `"23"`으로 받아들입니다.

이 사실을 무시하고 계산을 시도하면 어떻게 될까요?

```python
num = input("숫자를 입력하세요: ")
result = num + 10
```

`5`를 입력했다면 `"5" + 10`이 되는데, 문자열 `"5"`와 숫자 `10`을 `+`로 연결하려 했으니 **TypeError**가 발생합니다. 이게 바로 섹션 1에서 예고했던 그 에러입니다.

그러면 어떻게 해결할까요? 바로 **타입 변환** 함수를 사용하면 됩니다.

---

### 타입 변환 함수 3종

파이썬에는 타입을 변환해주는 내장 함수가 있습니다. 오늘 배울 것은 `int()`, `float()`, `str()` 세 가지입니다.

**첫 번째, `int()` — 문자열이나 float을 정수로 변환**

```python
num_str = "42"
num_int = int(num_str)
print(num_int + 10)    # 결과: 52
print(type(num_int))   # <class 'int'>
```

문자열 `"42"`를 `int()`로 감싸면 정수 `42`가 됩니다. 이제 `10`을 더하면 `52`가 정상적으로 계산됩니다.

`input()`과 함께 사용하면 이렇게 씁니다.

```python
age = int(input("나이를 입력하세요: "))
print("내년에는", age + 1, "살이 됩니다.")
```

`input()`의 결과를 바로 `int()`로 감싸서 정수로 변환합니다. `20`을 입력하면 `내년에는 21 살이 됩니다.`가 출력됩니다.

**두 번째, `float()` — 문자열이나 int를 부동소수점으로 변환**

```python
height_str = "175.5"
height_float = float(height_str)
print(height_float + 2.0)    # 결과: 177.5
print(type(height_float))    # <class 'float'>
```

소수점이 포함된 숫자를 입력받을 때 사용합니다.

```python
score = float(input("점수를 입력하세요: "))
print("점수:", score)
```

**세 번째, `str()` — 정수나 float을 문자열로 변환**

이번엔 반대 방향입니다. 숫자를 문자열로 바꿔야 할 때가 있습니다. 예를 들어 숫자와 문자열을 `+`로 이어붙이고 싶을 때 씁니다.

```python
age = 20
message = "현재 나이는 " + str(age) + "살 입니다."
print(message)
```

실행 결과:
```
현재 나이는 20살 입니다.
```

`age`는 정수인데 `str(age)`로 변환했더니 문자열 `"20"`이 됩니다. 그래서 `+`로 이어붙일 수 있게 됩니다. 이렇게 하지 않고 `str()` 없이 `"현재 나이는 " + age + "살 입니다."`라고 쓰면 TypeError가 발생합니다.

변환 함수를 쓸 때 주의할 점이 하나 있습니다. 변환할 수 없는 값을 억지로 변환하려 하면 `ValueError`가 발생합니다.

```python
int("안녕하세요")    # ValueError! 문자 "안녕하세요"를 정수로 바꿀 수 없음
int("3.14")          # ValueError! 소수점이 있는 문자열을 int()로 바로 바꿀 수 없음
float("abc")         # ValueError! 영문자를 소수로 바꿀 수 없음
```

`int("3.14")`를 바꾸고 싶다면 `float("3.14")`를 먼저 하고 나서 `int(float("3.14"))`처럼 두 번 변환해야 합니다. 조금 복잡하게 느껴지지만, 이런 상황은 실제로 자주 만납니다. 오늘은 이런 경우가 있다는 것만 알아두세요.

타입 변환 세 가지를 표로 정리해 드릴게요.

| 함수 | 변환 방향 | 예시 | 결과 |
|:----:|:---------|:-----|:----:|
| `int()` | str/float → 정수 | `int("42")` | `42` |
| `float()` | str/int → 소수 | `float("3.14")` | `3.14` |
| `str()` | int/float → 문자열 | `str(100)` | `"100"` |

### 에러는 조언이다 — SyntaxError vs TypeError

마지막 주제입니다. 코딩을 배우다 보면 빨간 에러 메시지를 만나게 됩니다. 처음에는 이게 굉장히 당황스럽고 무섭게 느껴질 수 있어요. 그런데 저는 이 시간을 빌려 한 가지 인식을 바꿔드리고 싶습니다.

**에러 메시지는 파이썬이 여러분에게 보내는 안내문입니다.**

파이썬이 "이 부분에 문제가 있어요, 여기를 확인해 보세요"라고 정확히 알려주는 거예요. 에러가 났다는 게 여러분이 틀렸다는 뜻이 아닙니다. 모든 개발자가 매일 에러를 만납니다. 에러 메시지를 잘 읽는 능력이 좋은 개발자의 조건 중 하나입니다.

실제로 전문 개발자들도 새로운 라이브러리를 쓰거나 낯선 코드를 쓸 때는 에러를 자주 냅니다. "에러가 전혀 안 난다" = "코드를 전혀 안 짠다"와 거의 같은 말입니다. 에러를 만나는 것 자체가 코딩을 하고 있다는 증거입니다. 그러니 에러 화면을 볼 때마다 "파이썬이 나한테 뭔가 알려주려 한다"는 마음으로 읽어보세요. 에러 메시지 안에 해답이 거의 다 들어있습니다.

오늘은 가장 자주 만나게 될 두 가지 에러 타입을 소개합니다.

---

**SyntaxError — 문법이 잘못됐습니다**

`SyntaxError`는 파이썬의 문법 규칙을 어겼을 때 발생합니다. 코드를 실행하기도 전에, 파이썬이 코드를 읽는 단계에서 바로 발견합니다.

```python
print("안녕하세요"
```

이 코드를 실행하면 이런 에러가 납니다.

```
SyntaxError: '(' was never closed
```

괄호를 열었는데 닫지 않았습니다. 파이썬이 "열린 괄호가 닫힌 적이 없어요"라고 알려주는 거예요.

또 다른 예입니다.

```python
prit("안녕하세요")
```

`print`를 `prit`으로 오타를 냈습니다. 이 경우는 `NameError`가 납니다. `prit`이라는 이름의 함수가 없다는 뜻이에요.

`SyntaxError`의 특징은 **코드가 실행되기 전에 발생**한다는 겁니다. 즉, 코드의 어느 부분도 실행되지 않고 바로 에러가 나옵니다. 고치는 방법은 에러 메시지가 가리키는 줄의 코드를 다시 읽어보는 것입니다.

---

**TypeError — 타입이 맞지 않습니다**

`TypeError`는 **코드가 실행되는 도중에** 발생합니다. 문법은 맞는데 데이터 타입이 서로 맞지 않을 때 나옵니다.

```python
num = input("숫자 입력: ")   # "5" 를 입력했다고 가정
result = num + 3
```

이 코드의 문법 자체는 완벽합니다. 문제는 `num`이 `str`인데 `3`은 `int`라는 겁니다. 실행 시 `TypeError: can only concatenate str (not "int") to str`라는 메시지가 나옵니다.

해석하면 "문자열끼리만 이어붙일 수 있는데, 정수를 붙이려 했어요"입니다. 파이썬이 아주 친절하게 설명해주고 있죠? 해결 방법은 타입 변환 함수를 쓰면 됩니다.

```python
num = int(input("숫자 입력: "))
result = num + 3
print(result)   # 5 입력 시 결과: 8
```

`SyntaxError`와 `TypeError`의 차이가 명확하게 느껴지나요? 한 가지 더 예를 들어볼게요. 우리가 공사 현장에서 일하는 상황을 생각해 보겠습니다. 설계 도면에 오타가 있어서(문법 오류) 공사 시작 전에 발견된 경우, 이게 `SyntaxError`입니다. 공사를 시작했는데 콘크리트 자리에 모래를 붓는 잘못된 재료 사용(타입 오류)이 공사 도중 발견된 경우, 이게 `TypeError`입니다. 설계 단계 오류 vs 시공 단계 오류라고 생각하면 됩니다.

`SyntaxError`와 `TypeError`의 차이를 간단히 정리하면 이렇습니다.

| 에러 | 발생 시점 | 원인 | 해결 방법 |
|:-----|:--------:|:-----|:---------|
| `SyntaxError` | 실행 전 | 문법 오류 (괄호 누락, 오타 등) | 해당 줄 문법 재확인 |
| `TypeError` | 실행 중 | 타입 충돌 (str + int 등) | 타입 변환 함수 사용 |

에러를 읽는 실전 팁을 하나 드릴게요. 파이썬의 에러 메시지는 항상 맨 아래 줄이 핵심입니다. 에러가 났을 때 긴 붉은 텍스트가 쭉 나오면 당황하지 말고 **맨 마지막 줄**부터 읽으세요. 거기에 에러 종류(TypeError, SyntaxError 등)와 간단한 설명이 나옵니다. 그 다음에는 에러 메시지 위쪽에 적힌 "어느 줄에서 에러가 났는지"를 확인하면 됩니다. 화살표(`-->`) 나 `line N`이라고 표시된 부분이 에러가 발생한 줄입니다.

그리고 에러 메시지를 읽어도 이해가 안 될 때는 **AI를 활용**하세요. 에러 메시지를 그대로 복사해서 ChatGPT나 Gemini에게 "이 에러가 왜 났는지 설명해줘"라고 물어보면 됩니다. 단, AI가 알려준 해결책을 바로 복사하지 말고, 왜 그런 해결책인지 이해한 다음에 직접 타이핑해 보세요. 이것이 이번 주 마이크로 과제의 핵심이기도 합니다.

오늘 섹션 3에서 배운 것을 정리합니다. 이번 섹션에서 가장 중요한 한 가지만 꼽으라면 단연 **"`input()`의 반환값은 항상 `str`"** 입니다. 이것을 잊으면 계산이 필요한 프로그램을 만들 때마다 TypeError를 만납니다. 반대로 이것을 기억하면 `int(input(...))` 패턴을 자연스럽게 쓸 수 있게 됩니다.

`input()`은 사용자로부터 값을 입력받는 함수이고, 반환값은 **항상 `str`**입니다. 계산에 사용하려면 `int()` 또는 `float()`으로 변환해야 합니다. 반대로 숫자를 문자열에 이어붙이려면 `str()`으로 변환합니다. 그리고 에러를 만났을 때, `SyntaxError`는 문법을 고치고 `TypeError`는 타입 변환으로 해결합니다.

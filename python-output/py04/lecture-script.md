## [S1] 섹션 1: 인생은 선택의 연속, 파이썬의 갈림길 if문 (25분)

*(강사 참고: 카메라를 보며 친근하게 손인사를 하며 시작합니다.)*

안녕하세요! 서울신학대학교 1학년 학생 여러분, 그리고 멀리 해외에서 우리 강의를 수강하고 계시는 외국인 유학생 여러분! 여러분의 파이썬 마스터, 정원혁 교수입니다. 4주차 강의에 오신 것을 환영합니다!

우리가 지난 3주 동안 변수 상자도 만들어보고, 그 상자에 글자도 넣고 숫자도 넣어봤습니다. 아주 훌륭해요! 그런데 여러분, 지금까지 우리가 작성한 코드는 위에서부터 아래로 마치 폭포수가 떨어지듯이 한 번에 촤악~ 하고 무조건 실행되는 구조였습니다. 하지만 여러분의 일상을 한 번 생각해 볼까요? "비가 오면 우산을 쓰고, 비가 오지 않으면 우산을 안 쓴다." 이렇게 상황에 따라 행동이 달라지죠? 프로그래밍도 이와 똑같습니다! 우리의 프로그램이 지능을 가지려면 상황에 따라 스스로 다른 길을 선택할 수 있어야 합니다.

오늘은 파이썬에게 "이런 상황이면 이렇게 하고, 저런 상황이면 저렇게 해라!"라고 명령하는 마법의 분기점, **조건문(Conditional Statement)**을 배워보겠습니다. 

이 조건문의 가장 핵심이 되는 요술 단어는 바로 **`if`** 입니다. 영어로 "만약 ~라면"이라는 뜻이죠. 사용법은 아주 직관적입니다. 

화면을 보실까요? 파이썬 에디터에 다음과 같이 적어보겠습니다.
```python
score = 85
if score >= 80:
    print("합격입니다! 축하합니다!")
```
이 3줄짜리 코드 속에 오늘 배울 모든 철학이 다 들어가 있습니다. 
첫 줄에서 `score`라는 상자에 85를 넣습니다.
두 번째 줄, `if` 다음에는 반드시 **'참(True)이거나 거짓(False)으로 판명나는 질문'**이 와야 합니다. `score >= 80`은 "점수가 80보다 크거나 같니?" 라는 질문입니다. 맞으면 True, 틀리면 False가 되겠죠? 지금은 85니까 True입니다!
그리고 **가장 중요한 부분!** `if` 문장 끝에는 반드시 콜론(`:`)을 찍어야 합니다. 콜론은 파이썬에게 "자, 이제부터 진짜 명령어가 쏟아진다, 준비해!"라고 알려주는 신호탄입니다. 

그리고 세 번째 줄의 들여쓰기를 보세요! 파이썬은 스페이스 4칸(또는 탭 1번)으로 **들여쓰기(Indentation)**를 합니다. 이것은 파이썬만의 아주 아름다운 규칙입니다. 들여쓰기 된 부분은 "위의 if 조건이 참일 때만 실행되는 묶음"이라는 뜻입니다. 만약 들여쓰기를 하지 않으면? 파이썬은 에러를 내며 화를 냅니다. "어느 부분이 if에 속한 건지 모르겠어!" 라고 말이죠. 

그런데 여기서 질문! 만약 점수가 70점이라면 어떻게 될까요? 조건이 거짓(False)이 되어 파이썬은 들여쓰기 된 부분을 그냥 건너뜁니다. 즉, 아무 일도 일어나지 않겠죠. 

이때 등장하는 것이 바로 **`else`** 입니다! "그렇지 않으면~" 이라는 뜻이죠.
```python
score = 70
if score >= 80:
    print("합격입니다!")
else:
    print("아쉽네요, 다음 기회에!")
```
이제는 둘 중 하나는 무조건 실행되는 아주 멋진 이중 갈림길이 완성되었습니다! 

자, 이번엔 비교 연산자 6총사를 재빨리 알아보겠습니다. 두 값을 비교할 때 쓰는 기호들입니다. 
1. `>` (크다)
2. `<` (작다)
3. `>=` (크거나 같다)
4. `<=` (작거나 같다)
5. **`==` (정확히 같다!)**: 아주 중요합니다! 수학의 `=`은 파이썬에서 대입 연산자라고 말씀드렸죠? 양쪽 값이 똑같은지 물어볼 때는 반드시 등호 두 개 `==`를 써야 합니다! (예: `if age == 20:`)
6. **`!=` (다르다!)**: 앗, 느낌표가 붙으면 프로그래밍에서는 항상 부정(Not)을 뜻합니다. "같지 않다"라는 뜻이죠. 

만약 길이 2개가 아니라 3, 4개라면 어떡할까요? 예를 들어 90점 이상은 A, 80점 이상은 B, 그 이하는 C라고 하고 싶다면요? 이때는 `else`와 `if`를 퓨전시킨 **`elif`** 라는 요술봉을 씁니다!
```python
score = 85
if score >= 90:
    print("최우수 A등급")
elif score >= 80:
    print("우수 B등급")
else:
    print("조금 더 분발 C등급")
```
`if`부터 시작해서 위에서 아래로 차례대로 조건을 검사하다가, 하나라도 참(True)에 걸리면 그곳만 실행하고 나머지는 쳐다보지도 않고 끝을 맺습니다. 아주 현명하죠! 

자, 오늘 첫 번째 영상에서는 파이썬이 스스로 생각해서 길을 선택하게 하는 마법의 `if-elif-else`를 배웠습니다. 들여쓰기 4칸의 마법, 잊지 마세요! 숨 돌리시고, 다음 영상에서는 이 조건문을 두 세 배 더 복잡하고 똑똑하게 엮어내는 논리 연산자에 대해 배워보겠습니다. 다음 시간에 만나요!

## [S2] 섹션 2: 조건의 마법사, 논리 연산자의 세계 (25분)

*(강사 참고: 살짝 속도감을 내어 활기차게 시작합니다.)*

여러분 환영합니다! 4주차 두 번째 섹션, **논리 연산자**의 시간입니다. 정원혁 교수입니다. 

자, 앞선 강의에서 우리는 "점수가 80점 이상일 때"라는 단 하나의 조건문을 썼습니다. 하지만 우리가 사는 현실 세계는 그렇게 단순하지 않죠. 놀이공원 롤러코스터에 탈 수 있는 조건을 생각해 보세요. 
"키가 130cm 이상이어야 하고, **동시에** 심장 질환이 없어야 한다!" 
조건이 무려 두 개나 묶여 있죠? 이럴 때 파이썬에서 사용하는 가장 강력한 접착제가 바로 **논리 연산자(Logical Operators)** 입니다. 

파이썬에는 아름다운 영단어 3인방, **`and`**, **`or`**, **`not`**이 존재합니다. 

첫 번째, **`and` (그리고)** 입니다. 
`and`는 아주 엄격한 문지기입니다. 왼쪽 조건과 오른쪽 조건이 **모두 다 참(True)** 이어야만 통과시켜 줍니다.
```python
height = 135
has_disease = False

if height >= 130 and has_disease == False:
    print("롤러코스터 탑승을 환영합니다!")
else:
    print("죄송하지만 탑승하실 수 없습니다.")
```
어떻습니까? 키도 만족하고 심장병도 없어야만 참(True)이 되어 문을 열어줍니다. 둘 중 하나라도 틀리면 칼같이 `else`로 튕겨냅니다. 아주 보수적이죠! 

두 번째는 인심이 후한 아저씨 같은 **`or` (또는)** 입니다.
`or`는 "너네 둘 중에 하나만 맞으면 나 무조건 통과시켜 줄게!" 라는 너그러운 연산자입니다. 
마트에서 할인을 해주는 상황을 생각해 볼까요. "당신이 VIP회원이거나(**or**) 구매 금액이 10만원이 넘으면 무조건 할인해 드릴게요!" 라는 조건에 딱 맞습니다.
```python
is_vip = False
purchase_amount = 120000

if is_vip == True or purchase_amount >= 100000:
    print("10% 파격 할인을 해드립니다!")
```
VIP 회원이 아니더라도(`False`), 금액이 12만원이니까(`True`), 둘 중 하나가 참이므로 여지없이 할인을 해줍니다. 정말 멋지죠?

세 번째는 청개구리 요정 **`not` (부정)** 입니다.
이 녀석은 참을 거짓으로, 거짓을 참으로 완전히 반대로 뒤집어 버립니다. 
"만약 회원이 **아니면** 가입 창을 띄워라!" 라고 할 때 유용하게 씁니다.
```python
is_member = False
if not is_member:
    print("신규 회원 가입을 진행하시겠습니까?")
```
`is_member`가 `False`인데 앞에 `not`이 붙으니까 `True`로 반전되어서 화면에 출력문이 짠 하고 나타나게 됩니다.

여러분, 이 연산자들이 어렵게 느껴지시나요? 그렇다면 진리표라는 걸 떠올리시면 됩니다. 
`and`는 깐깐한 경비원이라 카드 2개가 다 있어야 문을 열어주고, `or`는 착한 경비원이라 카드 1개만 있어도 문을 열어준다고 기억하시면 평생 잊어버릴 일이 없습니다. 

추가로, **중첩 if문(Nested if)**도 짧게 짚고 넘어갈까요? `if`문 안쪽에 또 다른 `if`문을 넣을 수도 있습니다! 마치 마트료시카 인형처럼 말이죠!
```python
age = 25
if age >= 18:
    print("성인 구역에 오신 것을 환영합니다.")
    # 이 안에서 또 if문 시작
    if age >= 20: 
        print("맥주를 주문하실 수 있습니다!")
```
들여쓰기 블록 안에서 또 들여쓰기를 해서 한 번 더 조건을 묻는 구조입니다. 파이썬은 들여쓰기가 얼마나 깊어지냐에 따라 블록의 깊이를 파악하므로, 스페이스바 관리가 생명입니다!

자, 우리는 파이썬의 조건이 어떻게 동작하는지 완벽하게 파악했습니다. 두 번째 섹션은 여기서 마칩니다. 잠시 쉬었다가 마지막 3섹션에서 우리는 이 지식을 총동원하여 콜랩에서 여러분만의 **학점 계산기**를 라이브로 만들어 보도록 하겠습니다! 다음 시간에 뵙겠습니다!

## [S3] 섹션 3: 라이브 코딩 학점 계산기와 AI 협업 (25분)

*(강사 참고: 화면 우측 하단에 얼굴을 띄우고, 모니터에는 Colab 화면과 AI 챗봇 화면이 반반 띄워져 있습니다.)*

자, 여러분 수고 많으셨습니다. 4주차의 대미를 장식할 하이라이트, 섹션 3 라이브 코딩 시간입니다!
여러분도 브라우저에서 Colab을 여시고 저를 따라 작성해 보세요.

우리의 목표는 다음과 같습니다. "사용자에게 점수를 물어보고, 그 점수를 바탕으로 A, B, C, F 등급을 화면에 이쁘게 출력하는 학점 계산기"를 만드는 것입니다. 간단하죠? 시작합니다. 

*(타이핑 소리 - 강사 라이브 코딩 시작)*

먼저 지난 시간에 배운 `input()`으로 입력을 받겠습니다.
```python
score_input = input("학생의 점수를 입력하세요 (0~100): ")
```
여기서 잠깐! `input()`으로 받은 것은 글자 형식이므로 계산을 위해 숫자로 바꿔주는 마법 지팡이 `int()`를 씌우겠습니다.
```python
score = int(score_input)
```

이제 오늘 배운 `if-elif-else`를 본격적으로 써봅니다.
```python
if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "F"

print(f"당신의 점수는 {score}점이며, 학점은 {grade}입니다!")
```
자, 실행해 봅니다. 85를 넣으니 "당신의 점수는 85점이며, 학점은 B입니다!" 완벽하죠! 

그런데 여기서 돌발 상황! 만약 사용자가 실수로 점수에 `150`을 넣으면 어떻게 되죠? 파이썬은 90보다 크니까 그냥 "A"를 출력해 버리겠죠? 150점짜리 시험은 없는데 말입니다! 

이 대목에서 우리의 영원한 파트너 AI를 소환하겠습니다! 
제가 오른쪽 화면에 있는 AI에게 직접 물어보겠습니다.
**(강사가 AI에게 말하듯 타이핑)**: "내가 python으로 학점 계산기를 만들었는데, 점수가 0점보다 작거나 100점보다 크면 '잘못된 입력입니다' 라고 경고를 띄우고 싶어. 위 코드를 어떻게 고쳐야 할까?"

*(AI의 답변 로딩 중...)*
자, AI가 엄청난 속도로 정답을 내놓았습니다. 논리 연산자 `or`를 사용하라고 추천해 주네요. AI가 짜준 코드를 적용해 보겠습니다. 
```python
if score < 0 or score > 100:
    print("경고: 점수는 0에서 100 사이여야 합니다!!")
elif score >= 90:
    grade = "A"
    print(f"학점은 {grade}입니다!")
# ... (중략)
```
와우! 보셨습니까? AI는 우리가 생각지 못한 허점을 메꾸는 최고의 보조자입니다. 우리는 아이디어를 내고, AI는 코드를 강화합니다.

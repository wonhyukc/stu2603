지난 3주 동안 우리는 변수 상자를 만들고, 그 상자 안에 글자나 숫자를 넣는 기본적인 과정들을 배웠습니다. 코드를 위에서부터 아래로 차례대로 실행하는 순차적인 흐름을 이해하셨죠? 마치 폭포수가 위에서 아래로 떨어지듯, 작성된 코드는 무조건 순서대로 작동했습니다. 

하지만, 세상을 살아가면서 항상 정해진 순서대로만 행동할 수는 없습니다. "비가 오면 우산을 쓰고, 비가 오지 않으면 우산을 안 쓴다." 혹은 "지하철이 3분 이내로 남았으면 뛰고, 10분 남았으면 천천히 걸어간다." 이렇게 상황에 따라 행동 지침이 달라져야 합니다. 프로그램도 마찬가지로 지능을 가지려면 이러한 선택을 할 줄 알아야 합니다! 오늘은 프로그램에게 어떤 조건에서는 이 길을, 다른 조건에서는 저 길을 가도록 명령하는 마법의 분기점, **조건문(Conditional Statement)**에 대해 깊이 있게 파헤쳐 보겠습니다.

### 1-1. 단순 if문: 가장 기초적인 조건 확인
조건문의 핵심이 되는 단어는 바로 **`if`** 입니다. 영어로 "만약 ~라면"이라는 의미를 담고 있죠. 파이썬 문법에서도 이 뜻을 그대로 적용합니다.

우선, 현실 세계의 **실생활 비유 첫 번째: 놀이기구 키 제한**을 생각해보겠습니다. 놀이공원의 범퍼카 코너에 가보면 줄 자가 세워져 있고 "키 130cm 이상만 입장 가능!"이라는 푯말이 있습니다. 직원은 어떻게 할까요? 키를 재서 130 이상이면 들여보내고, 아니면 미안하지만 그냥 통과시킵니다. 

파이썬 에디터를 켜보세요. 이 상황을 코드로 써보겠습니다.
```python
height = 135
if height >= 130:
    print("범퍼카 탑승 가능! 즐거운 시간 되세요!")
```
이 두 줄의 코드 속에 오늘 배울 모든 철학이 담겨 있습니다.
- 첫째, `if` 뒤에는 반드시 **'참(True)이거나 거짓(False)으로 대답할 수 있는 질문'**이 와야 합니다. 파이썬은 모호한 질문을 싫어합니다. "키가 큰 편이니?"라는 질문 대신 "키가 130 이상이니?"라고 명확히 물어보아야 합니다.
- 둘째, 줄 끝에 있는 콜론(`:`)입니다. 콜론은 여러분이 문을 여는 손잡이와 같습니다. 파이썬에게 "이 조건이 참이라면, 아래 열리는 문 안쪽의 내용들을 실행해!"라고 알리는 신호탄입니다.
- 셋째, **들여쓰기(Indentation)**입니다! 콜론 다음 줄은 탭(Tab)키나 스페이스바 4번을 눌러 안으로 들여써야 합니다. 이는 묶음(Block)을 의미하며 들여쓰기 된 부분은 오직 위 조건이 참일 때만 실행됩니다.

**실생활 비유 두 번째: 현금 지급기(ATM)**를 볼까요? 여러분이 통장에서 돈을 뽑을 때 잔액 검사를 합니다.
```python
balance = 50000
withdraw_amount = 30000

if balance >= withdraw_amount:
    print("현금을 인출합니다. 잠시만 기다려주세요.")
    balance -= withdraw_amount
    print("남은 잔액은", balance, "원입니다.")
```
이 코드에서 볼 수 있듯이 들여쓰기 된 코드가 세 줄이나 있죠? 이 세 줄은 하나의 운명 공동체가 되어 `if` 조건이 참일 때 한 덩어리로 동시에 실행됩니다.

### 1-2. if와 else: 양자택일의 길목
그런데 만약 놀이기구에 키가 120cm인 아이가 왔거나 잔액이 부족하면 어떻게 될까요? 위 코드에서는 들여쓰기를 건너뛰어버리기 때문에 아무 일도 없고 아무 메시지도 나오지 않습니다. 사용자는 프로그램이 멈췄나? 하고 당황할 수 있습니다. 

그래서 우리는 **`else`**라는 도구를 사용합니다. "만약 조건이 참이면 이거 하고, **그렇지 않으면** 저거 해!"라는 양자택일을 만드는 것이죠.

```python
score = 65
if score >= 60:
    print("축하합니다! 합격입니다.")
else:
    print("아쉽습니다. 재시험 대상자입니다.")
```
`else`에는 조건식을 적을 필요가 없습니다. `if`에서 던진 질문이 아니기만 하면 전부 다 받아주는 일종의 '나머지 바구니'이기 때문입니다.

### 1-3. 6개의 비교 연산자 (Comparison Operators)
조건문에서 빠질 수 없는 무기, 비교 연산자 6총사를 철저히 짚어봅시다.
1. `>` (크다 / 초과)
2. `<` (작다 / 미만)
3. `>=` (크거나 같다 / 이상)
4. `<=` (작거나 같다 / 이하)
5. **`==` (정확히 같다!)**: 매우 중요합니다! 프로그래밍 세계에서 `=` 기호 하나는 "오른쪽 값을 왼쪽에 넣어라(할당)"라는 아주 거만한 명령문입니다. 양쪽이 똑같은지 겸손하게 질문하고 싶다면 반드시 무조건 등호 2개인 `==`를 써야 합니다. (예: `password == "1234"`)
6. **`!=` (다르다!)**: 파이썬에서 느낌표 `!`는 부정(Not)을 상징합니다. 그러므로 `!=`는 "같지 않다"가 됩니다.

### 1-4. elif: 3가지, 4가지 길을 내고 싶을 때
현실은 항상 둘 중 하나(합격/불합격)로 나뉘지 않습니다. 
**실생활 비유 첫 번째: 고속도로 톨게이트 요금**을 상상해 보시죠. 경차는 50%, 일반 승용차는 100%, 화물차는 120% 요금을 받습니다. 이러한 다중 조건을 처리할 때는 어떻게 할까요? `else if`를 줄여 만든 **`elif`** 패스워드를 씁니다.

```python
car_type = "경차"

if car_type == "경차":
    print("통행료 50% 할인 대상입니다.")
elif car_type == "승용차":
    print("표준 통행료가 적용됩니다.")
elif car_type == "화물차":
    print("통행료 20% 할증이 적용됩니다.")
else:
    print("알 수 없는 차종입니다. 안내소로 문의하세요.")
```
`elif`의 가장 아름다운 점은 **계단식 검사**라는 것입니다. 첫 번째 `if`가 거짓이면 두 번째 `elif`로 가고, 그것도 거짓이면 세 번째로 넘어갑니다. 하지만 중간에 어느 하나라도 참(True)에 걸리면, 그 즉시 그 블록을 실행하고 아래에 있는 나머지 조건들은 쳐다보지도 않고 전체 `if-elif` 덩어리를 탈출합니다!

**실생활 비유 두 번째: 배달 앱 최소 주문 금액 검사**
```python
order_amount = 12000

if order_amount < 10000:
    print("최소 주문 금액 10,000원을 넘겨야 배달이 가능합니다.")
elif order_amount < 30000:
    print("배달비 3,000원이 추가됩니다.")
else:
    print("30,000원 이상 무표 배달 혜택이 적용됩니다!")
```
위 코드를 보세요. 숫자가 작을 때부터 차근차근 검사하고 있습니다. 파이썬은 여러분이 순서대로 꼼꼼히 로직을 짰기 때문에, 만약 45,000원을 주문했다면 마지막 `else` 구문까지 안전하게 흘러들어갈 것입니다. 만약 순서를 뒤죽박죽으로 짠다면 심각한 버그가 발생할 수 있습니다. 예를 들어 90점 이상 A, 80점 이상 B를 체크할 때, 만약 `if score >= 80:`부터 실수로 먼저 검사해 버리면 95점인 학생도 "B등급"에 갇혀 버리고 맙니다! 이 부분은 나중에 실습 시간에 여러분이 직접 디버깅하면서 뼈저리게 느끼게 될 것입니다.

첫 번째 섹션은 파이썬이 스스로 생각해서 길을 선택하게 하는 마법, 기초 조건문에 대해 배웠습니다. 잠시 쉬었다가 섹션 2에서 뵙겠습니다!

## [S2] 섹션 2: 조건의 마법사, 논리 연산자와 중첩 if의 세계 (25분)

*(활기차게 복귀하며 시작합니다.)*

여러분의 두뇌는 준비되셨나요? 4주차 두 번째 섹션, 더욱 심오한 **논리 연산자(Logical Operators)와 중첩 조건문**의 시간입니다. 정원혁 교수입니다. 

섹션 1에서 우리는 한 번에 한 가지 조건만을 물어보았습니다. 단면적인 질문이죠. "점수가 80점 이상인가?"
하지만 우리 주변의 복잡한 사회 시스템은 보통 두 가지 이상의 조건을 복합적으로 요구합니다.

### 2-1. And와 Or: 엄격함과 너그러움의 상징

파이썬에는 아름다운 세 가지 논리 영단어, **`and`**, **`or`**, **`not`**이 존재합니다. 

첫 번째, **`and` (그리고)** 입니다. 
`and`는 아주 깐깐하고 엄격한 문지기입니다. 왼쪽 조건과 오른쪽 조건이 **모두 다 참(True)** 이어야만 통과시켜 줍니다.

**실생활 비유 첫 번째: 영화관 청소년 관람불가 등급 확인**
영화를 보려면 나이가 성인이어야 하고, 그리고 동시에 신분증을 지참해야 한다고 생각해봅시다. 둘 중 하나라도 모자라면 매표소 직원은 단호하게 표를 팔지 않습니다!
```python
age = 20
has_id_card = True

if age >= 19 and has_id_card == True:  # (has_id_card만 써도 됩니다)
    print("입장을 허가합니다. 즐거운 관람 되십시오.")
else:
    print("나이 조건 미달이거나 신분증을 지참하지 않으셨습니다. 입장 불가!")
```
이 깐깐한 `and` 연산자는 어느 한쪽이라도 거짓(False)이 끼어있으면 전체 결과를 얄짤없이 거짓으로 만들어버립니다. 

두 번째는 인심이 후한 아저씨 같은 **`or` (또는)** 입니다.
`or`는 "너네 둘 중에 하나만 맞으면 나 무조건 통과시켜 줄게!" 라는 너그러운 연산자입니다. 

**실생활 비유 두 번째: 주차 톨게이트 무인정산기 할인 로직**
여러분이 마트에서 쇼핑을 마치고 나가려 합니다. "저희 마트 VIP 회원이시거나 OR 마트에서 5만원 이상 구매하신 영수증이 있으시면 언제든지 1시간 무료 주차를 드립니다!" 
```python
is_vip_member = False
shopping_amount = 65000

if is_vip_member or shopping_amount >= 50000:
    print("고객님, 1시간 주차 요금 무료 처리되었습니다.")
else:
    print("요금 3,000원을 결제해 주세요.")
```
고객이 VIP 회원이 아니더라도(`False`이지만) 오늘 65,000원을 구매했으므로 오른쪽 조건이 `True`가 되어, 마음씨 좋은 `or` 연산자는 고객을 무료로 출차시켜 줍니다. 얼마나 자애로운가요?

### 2-2. not 연산자: 청개구리 거울의 마법
세 번째는 바로 청개구리 요정 **`not` (부정)** 입니다.
이 녀석은 참을 거짓으로, 거짓을 참으로 완전히 반대로 뒤집어 버립니다. 조건식 앞에 무심하게 툭 얹어 놓기만 하면 작동합니다.

**실생활 비유: 게임 아이템 장착 여부 검사**
게임에서 특정 방패를 장착하지 **않은** 사람만 데미지를 100 입도록 설계해 볼까요.
```python
has_shield = False

if not has_shield:
    print("치명타를 입었습니다! HP가 100 깎입니다!")
```
방패가 없으니 `has_shield`는 `False`인데, 앞에 `not`이 붙으니까 마법처럼 반전되어 전체 식이 `True`가 되면서 피해를 주는 코드 블록이 무섭게 실행됩니다. 

### 2-3. 논리 연산자 우선순위와 괄호의 미학
우리가 `+`와 `*`가 섞인 수학 연산을 할 때 `*`를 먼저 계산하듯, 논리 연산자에도 엄격한 조폭 같은 계급사회가 존재합니다. **우선순위는 반드시 `not` -> `and` -> `or` 순서**입니다.

만약 우선순위를 고려하지 않는다면 여러분의 시스템은 재앙을 맞이할 것입니다. 
예를 들어 `A or B and C` 라고 코드를 쓰면, 파이썬은 암묵적으로 `B and C`를 한 덩어리로 먼저 묶고 계산한 뒤 나중에 `A or`를 처리합니다! 개발자의 원래 의도가 `(A or B) and C` 였다면 어떨까요? 완전히 다른 엉뚱한 결과가 나오겠죠.
**결론: 복합 조건문을 작성할 때는 자신이 없더라도 반드시 소괄호 `()`를 쳐서 의미를 명확히 해야 합니다.** 나 자신을 위해서, 그리고 코드를 읽을 동료를 위해서 괄호를 남발하세요. 절대 부끄러운 일이 아닙니다.

### 2-4. 중첩 if문 (Nested Conditional Statements)
조건문 안에 또 다른 조건문이 들어갈 수도 있을까요? 그렇습니다! 러시아의 마트료시카 인형을 까보면 작은 인형이 계속 나오듯 `if` 블록 안에 들여쓰기를 더 깊게 하여 들어가는 **중첩 if문** 구조가 아주 자주 쓰입니다.

**실생활 비유: 온라인 결제 비밀번호 인증 시스템**
먼저 아이디가 맞는지 1차 관문을 통과해야 하고, 맞다면 그 방 안으로 들어가서 다시 2차 관문인 비밀번호 일치 여부를 묻는 구조입니다.

```python
username = "admin"
password_input = "123456"

if username == "admin":
    print("관리자 계정이 확인되었습니다. 1차 관문 통과.")
    # 이제 이 안에서 2차 검사 시작! (들여쓰기가 한 번 더 파고 들어감)
    if password_input == "1234":
        print("비밀번호 일치! 시스템에 접속합니다.")
    else:
        print("비밀번호가 틀렸습니다. 경고!")
else:
    print("등록되지 않은 사용자입니다.")
```
이 방식의 장점은 문제의 원인을 다각적으로 분석하여 에러 메시지를 각각 상세하게 뿌려줄 수 있다는 것입니다. `and` 연산자로 한 줄에 쓰면 "아이디 또는 비밀번호가 틀렸습니다" 와 같이 뭉뚱그려져야 하지만 중첩 `if`를 쓰면 논리의 깊이를 가지게 됩니다.

## [S3] 섹션 3: 예외 방어가 포함된 완벽한 학점 계산기 라이브 코딩 및 AI 디버깅 협업 (25분)

*(화면 우측 하단에 강사 얼굴을 띄우고, 모니터에는 Colab 화면을 크게 띄워 놓습니다. 필요시 단축키를 이용 화면 전환을 합니다.)*

수고 많으셨습니다. 이번 4주차 조건문의 대미를 장식할 하이라이트, 섹션 3의 풀 사이즈 라이브 코딩 시간입니다.
단순 구문 연습을 넘어 현업 소프트웨어처럼 "사용자의 미친 짓(?)을 막아내는" **예외 방어(Validation 방어벽)** 로직을 포함한 강건한 코드를 짜보도록 하겠습니다.

우리의 목표는 다음과 같습니다. 
> "사용자에게 키보드로 점수를 입력받고, 0점에서 100점 사이 밖의 값이 들어오면 즉각 경고창을 뿜어 실행을 중지하고, 올바른 값일 경우에만 A, B, C 학점을 세세하게 판별하여 결과를 안내하는 학점 계산기"

시작합니다! *(타이핑 소리 시작)*

### Step 1. 입력과 형변환의 벽
과거에 배운 `input()` 함수가 무대 위로 올라옵니다.
```python
raw_input = input("이번 기말고사 학생의 점수를 입력하세요 (0~100 범위 내): ")
```
학생이 콘솔 창에 92를 입력했다고 해봅시다. 그런데 `input()`이 받아들인 "92"는 숫자가 아니라 문자열 'str' 타입이라는 것을 잊지 않으셨죠? 숫자 92와 글자 "92"는 다릅니다. 조건문에서 등급을 매기려면 크기 비교 연산(`>=`)을 해야 하는데, 글자와 숫자를 비교하면 파이썬은 즉각 사자처럼 포효하며 TypeError를 내뿜습니다.

이를 고치기 위해 정수 변환 마법사 `int()`를 뒤집어씌웁니다.
```python
score = int(raw_input)
print(f"변환 완료! 현재 score의 데이터 타입은 {type(score)} 입니다.") 
```
이걸로 입력 준비는 끝났습니다!

### Step 2. 단일 조건 분기 시뮬레이션
이제 90점 이상이면 A, 80점 이상이면 B를 주도록 `if-elif-else`를 작성해보겠습니다.
```python
if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "F"

print(f"해당 학생의 최종 학점은 '{grade}' 입니다!")
```
실행해 봅시다. 제가 `85`를 넣으니 B등급이 스무스하게 나옵니다.
여기서 방심하면 아마추어입니다!

### Step 3. Edge Case (돌발 시나리오) - 파괴적 테스트
돌발 테스트(Edge Case)를 시도해보죠. 사용자가 술에 취해 점수에 **"-50"** 이나 **"8500"** 을 입력하면 어찌 될까요?
실행시키고 `8500`을 입력해봤습니다. 
출력결과: `해당 학생의 최종 학점은 'A' 입니다!` 
이럴 수가! 100점 만점인데 8500점 맞았다고 A를 선사하는 멍청한 계산기가 탄생했습니다.

### Step 4. 방어 로직 (Validation) 추가 및 AI 협업
우리는 이 오류를 막는 검문소를 맨 위에 세울 것입니다. 이때 앞서 배운 논리 연산자 `or`가 강력한 무기가 됩니다. 제가 AI에게 어떻게 질문하는지 보세요.

**(강사가 AI 화면에 타이핑)**: "파이썬 학점 계산기를 만들었는데, 사용자가 score 변수에 0점 미만 값이나 100점 초과 값을 넣는 걸 원천 차단하는 방어막 코드를 or 연산자를 써서 짜줘."

*(AI 챗봇 로딩)*
훌륭합니다. AI 선생님께서 즉각 코드를 생성해 주셨네요! 통합해 보겠습니다.
```python
# 최종 버전
raw_input = input("점수를 입력하세요: ")
score = int(raw_input)

# Step 1. 검문소 (Validation 방어)
if score < 0 or score > 100:
    print("⚠ 치명적 오류: 점수는 사람이 맞을 수 있는 0점에서 100점 사이여야 합니다!!!")
else:
    # Step 2. 통과된 안전한 점수만 정밀 심사 (중첩 구조)
    print("점수가 정상 범위 내에 있습니다. 심사를 시작합니다...")
    if score >= 90:
        grade = "A"
    elif score >= 80:
        grade = "B"
    elif score >= 70:
        grade = "C"
    else:
        grade = "F"
        
    print(f"최종 판독 결과: {score}점 획득으로 '{grade}' 학점을 받으셨습니다.")
```
코드를 보세요. 바깥쪽에 튼튼한 성곽(Validation)을 치고, 그 성곽 안쪽 들여쓰기 공간(중첩 if) 안에서만 학점 계산 로직이 아늑히 작동하고 있습니다. 여러분도 오류 없는 안전한 웹 사이트를 경험해 보셨듯이, 현업의 서버 프로그래머들은 이런식으로 '경계값(Edge)'을 집요하게 검사하여 코드가 절대 무너지지 않도록 만듭니다!

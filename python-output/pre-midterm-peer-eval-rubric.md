## 1. 평가 목적 및 대상 (5~7주차 Pre-Midterm 전용)

1~8주차는 학생들의 **"디지털 스마트 도구 실무 활용 능력(번역, 자료 조사, 폼 활용, 클리닝 등)"**을 집중 육성하는 기간입니다. 파이썬 문법이나 코드를 다루지 않으므로, 이 기간 동안의 결과물 수집 및 동료 평가는 오직 "지정된 목표를 실제 소프트웨어/웹으로 달성하였는가"에만 집중합니다.

## 2. 채점 기준 (각 1점, 총 3점 만점)

평가자는 동료 학생이 제출한 링크(구글 드라이브나 폼) 혹은 첨부 파일(PDF, 스크린샷 이미지) 등을 열고 아래 세 개 문항에 대해 **Yes(1점) / No(0점)** 로 단순명료하게 채점합니다.

| 번호 | 단답형 객관적 평가 기준 (Yes/No) | 배점 | 확인 포인트 |
|:---:|:---|:---:|:---|
| **1** | **기한 내 제출**<br>과제를 기한 내에 지각하지 않고 제출하였습니까? | **5점** | 구글 폼, eCampus 등의 제출 기록이 마감 기한을 넘기지 않았는지 확인합니다. 지각 시 0점입니다. |
| **2** | 미션에서 요구한 **올바른 형태의 자료(PDF 묶음 파일, 구글 폼 링크, 정제된 엑셀 캡처 이미지 등)**가 eCampus에 잘 첨부 또는 링크되었는가? | **1점** | 결과물이 누락되지 않고 제자리에 있는지 시각적으로 확인합니다. (GitHub 링크 제출이 아님에 유의) |
| **3** | 제출된 파일(또는 링크)을 클릭/다운로드 했을 때, 권한 오류나 손상 없이 **정상적으로 내용이 열리고 화면에 보이는가**? | **1점** | 구글 문서의 경우 "권한 요청" 에러가 뜨지 않는지, 이미지의 글씨가 깨지지 않고 식별 가능한지 확인합니다. |
| **4** | 결과물의 내용이, 해당 주차가 요구하는 **최종 목표(예: 외국어에서 번역된 흔적, 쓸모없는 응답이 제거된 깔끔한 데이터 엑셀 화면 등)**를 물리적으로 증명하고 있는가? | **1점** | 얼마나 잘했는지 정성적으로 평가하지 말고, "번역된 최종 글씨가 있다", "데이터가 묶여 있다" 정도의 달성 여부만 가볍게 확인합니다. |

## 3. 평가자(피드백 작성자) 특별 가이드라인

- **코딩 관련 기준 제외**: 1~8주차에서는 코딩 개념을 전혀 묻지 않습니다. 노트북 파일(`.ipynb`)이나 코드 존재 여부를 물어선 안 됩니다.
- **객관적 결과 중심**: 번역체가 완벽한지, 설문지가 창의적인지 여부로 감점하지 마십시오. 요구받은 미션 결과 파일이 만들어져 권한 제한 없이 볼 수 있다면 감점 없이 **만점(3점)**을 주어야 합니다.
- 짧은 피드백 코멘트는 긍정적이고 실질적으로 작성합니다. (예: "결과물이 한글로 잘 번역되어 보기 편하네요. 링크도 바로 잘 열립니다!")

## 1. 기적의 핵심 도구 요약표 (컨닝 페이퍼)

### [필수 도구 링크]
- **구글 번역(Google Translate)**: [translate.google.com](https://translate.google.com/) (복사한 텍스트를 즉시 번역)
- **구글 렌즈(Google Lens)**: 스마트폰 스토어에서 다운로드 (복사가 안 되는 에러창 사진을 찍어 화면 위에서 직접 번역)
- **구글 검색(Google Search)**: [google.com](https://google.com) (번역된 의미를 넣고 '해결법' 등 키워드 조합)

### [대학 생활 4년을 책임질 단축키 5종]

| 단축키 (Windows 기준) | 기능 | 생존 의미 |
|:---|:---|:---|
| **`Ctrl + Z`** | 실행 취소 (Undo) | 방금 지워버린 글, 이동시킨 파일을 직전 상태로 완벽히 되돌립니다. 인생의 세이브 포인트입니다. |
| **`Ctrl + C`** | 복사하기 (Copy) | 선택한 텍스트/파일을 클립보드에 복사합니다. 원본은 그대로 유지됩니다. |
| **`Ctrl + V`** | 붙여넣기 (Paste) | 복사한 것을 커서 위치에 즉시 붙여넣습니다. |
| **`Ctrl + A`** | 전체 선택 (Select All) | 문서 전체 또는 폴더 내 모든 파일을 한 번에 선택합니다. |
| **`Ctrl + S`** | 저장 (Save) | 작업 중인 파일을 저장합니다. 10분에 한 번씩 습관적으로 누르세요. |

Mac 사용자는 `Ctrl` 대신 `Command(Cmd)` 키를 사용합니다.

### [에러 직독직해 3단계 플로우]

| 단계 | 텍스트 복사 가능할 때 | 텍스트 복사 불가능할 때 |
|:---|:---|:---|
| 1단계 | 에러창 텍스트를 마우스 드래그 후 `Ctrl+C` | 구글 렌즈 앱 켜서 모니터 화면 비추기 |
| 2단계 | [translate.google.com](https://translate.google.com/)에 `Ctrl+V` 붙여넣기 | 렌즈 앱에서 '번역' 모드 선택 |
| 3단계 | 번역 결과 + "해결법" 키워드로 구글 검색 | 번역된 내용 + "해결법" 키워드로 구글 검색 |

## 2. 실습 (조작) 워크플로우

오늘 실습의 목표는 **최신 AI 도구(Gemini)를 능동적으로 활용하여 낯선 개념(GitHub)을 학습하고, 그 내용을 실제 개발자들의 글로벌 플랫폼인 GitHub에 기록하여 제출하는 과정을 체득하는 것**입니다. 아래 워크플로우에 따라 [Task A] 제출을 위한 실습을 진행하십시오.

1. **단축키 구글 문서 만들기**: **그동안 내가 모르던 새로운 윈도우 단축키 5개 이상, 크롬 브라우저 단축키 5개 이상**을 찾아 구글 문서(Google Docs)로 작성합니다. 우측 상단 공유를 눌러 **'링크가 있는 모든 사용자'가 '읽기' 권한**을 갖도록 설정하고 링크를 준비합니다. 
**반드시 읽기 권한을 부여해야합니다.** 시크릿 모드로 그 링크에 접근해서 읽기가 되는지 스스로 확인하세요.
2. **AI 비서(Gemini)와 학습 및 대화 링크 생성**: Gemini를 열고 "GitHub이 무엇인가?", "현대 AI 사회에서 대학생에게 GitHub을 알면 어떤 도움이 되는가?"를 질문합니다. 스스로 완벽히 이해할 때까지 질문한 뒤, 대화가 끝나면 우측 상단의 '공유' 기능을 사용해 **대화 전체 공개 링크**를 생성합니다.
3. **GitHub 가입 및 Public Repository 생성**: [GitHub.com](https://github.com/)에 가입한 뒤, 5주차 과제를 기록할 새로운 **전체 공개(Public) 레포지토리**를 생성합니다.
4. **마크다운(.md) 파일 작성 및 저장 (Commit)**: 생성한 레포지토리 내에서 새 파일(예: `week5.md`)을 생성하고, 본인이 요약한 배움의 내용과 **Gemini 대화 공개 링크**, 그리고 앞서 복사한 **구글 문서(단축키 목록) 공유 링크**를 본문에 모두 포함하여 작성합니다. 작성이 완료되면 하단의 **Commit changes** 버튼을 눌러 버전을 저장(Commit)합니다.
5. **Commit Hash(해시값) 복사**: 저장이 완료된 커밋 내역(History)으로 들어가, 해당 저장 지점의 고유 식별자인 영문/숫자 조합의 **해시(Hash)값**을 복사하여 구글 폼 제출을 준비합니다.

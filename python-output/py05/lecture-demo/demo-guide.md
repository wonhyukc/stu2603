# 5주차 강사 라이브 데모 가이드

강의 대본(`lecture-script.md`) S3의 라이브 시연 단계를 현장에서 재현하기 위한 체크리스트 및 조작 순서입니다.

---

## 사전 준비

- 크롬 브라우저가 설치된 노트북 또는 강의실 PC 준비
- 스마트폰에 구글 렌즈 앱 설치 확인
- 빔 프로젝터 또는 스크린 화면 공유 준비

---

## 데모 1: 에러창 번역하고 구글링으로 해결하기

**목적**: 에러를 만났을 때 번역 → 검색 → 해결의 5단계 흐름을 직접 보여줍니다.

1. 크롬 주소창에 `thisisnotarealwebsite12345.com` 같은 존재하지 않는 도메인을 입력합니다.
2. "ERR_NAME_NOT_RESOLVED" 또는 "이 사이트에 연결할 수 없음" 오류 화면을 학생들에게 보여줍니다.
3. 스마트폰 구글 렌즈 앱을 켜고 모니터 에러 화면을 향해 비추며 번역 모드를 실행합니다.
   - 번역 결과 예시: "DNS 주소를 찾을 수 없음"
4. 크롬 검색창에 `DNS 주소를 찾을 수 없음 크롬 해결법`을 입력하고 검색합니다.
5. 상단 결과의 블로그를 클릭해서 조치 방법을 화면에 보여줍니다.
6. 안내에 따라 크롬 설정(`chrome://settings/security`)을 열어 보안 DNS 설정을 토글합니다.
7. `F5` 새로고침 후 정상 결과 확인. (다른 정상 사이트로 이동해도 됩니다.)

**예상 소요 시간**: 약 7~8분

---

## 데모 2: 바탕화면 폴더 구조 만들기

**목적**: 트리 구조 파일 관리 방법을 직접 시연합니다.

1. 바탕화면이 파일로 가득한 상태를 보여줍니다. (또는 미리 준비한 지저분한 바탕화면 스크린샷 슬라이드 활용)
2. 바탕화면 빈 공간에서 마우스 오른쪽 클릭 → 새로 만들기 → 폴더
3. 폴더 이름: `2026_학교자료` 입력 후 엔터
4. 같은 방법으로 `2026_개인자료` 폴더 생성
5. `2026_학교자료` 폴더 열기 → 안에 `컴퓨팅사고` 폴더 생성 (하위 트리 구조 시각적으로 보여주기)
6. 임의의 파일을 드래그하여 해당 폴더 안으로 이동

**예상 소요 시간**: 약 5분

---

## 데모 3: 시크릿 모드로 공용 PC 안전하게 사용하기

**목적**: 공용 PC에서의 보안 사용법을 보여줍니다.

1. 일반 크롬 창에서 구글 계정 로그인 상태를 확인합니다.
2. `Ctrl + Shift + N`을 눌러 시크릿 창을 엽니다.
3. 시크릿 창 안내 화면 텍스트를 읽어줍니다: "방문 기록, 쿠키 및 사이트 데이터를 저장하지 않습니다."
4. 시크릿 창에서 구글 계정에 로그인합니다.
5. 이메일 확인 등 작업 시연
6. 시크릿 창을 닫습니다.
7. 일반 크롬 창으로 돌아와서 구글이 로그아웃 상태임을 보여줍니다.

**예상 소요 시간**: 약 5분

---

## 단축키 직접 시연

| 단축키 | 시연 방법 |
|:---|:---|
| `Ctrl + Z` | 메모장에서 글자 입력 후 삭제, 다시 Ctrl+Z로 복원 |
| `Ctrl + C / V` | 텍스트 드래그 선택 → 복사 → 다른 창 열어 붙여넣기 |
| `Ctrl + A` | 긴 문서에서 전체 선택 후 복사, 새 창에 붙여넣기 |
| `Ctrl + S` | 메모장 작성 후 저장. 파일 탐색기에서 저장된 위치 확인 |

## 에러 대처 참고 (현장 에러 발생 시)

| 상황 | 대처 방법 |
|:---|:---|
| 프로젝터 화면 공유 안 됨 | 슬라이드의 스크린샷 이미지로 대체 진행 |
| 인터넷 연결 불안정 | 구글 렌즈 앱 오프라인 번역 기능 사용 |
| 시크릿 모드 안 켜짐 | 설정 메뉴 → 개인정보 → 시크릿 모드 확인 |

## 1. 평가 목적 및 대상 (5주차 Pre-Midterm)

5주차는 학생들의 **"디지털 에러 직독직해 능력 — 에러 화면 번역 및 구글링으로 스스로 해결하는 과정 증명"**을 평가하는 주차입니다. 파이썬 문법이나 코드를 다루지 않으므로, 동료 평가는 오직 "에러 화면을 스스로 번역하고 검색하여 해결한 과정이 시각적으로 증명되는가"에만 집중합니다.

## 2. 채점 기준 (각 1점, 총 3점 만점)

평가자는 동료 학생이 제출한 첨부 파일(PDF 또는 이미지 묶음)을 열고 아래 세 개 문항에 대해 **Yes(1점) / No(0점)** 로 단순명료하게 채점합니다.

| 번호 | 단답형 객관적 평가 기준 (Yes/No) | 배점 | 확인 포인트 |
|:---:|:---|:---:|:---|
| **1** | 구글 폼으로 제출된(또는 공유된) **GitHub 레포지토리 링크**가 정상적으로 열리며, Public으로 전체 공개되어 있는가? | **1점** | 링크를 직접 클릭해 보세요. 404 Not Found 에러가 뜨거나 로그인을 요구한다면 Private 상태이므로 No입니다. |
| **2** | 레포지토리 내 마크다운(`.md`) 파일에 "GitHub이 무엇인가?" 요약과 함께 **Gemini 대화방 공유 링크**가 존재하는가? | **1점** | 파일 내용에 요약 텍스트와 AI(Gemini)와 반복 학습을 나눈 채팅 링크가 필수로 포함되어야 합니다. |
| **3** | 해당 요약 내용이 단순 검색 결과 복사-붙여넣기가 아닌, AI의 설명을 본인이 완전하게 이해하여 본인만의 언어로 다시 정리되었는가? | **1점** | 작성자가 지시된 주제에 맞게 내용을 소화하여 쉬운 텍스트로 풀어썼는지 점검합니다. |

## 3. 평가자(피드백 작성자) 특별 가이드라인

- **코딩 관련 기준 제외**: 5주차에서는 코딩 개념을 전혀 묻지 않습니다. 노트북 파일(`.ipynb`)이나 코드 존재 여부를 물어선 안 됩니다.
- **객관적 결과 중심**: 에러를 완벽하게 해결했는지, 번역이 정확한지 여부로 감점하지 마세요. 요구받은 과정 캡처 파일이 만들어져 권한 제한 없이 볼 수 있다면 감점 없이 **만점(3점)**을 주어야 합니다.
- 짧은 피드백 코멘트는 긍정적이고 실질적으로 작성합니다. (예: "에러 화면과 해결 과정이 순서대로 잘 정리되어 있어요. Before/After 캡처가 명확해서 보기 좋습니다!")

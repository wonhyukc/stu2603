## 1. 평가 목적 및 대상 (5주차 Pre-Midterm)

5주차는 학생들의 **"디지털 에러 직독직해 능력 — 에러 화면 번역 및 구글링으로 스스로 해결하는 과정 증명"**을 평가하는 주차입니다. 파이썬 문법이나 코드를 다루지 않으므로, 동료 평가는 오직 "에러 화면을 스스로 번역하고 검색하여 해결한 과정이 시각적으로 증명되는가"에만 집중합니다.

## 2. 채점 기준 (총 5점 만점)

평가자는 동료 학생이 제출한 첨부 파일(PDF 또는 이미지 묶음)을 열고 아래 네 개 문항에 명시된 배점에 따라 객관적으로 단답 채점합니다.

| 번호 | 단답형 객관적 평가 기준 (Yes/No) | 배점 | 확인 포인트 |
|:---:|:---|:---:|:---|
| **1** | 구글 폼으로 제출된 **GitHub 커밋 해시(Hash)**의 실제 커밋 시각이 마감 기한 이내인가? | **3점 / 0점** | 기한 내이면 3점 아니면 0점 중간 점수 없다. GitHub 레포지토리의 커밋 내역(History)에서 해당 해시의 시간을 확인합니다. (지각일 경우 나머지 항목은 정상 채점) |
| **2** | 제출된 **GitHub 레포지토리 링크**가 404 에러나 로그인 요구 없이 Public으로 정상 접속되는가? | **1점** | 링크를 직접 클릭해 봅니다. 안 보이면 0점입니다. |
| **3** | 마크다운 파일(`.md`) 내의 **단축키 구글 문서 공유 링크**가 접근 권한 요구 없이 정상적으로 열리고, **그동안 본인이 모르던 새로운 윈도우 단축키 5개 이상, 크롬 단축키 5개 이상**이 명시된 목록이 보이는가? | **1점** | 구글 문서 링크를 클릭하여 다른 로그인이나 권한 요청 창 없이 문서를 즉시 볼 수 있는지, 그리고 요구된 새로운 단축키 개수(윈도우 5개 이상, 크롬 5개 이상)가 충족되었는지 확인합니다. |
| **4** | 마크다운 파일(`.md`) 내의 **Gemini 대화 공유 링크**를 클릭했을 때, 사용자의 질문과 AI의 답변이 최소 3회(Turn) 이상 이어졌는가? | **1점** | 대화 링크를 직접 열어 사용자가 입력한 프롬프트 횟수를 세어봅니다. 주관적 요약 평가는 절대 하지 않습니다. |

## 3. 평가자(피드백 작성자) 특별 가이드라인

- **주관적 평가 금지**: 주관적 평가는 분쟁 소지가 있으므로 오직 위 4개의 객관적 조건만 확인하세요.
- 짧은 피드백 코멘트는 실질적인 체킹 사실만 기반으로 작성합니다. (예: "대화가 3회 이상 잘 이어졌고 링크도 정상 동작합니다.")

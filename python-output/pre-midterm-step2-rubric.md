# 공통 2단계 QA 루브릭 (K트랙 5~7주차 전용: Pre-Midterm)

> **🚨 [필수 원칙: 문항 단일 조건 원칙 (1문항 1점, 1조건)]**
> 동료 평가(peer-eval-rubric) 등 어떠한 평가 문항을 생성/검증할 때라도 **모든 문항은 항상 1점의 배점으로 '예/아니오'만 명확히 판별할 수 있는 단 1개의 조건**만 포함해야 합니다. A와 B를 동시에 묻는(And 조건) 문항은 절대 금지하며 반드시 2개의 별도 문항으로 분리하세요. (반만 참이거나 부분 점수가 발생하는 상황을 원천 차단할 것)

**평가 대상 파일** (특정 주차 폴더 `{week}/` 하위)
- `handout.md` — 필수
- `peer-eval-rubric.md` — 필수
*(※ `lab.ipynb`는 5~7주차 파이썬 코딩 배제 기간이므로 평가 대상 제외. `lecture-demo/`는 1단계에서 검증 완료. `lecture-script.md`는 작성하지 않음.)*

> **2단계 목적**: 1단계 PASS 이후 작성된 학생 배포 문서, 동료평가 루브릭이 "디지털 도구 활용"에 온전히 집중하고 있으며, 학생이 실생활 문제를 스스로 진단하고 앱/웹 서비스로 해결할 수 있도록 꼼꼼히 안내하는지 검증한다.
> **적용 범위**: 5주차·6주차·7주차만 해당. 8주차(중간고사 대체)는 본 루브릭 적용 제외.

**평가 기준 SSOT**
- `python-output/python-curriculum.md` — 5~7주차 학습 목표, 실습 미션 구조
- `docs/assignment-micro.md` — 해당 주차 과제 항목(과제 0.5·0.6·0.7 등) 전체 내용. handout.md의 과제 섹션은 이 파일을 SSOT로 삼아 항목이 누락 없이 반영되어야 한다.

**만점**: 100점
**PASS 기준**: 85점 이상 AND 파트별 PASS 하한 동시 충족
**FAIL 시**: 보완 항목 목록 출력 후 Auto-Fix
